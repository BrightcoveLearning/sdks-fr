Utilisation du plug-in SSAI

Le plugin SSAI vous permet de lire du contenu avec Dynamic Delivery et la publicité côté serveur.

Il existe deux options pour lire du contenu avec des publicités côté serveur :

  1. Fournir une URL pour le contenu SSAI
  2. Fournir une vidéo avec VMAP

Dans les deux cas, le plugin SSAI demandera la source VMAP, l'analysera et l'ajoutera automatiquement à VideoView.

Fournir une URL pour le contenu SSAI

Avec une URL d'élément SSAI, appelez la méthode suivante :

SSAIComponent.processVideo(String url)

Voici un exemple:

String ssaiUrl = "https://onceux.host.com/path/to/content.once"
SSAIComponent ssaiComponent = new SSAIComponent(context, brightcoveVideoView);
ssaiComponent.processVideo(ssaiUrl);

Fournir une vidéo avec VMAP

Pour cette option, vous fournirez une vidéo avec une source contenant une propriété VMAP avec l'URL de l'actif SSAI. Pour utiliser le plug-in SSAI, procédez comme suit :

  1. Pour demander du contenu VideoCloud SSAI, votre compte doit d'abord être configuré pour utiliser Dynamic Delivery et SSAI. Pour plus de détails, consultez le Présentation de Video Cloud SSAI document.
  2. Ensuite, vous devrez créer un objet de configuration d'annonce (adConfig). Pour plus de détails, consultez le Configuration des paramètres d'annonce côté serveur document. Veuillez noter que le Brightcove Native SDK pour Android ne contient en lui-même aucune fonctionnalité permettant de créer ou de gérer des adConfigs.
  3. Utilisez les adConfig id propriété en tant que paramètre d'URL (ad_config_id ) ajouté à l'appel de la méthode Catalog : 
    findVideoById

    ou

    findVideoByReferenceId
  4. Après avoir récupéré l'objet Video avec la méthode findVideo, appelez la méthode suivante dans le plugin SSAI : 
    SSAIComponent.processVideo(Video ssaiVideo)

Voici un exemple:

Catalog catalog = new Catalog(eventEmitter, myAccountId, myPolicyKey);
SSAIComponent plugin = new SSAIComponent(appContext, getBrightcoveVideoView());

// Set the ad_config_id URL parameter
HttpRequestConfig httpRequestConfig = new HttpRequestConfig.Builder()
        .addQueryParameter("ad_config_id", myAdConfig)
        .build();

// Find the Video by ID
catalog.findVideoByID(myVideoId, httpRequestConfig, new VideoListener() {
        @Override
        public void onVideo(Video video) {

        // Process the Video
        plugin.processVideo(video);
    }
});

Utilisation des annonces compagnons

Si vos annonces ont des compagnons, vous pouvez facilement les afficher dans votre application en fournissant un ViewGroup pour conserver l'image Companion, en appelant :

SSAIComponent.addCompanionContainer(ViewGroup companionContainer)

Voici un exemple. Tout d'abord, ajoutez le FrameLayout à votre mise en page d'activité ou de fragment :

// my_activity.xml
<RelativeLayout xmlns:android="http://schemas.android.com/apk/res/android"
                xmlns:app="http://schemas.android.com/apk/res-auto"
                android:layout_width="match_parent"
                android:layout_height="match_parent">
 ...
  <FrameLayout
        android:id="@+id/ad_frame"
        android:layout_width="300dp"
        android:layout_height="250dp"
        android:visibility="invisible" />
…
</RelativeLayout>

Ensuite, utilisez cela FrameLayout pour le passer au plugin :

ViewGroup viewGroup = findViewById(R.id.ad_frame);
plugin.addCompanionContainer(viewGroup);

// process your Video
plugin.processVideo(video);

Gestion des événements

Pour les événements spécifiques à SSAI, vous pouvez les écouter en utilisant le EventEmitter. Ces événements se trouvent dans le SSAIEventType classer.

Les START_AD_SEGMENT et END_AD_SEGMENT les événements ont été remplacés par EventType.AD_BREAK_STARTED et EventType.AD_BREAK_COMPLETED respectivement. Pour plus de détails, voir Appendice B et Annexe C.

Pour obtenir les positions relatives et absolues, obtenez le com.brightcove.ssai.AdPod avec le SSAIEvent.AD_POD clé. Voici un exemple:

private void registerEventHandlers() {
  EventEmitter eventEmitter = brightcoveVideoView.getEventEmitter();
  eventEmitter.on(EventType.AD_BREAK_STARTED, new EventListener() {
    @Override
    public void processEvent(Event event) {
      Object adPodObject = event.getProperties().get(SSAIEvent.AD_POD);
      if (adPodObject instanceof AdPod) {
        AdPod adPod = (AdPod) adPodObject;
        adPod.getRelativeStartPosition();
        adPod.getAbsoluteStartPosition();
        adPod.getAbsoluteEndPosition();
      }
    }
  });
}

Gestion des erreurs

Toutes les erreurs seront signalées au développeur à l'aide du EventType.ERROR événement, comme indiqué ici :

eventEmitter.on(EventType.ERROR, new EventListener() {
  @Override
  public void processEvent(Event event) {
    String errorMessage = event.getProperty(Event.ERROR_MESSAGE, String.class);
    Throwable error = event.getProperty(Event.ERROR, Throwable.class);
    Log.w(TAG, String.format("Received the error message: %s with error: %s", errorMessage, error));
  }
});

Bibliothèque IAB Android

Le plugin SSAI utilise une bibliothèque interne chargée d'analyser l'objet VMAP. Si votre application utilise directement des objets de cette bibliothèque (package com.brightcove.iab), passez en revue les modifications apportées à la dernière version.

Notes de version

Bibliothèque IAB Android 1.5.0

  • Ajout des méthodes suivantes pour renvoyer des valeurs chronométrées en microsecondes :
    • com.brightcove.iab.ssai.BrightcoveSSAI.getContentLengthUs():long
    • com.brightcove.iab.ssai.BrightcoveSSAI.getPayloadLengthUs():long
    • com.brightcove.iab.vmap.AdBreak.getTimeOffsetUs():long
    • com.brightcove.iab.vmap.AdBreak.getDurationUs():long
    • com.brightcove.iab.vast.Linear.getDurationUs():long
    • com.brightcove.iab.vast.Linear.getSkipOffsetUs():long
    • com.brightcove.iab.vast.NonLinear.getMinSuggestedDurationUs():long
    • com.brightcove.iab.vast.Icon.getDurationUs():long
    • com.brightcove.iab.vast.Icon.getOffsetUs():long
    • com.brightcove.iab.vast.Tracking.getOffsetUs():long
  • Ajoutée AdBreak.POST_ROLL_TIME_OFFSET_US pour spécifier la valeur d'un AdBreak post-roll lors de l'utilisation AdBreak.getTimeOffsetUs().
  • Ajouté le BrightcoveSSAI getters de propriété suivant la convention de nommage camelcase.
  • Obsolète le BrightcoveSSAI getters de propriété ne suivant pas la convention de nommage camelcase.
