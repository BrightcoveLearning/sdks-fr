Aperçu

Le Brightcove Player SDK pour Android résout la plupart des problèmes liés à la lecture vidéo HLS sur les appareils Android. Pour plus de détails sur cette solution, consultez le Lecture HLS avec le Player SDK pour Android document.

Si vous devez utiliser Android MediaPlayer pour la lecture HLS, vous pouvez rencontrer certaines limitations. Voir la section ci-dessous pour plus de détails sur ces limitations.

Limites connues

Si vous utilisez Lecture HLS avec le Player SDK pour Android , les limitations de cette section ne s'appliquent pas.

Vous ne rencontrerez les problèmes de cette section que si vous choisissez d'utiliser Android MediaPlayer pour la lecture HLS sans la solution de Brightcove. Voici les limites :

  • Android 4.x
    • Les flux HLS ne recherchent que le début des segments en raison d'un bogue du système d'exploitation.

      Par conséquent, les annonces midroll ne sont prises en charge qu'avec les flux non HLS ou avec le Brightcove Player SDK (Android 4.1 ou plus récent).

    • Les flux HLS ne signalent pas systématiquement la durée correcte de la vidéo en raison d'un bogue du système d'exploitation.

      Le Brightcove Player SDK (Android 4.1 ou plus récent) n'a pas cette limitation.

    • Le minuteur, le compteur, la superposition du bouton En savoir plus et certains événements AdEvent sont manquants avec les midrolls Google IMA en raison d'un bogue signalé dans le SDK Google IMA.

    • Nous avons constaté des problèmes avec le contenu Widevine sur certains appareils Samsung. Les équipes Widevine conseillent de passer à la dernière version d'Android.
  • Android 4.4.2
    • La lecture commencera, s'arrêtera après quelques secondes, puis recommencera depuis le début.

FAQ

Comment puis-je faire fonctionner HLS sur Android ?

Pour pouvoir fournir un streaming adaptatif sur Android, le SDK Player de Brightcove fournit une fonctionnalité de lecture HLS pour Android 4.1 et versions ultérieures.

Et HLSe ?

HLSe a les mêmes limitations et recommandations que ci-dessus.
