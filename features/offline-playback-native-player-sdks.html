    Lecture hors ligne avec les kits SDK du lecteur natif

    Dans cette rubrique, vous apprendrez comment fonctionne la lecture hors ligne avec les SDK Brightcove Native Player.

    Présentation

    La lecture hors ligne avec les SDK Brightcove Native Player pour iOS et Android permet aux éditeurs d'atteindre leurs spectateurs de toute nouvelle manière tout en assurant la sécurité de leur contenu. Nos kits SDK offrent des solutions puissamment simples pour les problèmes complexes de lecture hors ligne, y compris la gestion des téléchargements, la gestion des catalogues hors ligne, l'analyse, la gestion des données DRM et bien sûr la lecture.

    Grâce à la lecture hors ligne, les utilisateurs peuvent télécharger du contenu vidéo protégé par DRM et effacer (non crypté) sur leurs appareils et le regarder lorsqu'ils ne sont pas connectés.

    Exigences

    Les kits SDK Brightcove Native Player prennent en charge la lecture hors ligne avec les versions suivantes :

    Version du kit SDK natif de Brightcove

    • SDK natif pour iOS 6.0.1+
    • SDK natif pour Android 5.0.3+

    Version du système d'exploitation

    • iOS 10.0+, mais 10.3+ est recommandé
    • Android 5.0+

    Configuration

    Pour commencer, procédez comme suit :

    • Contactez votre responsable de compte pour activer votre compte pour la lecture hors ligne.
    • Assurez-vous d'utiliser Dynamic Delivery lors de l'ingestion de vos vidéos.
    • Préparez chaque vidéo que vous souhaitez activer pour le téléchargement.

    Livraison de contenu

    Le streaming vidéo tire parti de la fonction de streaming multi-débit de Video Cloud qui détecte la bande passante à la source et offre la meilleure qualité pour l'appareil. La fonction de lecture hors connexion prend en charge les types de diffusion en continu suivants :

    • Android: Diffusion dynamique adaptative sur HTTP (DASH)
    • iOS: Diffusion en continu HTTP Apple (HLS)

    Sécurité

    La protection du contenu téléchargé est la première place à l'esprit avec notre solution hors ligne. Les kits SDK Brightcove Native Player fournissent les fonctionnalités suivantes pour garantir que votre contenu est stocké en toute sécurité sur les appareils des spectateurs :

    Contenu protégé par DRM

    • Pour le contenu protégé par DRM, les SDK natifs utilisent les technologies suivantes :
      • Android: DASH avec Widevine Modular

      • iOS: HLS avec FairPlay

    • Le contenu vidéo ne peut être visionné que sur l'appareil et l'application qui a téléchargé le contenu.

    • Le contenu vidéo ne peut être regardé que jusqu'à l'expiration d'une date et d'une heure spécifiques.

      Les SDK natifs permettent aux utilisateurs de louer ou d'acheter des vidéos. L'expiration s'applique aux locations et est définie dans la licence DRM à l'aide de l'API SDK natif pour le téléchargement de vidéos. En revanche, l'expiration définie dans le jeton Fastly s'applique au contenu en continu.

    • Les URL vidéo retournées au client expireront afin que les autres ne puissent pas les utiliser.

    Tout le contenu

    • Lorsque l'application est supprimée, tout le contenu téléchargé est supprimé.
      Pour Android, cela est vrai tant que l'emplacement de stockage par défaut n'est pas remplacé.
    • Votre application peut supprimer du contenu stocké localement à tout moment.

    Gestion des téléchargements

    Les kits SDK Brightcove Native Player traitent la complexité du téléchargement de contenu sur l'appareil d'un spectateur, simplifiant grandement le processus pour les applications natives.

    • Les éditeurs peuvent choisir la qualité de format associé du contenu à télécharger.
    • Les légendes peuvent être téléchargées et affichées hors ligne.
    • Plusieurs pistes audio sont prises en charge par les kits SDK natifs. Pour plus d'informations sur le téléchargement de pistes supplémentaires, consultez les sections suivantes :
      • Plusieurs pistes audio pour la lecture hors ligne avec le SDK natif pour Android
      • Guide du développeur d'applications iOS pour la lecture hors ligne.

    Fonctionnalité

    Les fonctionnalités suivantes peuvent être ajoutées à vos applications pour prendre en charge la gestion des téléchargements :

    • Afficher le contenu
    • Vérifier la taille du téléchargement
    • Afficher l'état du téléchargement
    • Gestion des téléchargements
    • Afficher l'espace de stockage local
    • Pause/Rsume/Annuler le téléchargement

    Afficher le contenu

    Affichez une liste des contenus disponibles en streaming ou en téléchargement.

    Lorsque vous définissez la offline_enabled propriété d'une vidéo sur true , les SDK natifs marquent cette vidéo comme téléchargeable.

    Préparer des vidéos

    Pour marquer une vidéo comme téléchargeable, vous devez effectuer les opérations suivantes :

    1. Si vous ne l'avez pas encore fait, contactez votre responsable de compte pour activer la lecture hors ligne de votre compte.

    2. Pour chaque vidéo que vous souhaitez télécharger, définissez hors connexion activée en effectuant l'une des opérations suivantes :

      Utiliser Video Cloud Studio

      Utilisez Video Cloud Studio pour activer la lecture hors ligne.

      Utiliser l'API CMS

      Vous pouvez également utiliser l' API CMS pour définir le champ offline_enabled sur la valeur true.

      Voici un exemple utilisant curl :

        curl --header "Authorization: Bearer $oauth_token" --request PATCH --data '
  {"offline_enabled" : true}
  ' https://cms.api.brightcove.com/v1/accounts/your account id/videos/your video id

    Lorsque vous utilisez un contenu clair, consultez la section Meilleures pratiques pour obtenir des conseils sur l'affichage de votre contenu.

    Vérifier la taille du téléchargement

    Brightcove recommande qu'avant de demander un téléchargement, assurez-vous qu'il y a suffisamment d'espace dans le stockage de l'appareil pour le téléchargement. Pour obtenir une estimation de la taille totale du téléchargement de la vidéo, procédez comme suit :

    • Pour iOS, consultez les informations de la section Vérifier la taille du téléchargement du Guide du développeur d'applications iOS pour la lecture hors ligne.

    • Pour Android, vous pouvez obtenir une estimation dans le rappel OnDownloadStarted à partir du DownloadEventListener. Voici un exemple de la façon de définir l'écouteur d'événement :

        MediaDownloadable.DownloadEventListener downloadEventListener = new MediaDownloadable.DownloadEventListener() {
  //Your implementation.
  }
  OfflineCatalog catalog = new OfflineCatalog(context, eventEmitter, accountId, policyKey);
  catalog.addDownloadEventListener(downloadEventListener);

     

    Afficher l'espace de stockage local

    Informez l'utilisateur lorsqu'il n'y a pas assez de stockage local pour terminer le téléchargement de l'élément en cours et des éléments de la file d'attente.

    Les kits SDK fournissent la taille estimée et la progression afin que vous puissiez déterminer s'il y a suffisamment d'espace pour les téléchargements.

    Afficher l'état du téléchargement

    Affichez le téléchargement en cours ainsi que son état.

    La taille totale estimée, l'état actuel du téléchargement et le pourcentage des informations de progression du téléchargement seront renvoyés par les SDK.

    Pour les appareils Android, l'état du téléchargement est également affiché dans la zone de notification. Le titre de la notification est défini sur le titre de la vidéo. La notification est supprimée si le téléchargement est interrompu ou annulé. Pour plus de détails, consultez les extraits de code utilisant le document Native SDK pour Android .

    Gestion des téléchargements

    Téléchargement de plusieurs vidéos

    Les kits SDK natifs permettent de mettre en file d'attente plusieurs vidéos pour les téléchargements. Par exemple, un utilisateur peut télécharger plusieurs épisodes d'une saison et les SDK natifs gèrent le processus en fournissant l'état du téléchargement pour chaque ressource.

    Si votre application cible iOS 11.0 - 11.2, nous ne recommandons pas les téléchargements simultanés car le comportement de pause et de reprise n'est pas fiable avec plusieurs téléchargements simultanés. Le problème a été résolu avec iOS 11.3.

    Téléchargement de pistes secondaires (iOS)

    Avec iOS 13, Apple a modifié le fonctionnement des téléchargements. Pour plus d'informations, consultez le Guide du développeur iOS App Developer to Video Downloading and Offline Playback Guide.

    En raison de ce changement, l'exemple d'application iOS Téléchargement hors connexion peut sembler retélécharger des vidéos avec plusieurs pistes audio. Cela est dû au fait que la progression du téléchargement est donnée pour chaque piste. Ainsi, la barre de progression redémarre à partir de zéro pour chaque piste audio supplémentaire.

    Voici un bref aperçu de AVMediaSelectionOptions, sous-titres et pistes audio alternatives : Ajout de sous-titres et de pistes audio alternatives

    Déterminer le téléchargement total

    Existe-t-il un moyen d'obtenir didProgressTo pour le téléchargement total pour un jeton vidéo ?

    Non. Pour un jeton vidéo hors ligne, il n'y a pas de valeur de progression de téléchargement agrégée. Le développeur de l'application détermine le nombre de téléchargements d'actifs qui se produiront. Le premier téléchargement d'actifs est la vidéo principale et les pistes audio et texte préférées. Il s'agit généralement du plus grand téléchargement et la progression passera de 0,00 à 100,00.

    Si une demande de téléchargement inclut des sélections de médias supplémentaires (que le développeur de l'application demande explicitement), le temps total de téléchargement est simplement celui de la vidéo principale et de la sélection multimédia préférée, plus celle de chaque sélection de média supplémentaire. Lors de la représentation de la progression totale dans l'interface utilisateur, choisissez une valeur d'échelle pour chaque téléchargement de telle sorte que la somme de tous les téléchargements totalisera 100 % et additionne les temps de téléchargement échelonné de chaque sélection de médias.

    Identification des types d'objets

    Existe-t-il un moyen d'identifier à quel type d'objet (vidéo ou piste de texte) chaque didProgressTo événement lié ?

    Oui. Les téléchargements supplémentaires sont des objets Apple Media Selection Option (AVMediaSelectionOption), et non des pistes. L'objet de sélection de média en cours de téléchargement est identifié dans le rappel de progression. (BCOVofflineVideoManagerDelegate.h)

    Apple fournit des méthodes pour examiner les attributs d'une option de sélection de médias. Pour plus de détails, consultez l'option AvMediaElectionOptiond'Apple.

    Vous pouvez voir les valeurs de métadonnées dans la console de débogueur Xcode lors de l'exécution de l' exemple d'application OfflinePlayer (ne soyez pas trompé par les deux options anglaises - c'est juste un exemple d'application) 

    OfflinePlayer[523:355259] AVMediaSelection option 0 | legible display name: English
OfflinePlayer[523:355259] AVMediaSelection option 0 | audible display name: English
OfflinePlayer[523:355259] AVMediaSelection option 1 | legible display name: English
OfflinePlayer[523:355259] AVMediaSelection option 1 | audible display name: English

    Vous pouvez voir comment cela se fait dans le code DownloadManager.

    Pause/Rsume/Annuler le téléchargement

    Autoriser l'utilisateur à suspendre, reprendre ou annuler le téléchargement de contenu.

    La fonctionnalité de téléchargement reprendra là où elle s'était arrêtée si l'utilisateur interrompt ou reprend ou si le téléchargement est interrompu. Par exemple, si vous perdez la connexion Wi-Fi pendant le téléchargement, il reprendra là où il s'est arrêté lorsque vous êtes reconnecté.

    Si vous annulez un téléchargement vidéo, les progrès réalisés seront effacés.

    Gestion des catalogues locaux

    Les kits SDK Brightcove Native Player simplifient le processus de gestion du contenu local téléchargé.

    Les fonctionnalités suivantes peuvent être ajoutées à vos applications pour prendre en charge la gestion des catalogues locaux :

    • Afficher le contenu téléchargé
    • Lire du contenu local
    • Supprimer le contenu local

    Afficher le contenu téléchargé

    Afficher tous les épisodes téléchargés pour chaque show/saison.

    Les kits SDK renvoient des métadonnées et des détails de lecture pour tout le contenu téléchargé. Ces métadonnées comprennent le titre de la vidéo, la description, les miniatures, les sous-titres disponibles, les pistes audio disponibles, etc.

    Il existe de nouvelles propriétés de métadonnées spécifiques à la lecture hors ligne, dont les suivantes :

    Contenu protégé par DRM

    • Identifiant de jeton
    • Date d'expiration de la licence

    Tout le contenu

    • Heure de début et de fin du téléchargement
    • Nom de la vignette téléchargée avec son URL de fichier
    • Nom de l'affiche téléchargé avec son URL de fichier
    • URL de fichier relative et dérivée du bundle vidéo

    Vous pouvez ensuite filtrer les résultats de votre implémentation spécifique.

    Vous pourrez voir le temps restant pour une licence de location.

    Supprimer le contenu local

    Autoriser les utilisateurs à supprimer du contenu du stockage local.

    Les kits SDK exposent une méthode de suppression d'une vidéo avec ses métadonnées et les ressources associées.

    Lecture de contenu

    Les kits SDK natifs vous permettent de lire du contenu dans les situations suivantes :

    • Lecture d'une vidéo pendant qu'elle est téléchargée.
    • Lecture d'une vidéo une fois qu'elle a été téléchargée. Le contenu téléchargé peut être lu localement lorsque l'appareil est hors ligne ou en ligne. La vidéo hors ligne ne peut être lue qu'à partir du stockage local.
    • Si vous choisissez de diffuser une version en ligne de la vidéo, vous devez récupérer un nouvel objet vidéo à partir du service de lecture.

    Statistiques

    Des analyses ont été introduites pour mesurer les données liées au flux hors ligne. Pour plus d'informations, consultez le document Analytics lié à la lecture hors connexion avec les SDK natifs .

    Remarques

    Les remarques suivantes s'appliquent à la fonctionnalité de lecture hors connexion prise en charge par les SDK natifs :

    Tous

    Advertising

    Ni la publicité côté client ni côté serveur n'est prise en charge par la lecture hors ligne.

    HLSe

    Actuellement, le contenu HLSE n'est pas pris en charge avec la lecture hors ligne à l'aide du SDK natif Brightcove pour Android, iOS ou tvOS.

    Mode basse consommation

    Le mode de faible consommation sur les appareils peut affecter certaines fonctionnalités, y compris le téléchargement. Pour plus d'informations, consultez les sections suivantes :

    • Gestion de l'alimentation Android 9 (API niveau 28)
    • Mode faible consommation sur votre iPhone
    • Réagir en mode basse consommation sur les iPhones

    Vidéos partagées

    Les vidéos partagées peuvent être téléchargées en mode hors connexion uniquement si la vidéo d'origine peut l'être. La propriété est héritée de la vidéo originale et ne peut pas être modifiée uniquement pour la vidéo partagée.

    Audio uniquement

    Les kits SDK natifs prennent en charge le contenu audio uniquement avec la lecture hors connexion. Vous contrôlez ce que les spectateurs voient, qu'il s'agisse d'une image d'affiche ou d'un écran personnalisé.

    Le SDK natif pour Android nécessite un VideoView, et le SDK natif pour iOS nécessite un UIView. Pour cette raison, l'audio d'arrière-plan n'est pas pris en charge. Cela signifie que l'audio ne continue pas à jouer lorsqu'un utilisateur envoie l'application à l'arrière-plan.

    Android

    Suppression de l'application

    Lorsque l'application est supprimée, tout le contenu téléchargé est supprimé. Ceci est vrai tant que l'emplacement de stockage par défaut n'est pas remplacé.

    iOS

    AirPlay

    En tant que fonctionnalité de sécurité, vous ne pouvez pas diffuser une vidéo HLS hors ligne vers un périphérique AirPlay pour la lecture. Apple confirme qu'il s'agit d'une AVFoundation limitation.

    FAQ

    Ci-dessous vous trouverez les réponses à quelques questions générales.

    Contenu protégé par DRM

    L'utilisateur peut-il changer l'horloge système pour contourner la date d'expiration ?

    Android: L'expiration absolue détermine la date et l'heure jusqu'à laquelle une licence est valide. Ceci est appliqué par Widevine et la couche DRM Android. Théoriquement, la couche DRM dans Android doit empêcher l'utilisateur de contourner la période d'expiration. Tous les problèmes que nous avons pu trouver sont décrits dans les notes de mise à jour du système d'exploitation Android.

    iOS: L'expiration de la licence est appliquée par FairPlay. iOS gère toutes les vérifications des changements d'horloge, etc. Une application a la possibilité de vérifier l'expiration de la licence pour alerter l'utilisateur que la licence a expiré. Il peut être préférable de lire simplement une vidéo expirée et de recevoir une erreur de lecture.


    La licence et les vidéos hors ligne continueront à être chargées alors que l'application est en arrière-plan ?

    Android: Oui. Généralement, la licence est téléchargée immédiatement, mais même la licence peut être téléchargée pendant que l'application est back-base.

    iOS: Le chargement des licences devrait se produire au premier plan. Nous avons une méthode pour précharger la licence. Le téléchargement du contenu vidéo principal peut se produire lorsque l'application est en arrière-plan.

    Tout le contenu

    Quel est le format vidéo utilisé pour télécharger du contenu hors ligne ?

    Android: DASH protégé par DRM ou DASH non chiffré.

    iOS: HLS à la vapeur. Vous pouvez spécifier un débit pour le téléchargement afin d'utiliser une variante spécifique.


    Les vidéos téléchargées sont-elles lues localement ou en streaming lorsque l'utilisateur est en ligne ?

    Une vidéo téléchargée peut être lue en ligne ou hors ligne, mais elle utilisera toujours la vidéo téléchargée. Si vous choisissez de lire une version en ligne de la vidéo, vous devez récupérer un nouvel objet vidéo à partir du service de lecture. La vidéo hors ligne ne peut être lue qu'à partir du stockage local.


    Meilleures pratiques

    Les instructions suivantes devraient vous aider lors du développement de votre application pour la lecture hors ligne.

    Affichage des boutons pause/annulation

    Les boutons Pause et Annuler doivent s'afficher lorsque le téléchargement de la vidéo a démarré.

    Android : Vous devez afficher les boutons pause et annulation dans le cadre de l'appel du onDownloadStarted rappel dans le DownloadEventListener. Pour plus d'informations, consultez l' exemple d'application Lecture hors connexion.

    iOS : Pour plus d'informations sur la création d'une application pour la lecture hors ligne, reportez-vous au Guide du développeur d'applications iOS.

    Affichage du contenu clair

    Si votre compte n'est pas compatible DRM ou si vous travaillez avec un contenu clair, les kits SDK natifs Brightcove n'exécuteront pas de demande de licence côté serveur. Par conséquent, le contenu clair n'est pas considéré comme disponible à la location ou à l'achat. Dans la plupart des cas, pour un contenu clair, il est recommandé de masquer les boutons de location et d' achat et d'afficher un seul bouton de téléchargement à la place.

    Activation de la fonction

    Contactez votre responsable de compte pour en savoir plus sur la tarification et pour activer la fonctionnalité de lecture hors connexion pour votre compte.

    dépannage

    Les conseils suivants peuvent vous aider à étudier les problèmes lors du téléchargement de vidéos.

    Une erreur peut se produire lorsqu'un utilisateur tente de télécharger une ressource à un débit binaire supérieur à celui que l'appareil peut gérer en raison d'un réseau lent.

    Pour éviter cela, vous pouvez ajouter du code pour vérifier les capacités de l'appareil et empêcher les périphériques ayant des connexions lentes de tenter de télécharger des ressources à débit élevé. Vous pouvez tester avec un appareil réel tout en bloquant la connexion, et afficher le trafic réseau avec l'outil proxy Charles .

    Android

    Pour plus d'informations sur la gestion des erreurs, consultez les messages d'erreur du document Native SDK pour Android . Cela inclut les messages d'erreur liés à la lecture hors ligne.

    iOS

    Pour plus d'informations sur la gestion des erreurs, consultez le document Gestion des erreurs avec le SDK natif pour iOS .

    Lorsque le code de lecture hors ligne reçoit une erreur, nous l'attrapons et lançons une erreur NSError. Pour obtenir l'erreur sous-jacente, vous pouvez essayer ceci :

      NSError *underlyingError = initialError.userInfo[NSUnderlyingErrorKey];

    Pour plus d'informations, consultez le Guide des développeurs d'applications iOS pour la lecture hors ligne.

    Problèmes connus

    Pour obtenir la liste des problèmes connus pour les systèmes d'exploitation Android et iOS qui affectent la lecture hors connexion avec les SDK natifs, consultez le document de support Problèmes connus .
