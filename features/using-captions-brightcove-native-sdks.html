Aperçu

Les sous-titres codés fournissent des informations supplémentaires sur votre vidéo aux personnes qui souhaitent y accéder. Les sous-titres sont nécessaires pour que les sourds et malentendants accèdent à la partie audio de vos vidéos. Si vous n'êtes pas familiarisé avec les sous-titres, consultez le Aperçu: Ajouter des sous-titres à une vidéo document.

Il est important de comprendre quels types de sous-titres fonctionnent le mieux pour chaque format vidéo. Commencez par revoir le Formats vidéo et sous-titrage document. Brightcove recommande d'utiliser des vidéos HLS ou DASH avec des sous-titres WebVTT.

Lorsque vous utilisez le API d'ingestion dynamique pour ajouter des sous-titres SRT ou DFXP aux vidéos Dynamic Delivery, les sous-titres seront automatiquement convertis au format WebVTT. En savoir plus sur le ajout de pistes de texte avec l'API Dynamic Ingest.
Le SDK natif pour Android ne prend pas en charge les balises WebVTT Ruby (<ruby> et <rt>).

Légendes

Avec la vidéo HLS et DASH, il existe deux options pour la lecture des sous-titres :

  • WebVTT dans le manifeste
  • WebVTT externe (side-car)

WebVTT dans le manifeste

Lorsque vous utilisez des sous-titres WebVTT In-manifest avec des vidéos HLS ou DASH, vos sous-titres fonctionneront simplement. Vous n'avez rien d'autre à faire pour implémenter ces sous-titres.

Les sous-titres WebVTT dans le manifeste sont ingérés avec vos vidéos. Vous ne pouvez créer des fichiers in-manifest que si vous utilisez Dynamic Ingest avec Dynamic Delivery. Pour plus de détails, voir ce qui suit :

  • Ingérer des fichiers WebVTT
  • Ingérer des sous-titres à l'aide de l'API Dynamic Ingest

Les sous-titres sont reconnus comme dans le manifeste lorsque le m3u8 le fichier contient la balise suivante :

#EXT-X-MEDIA:TYPE=SUBTITLES

Exemple

Scénario Emplacement de la piste de texte Type de piste de texte #EXT-X-MEDIA:TYPE=balise SOUS-TITRES
Vidéo ingérée par Dynamic Delivery
API d'ingestion dynamique		 CDN Segmenté Oui

Remarques:
  • L'utilisation du module multimédia dans Studio pour ajouter des sous-titres à une vidéo ne créera pas de sous-titres dans le manifeste.
  • L'utilisation de l'intégration héritée ne crée pas de sous-titres dans le manifeste.

WebVTT externe (side-car)

Les sous-titres WebVTT externes sont des fichiers texte distincts de la vidéo. Vous pouvez associer des fichiers de sous-titres à une vidéo comme suit :

  • Ajout de sous-titres à l'aide du module multimédia
  • Ajout de sous-titres à l'aide de l'API CMS

Android

Les WebVTTSampleApp vous montre comment ajouter des fichiers de sous-titres WebVTT pour la lecture vidéo.

iOS/tvOS

Lorsque vous utilisez des sous-titres WebVTT externes, vous devez utiliser le Sous-titres Sidecar fonctionnalités pour le SDK Player pour iOS et tvOS.

Lorsque vous récupérez des vidéos de votre bibliothèque Video Cloud à l'aide du BCOVPlaybackService , vos vidéos et pistes de texte seront formatées pour les sous-titres Sidecar.

Pour les clients Brightcove Player sans Video Cloud ou si vous fournissez vos propres vidéos, vous devez structurer les pistes de texte comme indiqué dans le Remplissage manuel des données de sous-titres section de la documentation Sidecar Subtitles.

Synchronisation des sous-titres WebVTT

Si vous utilisez des sous-titres WebVTT, assurez-vous que la vidéo et l'audio sont synchronisés avec les sous-titres. Apprenez à faire cela dans le Synchronisation des sous-titres WebVTT document.

Légendes de style

Les légendes WebVTT peuvent être stylisées pour correspondre aux préférences de votre entreprise. Pour plus de détails, consultez le Section des légendes de style du document Ajouter des sous-titres aux vidéos.
