    Implémentation des annonces côté serveur avec les kits SDK natifs

    Dans cette rubrique, vous apprendrez comment lire des annonces côté serveur avec les SDK Brightcove Native Player, à partir de vidéos ingérées pour la diffusion dynamique.

    Présentation

    L'insertion d'annonces côté serveur (SSAI) vous permet d'incorporer des annonces dans vos vidéos afin qu'elles ne puissent pas être bloquées par les bloqueurs d'annonces dans le navigateur. Dynamic Delivery est le système d'ingestion et de livraison de nouvelle génération qui réduit votre encombrement de stockage et emballe dynamiquement les supports. En savoir plus sur l'utilisation de la livraison dynamique pour vos vidéos.

    SSAI fonctionne avec des contenus DRM et non-DRM.

    Remarques :
    • Pour utiliser SSAI, votre compte Video Cloud doit être configuré pour la livraison dynamique et activé pour SSAI. Contactez votre gestionnaire de compte pour commencer à utiliser cette fonctionnalité.
    • SSAI fonctionne avec la vidéo à la demande (VOD) et prend en charge la lecture en direct.
    • Pour obtenir la liste des limitations, reportez-vous au document SSAI Overview .

    Toute vidéo que vous récupérez à partir de Video Cloud qui a été ingérée avec Dynamic Delivery inclut les annonces spécifiées dans le fichier VMAP dans la configuration de votre annonce.

    Pour lire des annonces côté serveur avec votre contenu vidéo stocké dans Video Cloud, procédez comme suit :

    1. Créer une configuration d'annonce

    2. Créez votre application :

      • implémentation Android
      • Mise en œuvre iOS
      • Mise en œuvre de TVOS
    3. Lire une vidéo avec des annonces

    Créer une configuration d'annonce

    La configuration de l'annonce définit divers aspects de la lecture SSAI, notamment une URL vers votre fichier VMAP (Video Multiple Ad Playlist), des balises et d'autres configurations. Pour créer une configuration d'annonce, procédez comme suit :

    1. Dans Video Cloud Studio, développez le menu ADMIN et sélectionnez Paramètres publicitaires côté serveur.

    2. Ajoutez des informations pour la configuration de votre annonce et sélectionnez Enregistrer.

    Pour plus d'informations, consultez le document Configuration des paramètres publicitaires côté serveur .

    Pour créer une configuration d'annonce à l'aide de l'API SSAI, consultez le document API Video Cloud SSAI Ad Config .

    implémentation Android

    Suivez ces étapes au sein de votre application pour obtenir la configuration de votre annonce et lire votre vidéo :

    1. Dans Android Studio, ouvrez votre fichier MainActivity.java .
    2. Ajoutez les instructions d'importation suivantes : 
      import com.brightcove.ssai.SSAIComponent;
import com.brightcove.player.network.HttpRequestConfig;
    3. Définissez des constantes pour l'ID de compte, la clé de stratégie, l'ID vidéo et l'ID de configuration de votre annonce. 
      private String accountId = "your account id";
private String policyKey = "your policy key";
private String videoId = "your video id";
private String adConfigId = "your ad configuration id";
      Découvrez comment obtenir ou générer une clé de stratégie dans le document Clés de stratégie .
    4. Créez une instance du service de catalogue, qui fournit des méthodes asynchrones pour récupérer des données à partir de l'API de lecture. Créez une instance du composant SSAI. 
      Catalog catalog = new Catalog(eventEmitter, myAccountId, myPolicyKey);
SSAIComponent plugin = new SSAIComponent(appContext, getBrightcoveVideoView());
    5. Définissez l'AdConfigID en tant que paramètre URL. 
      HttpRequestConfig httpRequestConfig = new HttpRequestConfig.Builder()
  .addQueryParameter("ad_config_id", myAdConfig)
  .build();
    6. Appelez le service de catalogue pour récupérer votre vidéo ainsi que la configuration de votre annonce à partir de l'API de lecture. Commencez la lecture vidéo avec vos annonces spécifiées. 
      catalog.findVideoByID(myVideoId, httpRequestConfig, new VideoListener() {
    @Override
    public void onVideo(Video video) {

    // Process the Video
    plugin.processVideo(video);
  }
});

    Sous-titres

    Le SDK natif pour Android ne prend en charge que les sous-titres dans le manifeste avec SSAI. Les légendes Sidecar (ou distantes) peuvent ne pas se synchroniser avec le contenu et les sauts d'annonce.

    Mise en œuvre iOS

    Suivez ces étapes au sein de votre application pour obtenir la configuration de votre annonce et lire votre vidéo :

    1. Dans Xcode, ouvrez votre fichier ViewController.m .
    2. Importez le plugin SSAI pour iOS. 
      import BrightcoveSSAI
    3. Définissez des constantes pour l'ID de compte, la clé de stratégie, l'ID vidéo et l'ID de configuration de votre annonce. 
      struct Constants {
  static let AccountID = "your account id"
  static let PlaybackServicePolicyKey = "your policy key"
  static let VideoId = "your video id"
  static let AdConfigId = "your ad configuration id"
}
      Découvrez comment obtenir ou générer une clé de stratégie dans le document Clés de stratégie .
    4. Définissez la BCOVPlaybackService classe, qui fournit des méthodes asynchrones pour récupérer des données à partir de l'API Playback. 
      private lazy var playbackService: BCOVPlaybackService = {
  let factory = BCOVPlaybackServiceRequestFactory(accountId: Constants.AccountID, policyKey: Constants.PlaybackServicePolicyKey, baseURLStr: "https://edge.api.brightcove.com/playback/v1")
  return BCOVPlaybackService(requestFactory: factory)
}()
    5. Définissez la valeur du paramètre de requête sur votre identifiant de configuration d'annonce. 
      let queryParameters = [kBCOVPlaybackServiceParamaterKeyAdConfigId: Constants.AdConfigId]

    6. Appelez le playbackService pour récupérer votre vidéo ainsi que la configuration de votre annonce à partir de l'API de lecture. Commencez la lecture vidéo avec vos annonces spécifiées.

      playbackService.findVideo(withVideoID: Constants.VideoId, parameters: queryParameters) { [weak self] (video: BCOVVideo?, jsonResponse: [AnyHashable: Any]?, error: Error?) -> Void in

  guard let _video = video else {
    print("ViewController Debug - Error retrieving video: \(error?.localizedDescription ?? "unknown error")")
    return
  }

  self?.playbackController?.setVideos([_video] as NSFastEnumeration)
}

    Notes supplémentaires

    Il y a quelques notes techniques à prendre en compte lors de l'utilisation de SSAI avec les SDK natifs.

    SDK natif Brightcove pour iOS/TVOS

    • Post-rolls : Le plugin OnCeux traduit la timeOffset valeur de end sur le dernier break publicitaire (post-roll) en une CMTime valeur de +INFINITY , pour la vitesse et l'efficacité. Cette traduction est effectuée APRÈS la réception du document VMAP (ce qui se produit après la réponse de l'API de lecture) et AVANT le début de la lecture.
