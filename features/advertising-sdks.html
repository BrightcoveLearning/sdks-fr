Aperçu

Les kits SDK natifs Brightcove offrent une suite d'options pour vous aider à monétiser votre contenu. Les SDK natifs et leurs intégrations permettent aux éditeurs de décider comment optimiser au mieux la publicité dans leur lecteur avec des fonctionnalités telles que les publicités pré-, mid-roll et post-roll, ainsi que les publicités associées.

Remarques:

  • Les SDK natifs Brightcove ne prennent PAS en charge les publicités VPAID.
  • Pour Android MediaPlayer , il existe des limitations connues concernant l'utilisation d'annonces vidéo mid-roll lors de la lecture de vidéos HLS. L'utilisation du kit de développement logiciel Native Player de Brightcove résout bon nombre de ces problèmes. En savoir plus sur Lecture HLS avec le SDK natif pour Android.

Points de repère

Les URL publicitaires peuvent être définies avec des points de repère publicitaire ou dans une URL de tag de redirection VMAP :

  • Avec les points de repère publicitaire, chacun doit être configuré avec une URL de tag d'emplacement publicitaire VAST.
  • Sans points de repère publicitaire, vous pouvez utiliser une URL de tag d'emplacement publicitaire VMAP pour définir le moment où les annonces seront diffusées.
L'URL VMAP est une liste de lecture d'annonces, où une URL VAST définit une annonce individuelle. Pour des informations générales, consultez le Guide d'apprentissage : Publicité vidéo document.

Plugins côté client

La publicité côté client est l'approche traditionnelle consistant à effectuer des demandes d'annonces et l'insertion d'annonces sur l'appareil client. Avec cette méthode, la vidéo principale est lue jusqu'à ce qu'un point de repère publicitaire spécifique soit atteint. À ce stade, une demande est adressée au serveur publicitaire configuré, qui répond avec les détails appropriés de l'annonce. Le lecteur commence alors la lecture de l'annonce demandée.

Les intégrations de Brightcove simplifient ce processus en gérant le travail de création de demandes d'annonces, d'interprétation des spécificités publicitaires et d'affichage des annonces.

Annonces multimédias interactives (IMA) de Google

Brightcove a été la première plateforme vidéo en ligne à s'associer Google DoubleClick et développer le soutien aux SDK IMA de Google sur les plates-formes Web (Flash/HTML5) et mobiles (iOS/Android). Notre relation étroite avec Google nous permet de rester au courant des fonctionnalités à venir et des corrections de bugs.

Les éditeurs peuvent utiliser le Brightcove IMA SDK pour demander et afficher des annonces diffusées par DoubleClick for Publishers (DFP), le réseau Google AdSense. Ils peuvent également l'utiliser pour analyser leur propre liste de lecture VMAP tant qu'elle respecte la norme VMAP telle qu'elle est interprétée par le Brightcove IMA SDK.

En utilisant le SDK IMA de Google pour iOS et Android, nous proposons une intégration simple à utiliser pour les deux plates-formes, ce qui garantit que vous resterez à jour avec toutes les mises à jour de Google.

Insertion d'annonces côté serveur (SSAI)

L'assemblage d'annonces côté serveur est une approche innovante pour diffuser de la publicité à vos téléspectateurs. Fournissant un flux continu unique, les publicités sont intégrées dans le contenu. Cela supprime les demandes d'annonces et réduit les temps de chargement. Cela empêche également les téléspectateurs de bloquer les publicités. Les kits de développement logiciel Brightcove Native Player fournissent la logique et l'interface utilisateur publicitaires appropriées pour garantir que l'expérience de lecture du basculement entre les publicités et le contenu est plus fluide que jamais.
