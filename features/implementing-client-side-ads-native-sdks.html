    Implémentation des annonces côté client avec les kits SDK natifs

    Dans cette rubrique, vous apprendrez comment lire des annonces côté client avec les SDK natifs Brightcove.

    Présentation

    Les annonces côté client impliquent l'envoi d'une demande au serveur publicitaire à partir de l'appareil client. Cela peut entraîner un certain temps de mise en mémoire tampon pendant la récupération de la vidéo publicitaire. Pour une expérience plus similaire à la télévision, vous pouvez utiliser l' insertion d'annonces côté serveur.

    Avec les annonces côté client, la vidéo de contenu est lue jusqu'à ce qu'un point de repère publicitaire soit détecté ou qu'un timing publicitaire provenant de la réponse VMAP soit atteint. Une demande est adressée au serveur publicitaire configuré, qui répond avec les détails de l'annonce. Le joueur commence alors la lecture de l'annonce demandée.

    Les kits SDK natifs Brightcove fournissent des plugins pour les technologies suivantes :

    • Publicités multimédia interactives de Google (IMA)
    • Roue libre

    Les kits SDK natifs prennent en charge le VMAP généré lors de l'utilisation du Gestionnaire d'annonces Google et DoubleClick for Publishers (DFP), le réseau Google AdSense.

    Types d'annonces

    Il existe trois types principaux d'annonces que vous pouvez utiliser avec des vidéos :

    Publicités linéaires - Publicités qui interrompent la vidéo de contenu principal et sont affichées dans la pleine image du lecteur

    Publicités non linéaires : superposer les annonces affichées sur la vidéo de contenu principal. Il peut s'agir de texte, d'images graphiques ou même de superpositions vidéo.

    Publicités compagnons - Publicités affichées en dehors du lecteur vidéo

    Normes publicitaires

    Les fichiers XML standard peuvent être utilisés pour indiquer au lecteur quand lire vos annonces et quelles annonces jouer. L' Interactive Advertising Bureau (IAB) a mis au point les interfaces communes suivantes entre les lecteurs vidéo et les unités publicitaires :

    Modèle de service d'annonces vidéo (VAST) - Un schéma XML universel pour diffuser des annonces aux lecteurs vidéo numériques. Ce fichier spécifie les annonces à afficher.

    Playlist d'annonces multiples vidéo (VMAP) - Ce fichier décrit une liste de lecture d'annonces VAST, y compris les annonces pré-roll, mi-roll, post-roll et non linéaires. Ici, vous pouvez spécifier quand les pauses publicitaires se produisent et quelles annonces seront diffusées pendant chaque pause.

    Video Player Ad-Serving Interface Definition (VPAID) - Établit une interface commune pour fournir des expériences publicitaires interactives.

    Les kits SDK natifs Brightcove ne prennent PAS en charge les annonces VPAID

    Si vous êtes novateur des termes publicitaires, commencez par consulter le Guide d'apprentissage pour la publicité vidéo.

    Lire des annonces depuis votre application

    Pour lire des annonces côté client, procédez comme suit :

    1. Définir des balises publicitaires
    2. Vérifier les balises publicitaires
    3. Créer des points repères

    4. Créez votre application

      • Mise en œuvre Android
      • Mise en œuvre iOS

    Définir des balises publicitaires

    La balise d'annonce définit l'annonce qui sera récupérée et jouée. Vous pouvez utiliser l'un des formats suivants :

    • VAST - Pour plus d'informations sur le schéma XML, reportez-vous à la spécification IAB VAST.
    • VMAP - Pour plus d'informations sur le modèle XML, reportez-vous à la spécification VMAP IAB.

    Vérifier les balises publicitaires

    Il est recommandé de vérifier que vos balises publicitaires sont valides et fonctionnent comme prévu. Cela aidera à déboguer d'autres problèmes pendant le processus de mise en œuvre.

    Google fournit un inspecteur Video Suite qui vous permet de vérifier vos balises publicitaires.

    FreeWheel fournit un prévisualiseur d'annonces pour vérifier la réponse du serveur publicitaire.

    Créer des points repères

    Si vous utilisez une balise publicitaire VMAP, vous pouvez ignorer cette étape. Le VMAP définit une playlist pour vos annonces et spécifie quand elles seront diffusées. Notez que les règles publicitaires IMA de Google utilisent VMAP.

    Si vous utilisez des balises d'annonces VAST, vous devez créer des points de repère pour indiquer au joueur quand insérer des annonces. Vous devez les créer dans votre application. Voici quelques exemples pour savoir comment faire cela :

    Mise en œuvre Android

    Le SDK natif pour Android prend en charge VMAP, VAST, les règles publicitaires côté serveur et les annonces complémentaires avec les technologies suivantes :

    Google IMA

    Procédez comme suit pour utiliser les règles publicitaires Google IMA :

    1. Recherchez et vérifiez l'exemple d'application IMA Règles publicitaires .
    2. Ouvrez le fichier MainActivity.java .

    3. Personnalisez le projet avec vos valeurs. Les valeurs suivantes sont nécessaires pour récupérer du contenu de votre bibliothèque Video Cloud :

      • Clé de stratégie API de lecture
      • ID de compte
      • ID de la vidéo

    4. Inclure des valeurs pour accéder à vos annonces IMA :

      • URL des règles d'annonce
    5. La setupGoogleIMA() méthode établit une connexion avec le plugin IMA Brightcove.
    6. Écoutez l' GoogleIMAEventType.ADS_REQUEST_FOR_VIDEO événement. Ici, vous allez créer un objet de demande d'annonces et pointer vers l'URL de votre balise publicitaire. Dans ce cas, il s'agit d'un fichier VMAP de règles publicitaires.
    7. La setupAdMarkers() méthode est facultative. Cette méthode appelle IMA AdManager et crée des marqueurs publicitaires sur la barre de recherche, afin que les utilisateurs puissent voir visuellement quand les annonces seront diffusées.

    Cherchez sans publicités

    Si vous souhaitez désactiver la lecture d'annonces pendant que l'utilisateur recherche via la vidéo, reportez-vous à l'extrait de code Seek without ads .

    Le SDK natif pour Android utilise le framework ExoPlayer pour lire du contenu et des annonces. Avec IMA, les utilisateurs ont besoin d'un deuxième clic sur le bouton de lecture pour démarrer la lecture du contenu, après avoir recherché les annonces.

    Des échantillons de code

    Voici des exemples de code complets :

    • Joueur IMA Vaste de base: Pour utiliser les balises VAST, votre application doit créer un ensemble de points de repère à partir des métadonnées de l' Video objet renvoyées par l'appel de l'API de lecture. Les métadonnées incluent les points de repère d'annonce que vous avez créés dans Video Cloud Studio.

    • Joueur IMA Ad Rules IMA: Alternativement, votre application peut créer un ensemble d'objets point de repère dans la Activity classe de lecteur.

    • Règles publicitaires IMA Widevine Modulaire: Combinez les annonces côté client à la protection du contenu.

    FreeWheel

    Ces annonces nécessitent l'achat de la bibliothèque FreeWheel AdManager.

    Procédez comme suit pour utiliser les annonces FreeWheel :

    1. Contactez FreeWheel pour acheter leur archive de bibliothèque ADManager.aar et l'installer dans votre répertoire libs/, qui se trouve généralement à la racine de votre répertoire /home/. Le SDK natif pour Android et les exemples d'applications FreeWheel nécessitent la version 6.28.0 de la bibliothèque FreeWheel AdManager.
    2. Recherchez et vérifiez l'exemple d'application FreeWheel .
    3. Ouvrez le fichier MainActivity.java .
    4. Personnalisez le projet avec vos valeurs. Cet exemple utilise une ressource vidéo distante.

    5. Inclure des valeurs pour accéder à vos annonces FreeWheel :

      • URL de l'annonce
      • ID de réseau publicitaire
      • Profil d'annonce
    6. La setupFreeWheel() méthode établit une connexion avec le plugin FreeWheel de Brightcove.
    7. Configurez le contrôleur FreeWheel pour qu'il pointe vers l'URL de votre annonce. freeWheelController.setAdURL("http://demo.v.fwmrm.net/");
    8. Cela sera utilisé plus tard pour instancier la AdRequestConfiguration classe FreeWheel.
    9. Écoutez l' FreeWheelEventType.WILL_SUBMIT_AD_REQUEST événement. Ici, vous obtiendrez le contrôleur FreeWheel et configurez les emplacements publicitaires pré-roll, mi-roll, post-roll et superposition.

    Cherchez sans publicités

    Si vous souhaitez désactiver la lecture d'annonces pendant que l'utilisateur recherche via la vidéo, reportez-vous à l'extrait de code Seek without ads .

    Mise en œuvre iOS

    Le SDK natif pour iOS prend en charge VMAP, VAST, les règles publicitaires côté serveur et les annonces complémentaires avec les technologies suivantes :

    Google IMA

    Pour plus d'informations sur l'installation, le démarrage rapide et l'utilisation, consultez le module IMA Plugin for Brightcove Player SDK pour référence iOS .

    Procédez comme suit pour utiliser Google IMA Ads :

    1. Recherchez et vérifiez l'exemple d'application Basic IMA Player .
    2. Ouvrez le fichier ViewController .

    3. Personnalisez le projet avec vos valeurs. Les valeurs suivantes sont nécessaires pour récupérer du contenu de votre bibliothèque Video Cloud :

      • Clé de stratégie API de lecture
      • ID de compte
      • ID de la vidéo

    4. Inclure des valeurs pour accéder à vos annonces IMA :

      • ID de l'éditeur
      • Langue
      • URL de balise d'annonce
    5. Créez le playbackController avec les paramètres IMA appropriés, y compris la stratégie de demande d'annonce (VAST, VMAP ou règles publicitaires côté serveur).
    6. Les événements publicitaires sont émis par le plugin BCOVIMA par le biais d'événements du cycle de vie. Ces événements sont définis dans le fichier BCOVImaComponent.h .
    7. L'exemple d'application inclut des fonctions pour masquer les contrôles du lecteur pendant la lecture des annonces et les afficher lorsque les annonces sont terminées.

    Cherchez sans publicités

    Lorsque vous souhaitez désactiver la lecture d'annonces pendant que l'utilisateur recherche via la vidéo, reportez-vous à la section Seek without ads de la référence du plugin.

    FreeWheel

    Le SDK Freewheel AdManager n'est pas installé à l'aide de CocoaPods. Au lieu de cela, vous devez l'ajouter manuellement à la cible d'application de votre projet.

    Procédez comme suit pour utiliser les annonces FreeWheel :

    1. Ajoutez manuellement le SDK FreeWheel Admananger à votre projet. Pour obtenir des étapes détaillées, reportez-vous au fichier README du Lecteur Freewheel Basic.
    2. Recherchez et vérifiez l'exemple d'application Basic FreeWheel Player .
    3. Ouvrez le fichier ViewController .

    4. Personnalisez le projet avec vos valeurs. Les valeurs suivantes sont nécessaires pour récupérer du contenu de votre bibliothèque Video Cloud :

      • Clé de stratégie API de lecture
      • ID de compte
      • ID de la vidéo
    5. Créez FreeWheel AdManager. Cela sera responsable de tous les contextes publicitaires.
    6. Créez le fournisseur de session et le contrôleur de lecture à l'aide de la stratégie de contexte publicitaire.
    7. Personnalisez le projet avec vos propres valeurs de configuration FreeWheel. Reportez-vous à la documentation de FreeWheel ou contactez votre responsable de compte FreeWheel.
    8. Configurez la vue publicitaire, les contrôles publicitaires et l'emplacement compagnon.

    Cherchez sans publicités

    Lorsque vous souhaitez désactiver la lecture d'annonces pendant que l'utilisateur recherche via la vidéo, reportez-vous à la section Seek without ads de la référence du plugin.
