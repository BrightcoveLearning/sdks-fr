    Formats vidéo et sous-titrage avec les SDK natifs

    Dans cette rubrique, vous découvrirez les formats vidéo et le sous-titrage pris en charge par les SDK Brightcove Native Player.

    Présentation

    Il est essentiel de bien comprendre les options qui s'offrent à vous lors du choix du format vidéo à utiliser et des sous-titres à appliquer, car cela vous permettra de diffuser l'environnement vidéo optimal aux utilisateurs.

    Grâce aux SDK de lecteur natifs de Brightcove pour les plates-formes mobiles, votre application est à même de toucher un plus large public tout en respectant les directives de sous-titrage optimal de la FCC.

    Pour plus d'informations sur les normes et techniques de sous-titrage, consultez l'article de blog Zencoder sur le sous-titrage.

    Passez en revue les formats vidéo et sous-titres de chaque catégorie :

    • Téléchargement progressif
    • Diffusion de débit adaptatif
    • Gestion des droits numériques (DRM)

    Téléchargement progressif

    Les formats vidéo qui utilisent cette méthode forcent le contenu à se télécharger intégralement sur le dispositif cible. Cela signifie que les visiteurs ne pourront pas aller à un minutage qui n'a pas déjà été mis en tampon.

    Les publicités mid-roll ne sont pas utilisées avec les formats vidéo de téléchargement progressif. En outre, les éditeurs d'applications doivent savoir que toute action effaçant la mémoire tampon entraîne un retéléchargement intégral du contenu en cas de reprise ultérieure de la lecture.

    Tableau de la compatibilité MP4/M4V

    Format vidéo SDK iOS SDK Android
    MP4/M4V
    Format de sous-titre    
    WebVTT sidecar
    TTML (DFXP) sidecar
    608/708

    Comme les sous-titres 608/708 sont intégrés au flux MP4, les fichiers vidéo doivent être recréés pour modifier les légendes. Cela peut être un problème si vous souhaitez localiser une vidéo. Il n'y a pas non plus aucun moyen d'ajouter de nouvelles légendes à un fichier MP4, car il n'y a pas de manifeste pour fonctionner. En revanche, les légendes peuvent être ajoutées aux fichiers HLS, car elles contiennent un manifeste.

    Diffusion à débit adaptatif

    Les formats vidéo utilisant cette méthode sont divisés en petits segments téléchargés par le lecteur quand ils sont requis. Cela permet aux utilisateurs de rechercher n'importe où dans le contenu et n'ont besoin que de télécharger le segment dans lequel ils ont sauté. En général, les formats qui utilisent cette méthode ont recours à un « manifeste » qui renferme tous les détails de la vidéo (nombre de segments, où les trouver…).

    Tableau de compatibilité de la diffusion en continu HTTP (HTTP Live Streaming ou HLS)

    Format vidéo SDK iOS SDK Android
    Diffusion en continu HTTP (HLS)
    Format de sous-titre    
    WebVTT via manifeste
    WebVTT sidecar
    TTML (DFXP) sidecar
    608/708

    Remarques concernant les kits SDK Brightcove Player pour iOS et TVOS :

    • La diffusion adaptative Bitrate Streaming, en particulier HLS, est requise pour la plupart de la lecture en ligne sur les appareils iOS. Pour plus de détails, consultez le document technique d'Apple sur HLS.
    • Le plugin Sidecar WebVTT prend en charge la lecture sur AirPlay avec des légendes.
    • Pour plus d'informations sur l'utilisation des légendes WebVTT pour le contenu HLS, consultez le document Synchronisation des légendes WebVTT .

     

    Similaire à HLS, MPEG-DASH permet une diffusion de contenu multimédia de haute qualité sur HTTP. Il fonctionne également en divisés en une séquence de segments de fichiers.

    Tableau pour la prise en charge de la diffusion dynamique adaptative sur HTTP (MPEG-DASH)

    Format vidéo SDK iOS SDK Android
    Diffusion dynamique adaptative sur HTTP (MPEG-DASH)
    Format de sous-titre    
    WebVTT via manifeste
    WebVTT sidecar
    TTML (DFXP) sidecar

     

    Gestion des droits numériques (DRM)

    Un troisième ensemble de formats digne d'intérêt sont les formats propriétaires utilisés par certains de nos partenaires fournisseurs de DRM.

    Les SDK de lecteur natifs Brightcove ne peuvent pas accéder à ou contrôler les propriétés et fonctionnalités de ces formats. Tout problème rencontré avec ces formats devra être résolu en contactant directement leur propriétaire.

    Tableau de la compatibilité de Widevine Media (.wvm)

    Format vidéo SDK iOS SDK Android
    Widevine Modular (DASH-CENC)
    Format de sous-titre    
    WebVTT sidecar
    TTML (DFXP) sidecar
    608/708

    Remarques relatives à Widevine :

    • Widevine Media n'est pris en charge qu'avec l'intégration Widevine.
    • Pour le SDK Android, Widevine Modular est pris en charge uniquement pour les versions Android 4.4+.
    • Pour les applications iOS créées à l'aide du SDK Brightcove Player pour iOS, vous devez utiliser FairPlay si vous avez besoin de DRM.

     

    La technologie FairPlay Streaming (FPS) d'Apple peut être utilisée pour diffuser du contenu sécurisé lors de l'utilisation de HTTP Live Streaming (HLS).

    Tableau pour la prise en charge de FairPlay Streaming (FPS)

    Format vidéo SDK iOS SDK TVOS
    Diffusion en continu HTTP (HLS)
    Format de sous-titre    
    WebVTT sidecar

    Notes FairPlay :

    • Problèmes Apple connus - Le contenu en streaming FairPlay ne fonctionnera pas dans les cas suivants :
      • Mode de mise en miroir AirPlay
      • Utilisation d'AirPlay pour lire des vidéos hors connexion protégées avec FairPlay
      • Vidéos représentant du contenu à 360°

     

    Pour plus d'informations sur la protection du contenu avec les kits SDK natifs, consultez la page Intégrations de droits numériques .
