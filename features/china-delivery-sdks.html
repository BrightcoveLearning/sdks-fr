introduction

Brightcove fournit une solution pour diffuser des vidéos vers la Chine depuis l'extérieur du pays. La solution est composée de deux parties :

  • Un serveur proxy hébergé en Chine
  • Un CDN connecté spécifique à la Chine

Le proxy achemine le trafic non vidéo (requêtes API de lecture, balises Analytics et lecteur Web lui-même) vers et depuis les téléspectateurs en Chine vers l'infrastructure Brightcove située à l'extérieur.

Le CDN est connecté au proxy et gère le contenu vidéo et image.

Conditions

Les conditions suivantes sont requises pour la livraison en Chine avec les SDK natifs de Brightcove :

Installer

  • Il y a des exigences d'installation en plus de la configuration de votre lecteur

    Pour plus de détails, voir Aperçu: Livraison en Chine

Version du système d'exploitation de l'appareil

  • Android 6.0 et plus récent

    Android 8.x peut avoir des problèmes avec les licences de location hors ligne.
  • iOS 11.0 et plus récent

Version du SDK Brightcove

  • SDK natif pour Android (à venir)
  • SDK natif pour iOS 6.5.0 et versions ultérieures

Implémentation Android

Le SDK natif pour Android prenant en charge la livraison en Chine sera disponible en pré-version au début du troisième trimestre 2021.

Implémentation iOS

Pour utiliser China Delivery avec le SDK natif pour iOS, procédez comme suit

  1. Les BCOVGlobalConfiguration class vous permet de définir des configurations personnalisées pour le SDK natif pour iOS.

  2. Dans cette classe, définissez le chineProxyDomain propriété à un nom de domaine entièrement qualifié (FQDN) personnalisé pour l'API de lecture, les services de collecte de métriques et d'analyses.

    @property (nonatomic, nullable) NSString *chinaProxyDomain

Limites

Pour plus de détails sur les limitations de cette fonctionnalité, consultez le Aperçu: Livraison en Chine document.
