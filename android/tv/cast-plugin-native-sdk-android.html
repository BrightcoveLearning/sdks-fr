    Plugin Cast pour le SDK natif pour Android

    Dans cette rubrique, vous allez apprendre à diffuser vos vidéos sur un téléviseur connecté avec Chromecast, à l'aide du plugin Cast pour le SDK Native Brightcove pour Android.

    Présentation

    Avec la technologie Google Cast , vous pouvez initier et contrôler le contenu vidéo diffusé depuis vos appareils mobiles vers la télévision haute définition et les systèmes audio domestiques. Depuis votre application, appuyez sur le bouton Cast pour diffuser votre contenu sur grand écran.

    Pour créer une application Cast, vous avez besoin des composants suivants :

    • Une application expéditeur - Cette application réside sur votre appareil mobile et détecte les périphériques récepteurs, établit une connexion sécurisée et reflète votre contenu. L'application expéditeur est le lecteur local utilisé pour diffuser du contenu vers l'application récepteur sur votre appareil Chromecast. Après avoir diffusé sur l'application récepteur, vous pouvez la considérer comme une télécommande pour le Chromecast.

      L'application expéditeur est fournie par le plugin Cast pour le SDK Brightcove Native pour Android. Vous en apprendrez plus dans ce sujet.

    • Une application récepteur - Cette application s'exécute sur le périphérique Chromecast. Il peut être considéré comme une application HTML d'une seule page avec des actifs CSS et JavaScript.

      Pour les tests, procédez comme suit :

      1. Commencez avec le récepteur de distribution d'échantillons de Google
      2. Consulter la documentation Google Cast Apps

      Pour une utilisation en production, ce document illustre comment utiliser Brightcove Receiver v2.0.0.

    Remarques :

    1. Google ne prend pas en charge l'utilisation de son application récepteur de démonstration dans un environnement de production.
    2. Le plugin Cast pour le SDK Brightcove Native pour Android ne fournit pas la fonctionnalité de l'application récepteur.

    Version SDK prise en charge

    Pour utiliser le plugin Cast avec le nouveau Brightcove Receiver v2.0, vous devez utiliser le SDK Native Brightcove pour Android version 6.16.0 et ultérieure.

    Comprendre le plugin Cast

    Le plugin Cast est construit sur le nouveau ExoPlayer Cast Extension et le Google Play Services Cast Framework. Après avoir ajouté la dépendance du plugin cast, gradle tirera la dépendance Play Services Cast Framework, la dépendance ExoPlayer Cast Extension ainsi que d'autres dépendances nécessaires.

    Le plugin Cast a été repensé pour minimiser vos efforts lors de l'intégration avec Video Cloud. Lors de l'utilisation de Video Cloud, la BrightcoveCastMediaManager classe recueille des informations provenant de la réponse Video Cloud, telles que le Brightcove Video et les Source objets, à chaque fois que le EventType.SET_SOURCE est émis. Ces informations sont mises en cache et prêtes à être utilisées lorsque l'utilisateur sélectionne la lecture pour faire la file d'attente de la vidéo.

    Intégration du plugin Cast

    Il y a deux façons d'intégrer votre application avec le plugin Native SDK pour Android Cast. Vous pouvez l'intégrer soit avec le récepteur Brightcove Cast Receiver v2.0, soit avec le Google Cast Demo Receiver.

    Pour l'une ou l'autre intégration, vous devez ajouter cette dépendance à votre projet d'application :

    implementation "com.brightcove.player:android-cast-plugin:6.16.0"

    Utilisation du récepteur Cast Brightcove v2.0

    Cette intégration est destinée aux clients Brightcove qui utilisent les API Brightcove pour diffuser leur contenu.

    Pour obtenir un exemple de code complet, consultez BasicCastBrightCovereCovereCampleApp.

    Utilisation du récepteur de démonstration Google Cast

    Cette intégration est destinée aux clients de Brightcove qui sont nouveaux dans le casting.

    Pour un exemple de code complet, consultez BasicCastGoogleReceiversampleApp.

    L'exemple d'application BasicCastGoogleReceiversAmpleApp était autrefois connu sous le nom BasicCastSampleApp

    Spécification de votre propre ID d'application Cast Receiver

    BasicCastGoogleReceiverAmpleApp définit un ID d'application Google Demo Receiver, ce qui peut être utile pour démarrer et tester, mais pas pour la production.

    Pour remplacer cette valeur avec votre application Cast Receiver, définissez la valeur de chaîne suivante dans le fichier strings.xml de l'application exemple :

    <string name="cast_receiver_app_id">4F8B3483</string>

    Le composant Brightcove GoogLecastComponent

    La GoogleCastComponent classe est la classe principale du plugin Brightcove Cast. Il instancie l'ExoPlayer CastPlayer et définit ses auditeurs. Il expose quelques méthodes essentielles pour charger une vidéo ou l'ajouter à la file d'attente. La GoogleCastComponent classe ajoute également plusieurs écouteurs d'événement Brightcove pour gérer les événements du cycle de vie d'activité et de fragment, ainsi que d'autres écouteurs d'événements que vous pouvez utiliser pour émettre Media Info pour charger une vidéo sur votre appareil Chromecast.

    Le utilise GoogleCastComponent maintenant un modèle Builder. Dans le SDK natif pour les versions Android antérieures à la version 6.16.0, vous aviez besoin d'instancier le composant Context et de EventEmitter passer le et au GoogleCastComponent constructeur. Ensuite, vous définissez les options du composant dans une série d'appels de méthode distincts.

    À partir du SDK natif pour Android v6.16.0, utilisez le modèle Builder pour créer une instance de l' GoogleCastComponent et définir ses options, le tout dans une seule chaîne d'appels de méthode Builder.

    CustomData

    Comme avec le GoogleCastComponent, la CustomData classe utilise un modèle Builder pour instancier l'objet et y ajouter des propriétés. Bien que Brightcove Receiver puisse utiliser CustomData pour récupérer des vidéos à partir de votre catalogue Brightcove, il n'est pas nécessaire d'envoyer un CustomData objet complet, par exemple pour lancer une ressource distante. Il est également important de noter que lors de l'utilisation de Google Demo Receiver, l'utilisation de n' CustomData est pas prise en charge. Dans le cadre de cette discussion, nous nous concentrerons sur l' CustomData envoi au destinataire utilisé pour récupérer les données vidéo du catalogue Brightcove.

    Qu'est-ce que CustomData ?

    CustomData est un objet JSON contenu dans l' MediaInfo objet. Son utilisation est prévue avec l'application Brightcove Cast Receiver v2.0.

    CustomData avec le récepteur Brightcove et les données du catalogue

    Lors de l'intégration avec Brightcove Receiver, l'objet CustomData JSON prend la forme suivante :

    "customData": {
	"accountId": "1234567890",
	"analyticsParams": {
		"application": "com.brightcove.player.test",
		"user": "abcde1c44b951234"
	},
	"catalogParams": {
		"adConfigId": null,
		"type": "video",
		"bcovAuthToken": null,
		"id": "2345678901",
		"policyKey": "BCpkPolicyKeyObject"
	}
}

    L'exemple CustomData d'objet ci-dessus contient tous les éléments de données nécessaires pour diffuser une vidéo à partir du récepteur Brightcove. Ces données sont les mêmes quel que soit le chiffrement, c'est-à-dire qu'il n'y a pas de structure supplémentaire nécessaire pour l'URL de licence dans le cas des vidéos DRM.

    Vous pouvez également trouver un exemple de l' CustomData objet dans BrightCoveCastBrightCoverEceIversampleApp.

    CustomData avec Google Demo Receiver

    Comme indiqué ci-dessus, n' CustomData est pas pris en charge avec le Google Demo Receiver.

    BrightCoveCastMediaManager

    Il est possible d'étendre le BrightcoveCastMediaManager, comme indiqué ci-dessus, pour remplacer ses méthodes ou pour implémenter les vôtres. Pour obtenir des exemples d'extension de la classe BrightCoveCastMediaManager, consultez les points suivants :

    • Étendre la section BrightCoveCastMediaManager
    • BasicCastCustomRemoteVideoCampleApp

    OptionsProvider

    Ensuite, vous devez spécifier l'implémentation OptionsProvider pour le framework Google Cast. L' OptionsProvider interface permet de configurer plusieurs options nécessaires à l'initialisation de la classe CastContext . C'est là que vous définissez l'ID de l'application Cast Receiver. Pour en savoir plus sur l'intégration du OptionsProvider, consultez le document Initialize the Cast Context de Google.

    Le plugin Brightcove Cast inclut une DefaultOptionsProvider classe, dans laquelle l'ID de l'application Cast Receiver est défini via une clé de chaîne définie dans le fichier de ressources strings.xml . Pour plus de détails et pour savoir comment le remplacer dans votre application, consultez la section Utilisation de votre propre ID d'application Cast Receiver ci-dessus.

    Que vous utilisiez la DefaultOptionsProvider classe ou votre propre OptionsProvider implémentation, vous devez définir le nom de la classe d' OptionsProvider implémentation dans AndroidManifest.xml de votre application comme métadonnées de paire clé-valeur, comme indiqué ici :

    <meta-data android:name="com.google.android.gms.cast.framework.OPTIONS_PROVIDER_CLASS_NAME" android:value="com.brightcove.cast.DefaultOptionsProvider" />

    Si vous utilisez la DefaultOptionsProvider classe, vous pouvez définir ExpandedControllerActivity pour activer/désactiver la notification Cast en définissant des informations de métadonnées similaires dans votre AndroidManifest.

    Activité du contrôleur étendu

    L' ExpandedControllerActivity est livré avec la bibliothèque Google Cast et vous permet de contrôler la vidéo diffusée sur votre appareil Chromecast. Cette classe offre une certaine flexibilité de personnalisation. Par exemple, cinq emplacements sont disponibles pour afficher les boutons, où le troisième emplacement est réservé au bouton Lecture non configurable. Le reste des boutons peut être défini comme d'autres boutons prédéfinis ou comme vos propres boutons personnalisés.

    Le plugin Brightcove Cast fournit la sous-classe DefaultExpandedControllerActivity. Nous avons activé les boutons suivants dans l'ordre suivant :

    • Sous-titres codés
    • Ignorer le précédent
    • Jouer
    • Ignorer Suivant
    • Basculer en sourdine

    De plus, la barre de recherche définit le même dessin par défaut que celui utilisé dans la norme BrightcoveMediaController:

    • Les progrès tirables : 
      R.drawable.default_scrubber_progress_horizontal
    • Le pouce dessinable : 
      R.drawable.default_scrubber_thumb

    Pour savoir comment personnaliser la barre de recherche, reportez-vous à l'application SeekBarColorsAmpleApp.

    Pour définir le DefaultExpandedControllerActivity ou le vôtre ExpandedControllerActivity , définissez les métadonnées suivantes dans votre fichier AndroidManifest.xml :

    <meta-data  android:name="com.brightcove.cast.DefaultOptionsProvider.EXPANDED_CONTROLLER_ACTIVITY_CLASS_NAME"
android:value="com.brightcove.cast.DefaultExpandedControllerActivity" />

    Notification Cast

    Lorsque la notification Cast est activée, la notification s'affiche lorsque vous diffusez une vidéo et placez votre application en arrière-plan, par exemple, après avoir appuyé sur la touche Accueil.

    <PPour activer la notification de diffusion, définissez les métadonnées suivantes dans votre fichier AndroidManifest.xml et indiquez le nom de l'activité à lancer lorsque vous cliquez sur la notification :

    <meta-data android:name="com.brightcove.cast.DefaultOptionsProvider.NOTIFICATION_TARGET_ACTIVITY_CLASS_NAME"
android:value="com.brightcove.cast.BrightcoveControllerActivity" />

    Si vous ne fournissez pas com.brightCove.cast.DefaulToptionsProvider.notification_target_activity_class_name ou si la valeur a un nom d'activité non valide, la notification Cast sera désactivée.

    Lorsque la suppression de la meta-data balise du fichier AndroidManifest.xml désactive les notifications de l'application de l'expéditeur, il sera toujours possible d'afficher une notification sur le périphérique de l'expéditeur indiquant qu'un appareil est diffusé sur votre réseau Wi-Fi réseau. Il ne s'agit pas d'une notification de l'application de l'expéditeur, mais plutôt d'une notification de l'appareil expéditeur, indépendamment de toute configuration de l'application.

    Le bouton Cast

    Le bouton cast vous permet de sélectionner un appareil Chromecast dans le même réseau que votre appareil, et vous permet de vous connecter et de créer une session. Pour ajouter le bouton Cast à votre application, suivez la diffusion intégrée de Google : Ajouter un document Cast Button .

    Le plugin Brightcove Cast fournit une méthode utilitaire pour configurer facilement le bouton Cast. Ceci est utile lorsque vous voulez simplement ajouter le bouton Cast au menu Activité/Fragment. Voir le code suivant pour plus de détails :

    //Activity
@Override
public boolean onCreateOptionsMenu(Menu menu) {
   super.onCreateOptionsMenu(menu);
   GoogleCastComponent.setUpMediaRouteButton(MainActivity.this, menu);
   return true;
}

    La mini manette

    Le Mini Controller est un fragment qui est attaché à votre activité, généralement situé en bas de la mise en page. Le Mini Controller vous permet de lire et de mettre en pause la vidéo, et indique quand une vidéo est en cours de lecture sur votre appareil Chromecast. Lorsque vous cliquez sur le Mini Controller, le contrôleur étendu est lancé.

    Pour activer le Mini Controller, ajoutez le code suivant à la mise en page de votre activité.

    <fragment
   android:id="@+id/castMiniController"
   android:layout_width="match_parent"
   android:layout_height="wrap_content"
   android:layout_alignParentBottom="true"
   android:visibility="gone"
   class="com.google.android.gms.cast.framework.media.widget.MiniControllerFragment" />

    Pour en savoir plus sur les meilleures pratiques avec le Mini Controller, consultez la liste de vérification de conception : Document Expéditeur Mini Controller . Pour plus d'informations sur l'implémentation, consultez la section Intégrer Ajouter un document Mini Controller .

    Rubriques avancées

    À l'écoute de la session Cast

    Si vous souhaitez que votre application réagisse lorsque la connexion Cast démarre ou se termine, vous pouvez ajouter le GoogleCastEventType.CAST_SESSION_STARTED ou GoogleCastEventType.CAST_SESSION_ENDED écouteurs d'événements comme indiqué ici:

    eventEmitter.on(GoogleCastEventType.CAST_SESSION_STARTED, new EventListener() {
   @Override
   public void processEvent(Event event) {
       // Session Started
   }
});
eventEmitter.on(GoogleCastEventType.CAST_SESSION_ENDED, new EventListener() {
   @Override
   public void processEvent(Event event) {
       // Session Ended
   }
});

    Vous pouvez également appeler pour GoogleCastComponent.isSessionAvailable() vérifier la disponibilité d'une session.

    Casting d'une vidéo

    Afin de diffuser une vidéo sur Chromecast après une connexion réussie, vous pouvez utiliser EventEmitter et émettre les informations Media Info ou utiliser directement les GoogleCastComponent méthodes.

    Si vous préférez émettre les informations médias, vous pouvez utiliser les événements suivants :

    • GoogleCastEventType.LOAD_MEDIA_INFO
    • GoogleCastEventType.LOAD_MEDIA_QUEUE_ITEM
    • GoogleCastEventType.ADD_MEDIA_INFO
    • GoogleCastEventType.ADD_MEDIA_QUEUE_ITEM

    Le tableau suivant présente les propriétés attendues pour chaque événement :

    Nom de l’événement Propriétés
    Clé Type de classe de valeur
    LOAD_MEDIA_INFO GoogleCastComponent.CAST_MEDIA_INFO
    GoogleCastComponent.CAST_MEDIA_PLAY_POSITION    		 MediaInfo Entier
    LOAD_MEDIA_QUEUE_ITEM GoogleCastComponent.CAST_MEDIA_QUEUE_ITEM MediaQueueItem
    ADD_MEDIA_INFO GoogleCastComponent.CAST_MEDIA_INFO
    GoogleCastComponent.CAST_MEDIA_PLAY_POSITION    		 MediaInfo Entier
    ADD_MEDIA_QUEUE_ITEM GoogleCastComponent.CAST_MEDIA_QUEUE_ITEM MediaQueueItem

    Vous pouvez également utiliser les GoogleCastComponent méthodes suivantes :

    • public void loadMediaInfo(MediaInfo mediaInfo, long positionMs)
    • public void loadMediaInfo(MediaInfo mediaInfo)
    • public PendingResult<RemoteMediaClient.MediaChannelResult> loadItem(MediaQueueItem mediaQueue, int playheadPosition)
    • public PendingResult<RemoteMediaClient.MediaChannelResult> addItems(MediaQueueItem... mediaQueue )

    Modification des données MediaInfo par défaut

    Par défaut, le plugin Cast rassemble des informations sur l'objet Vidéo et Source en cours émis par le EventType.SET_SOURCE event. Si vous souhaitez modifier ou ajouter des informations supplémentaires, telles que l'ajout d'un objet JSON personnalisé à votre MediaInfo récepteur d'application, vous pouvez le faire en écrasant loadMediaInfo() les addMediaInfo() à partir de la méthode BrightcoveCastMediaManager. Ensuite, votre BrightcoveCastMediaManager sous-classe est passée en tant que paramètre constructeur à la GoogleCastComponent classe.

    À l'intérieur de ces méthodes, vous pouvez créer vos MediaInfo objets et émettre les événements appropriés comme indiqué précédemment. Assurez-vous de vérifier com.brightcove.cast.util.CastMediaUtil, car il fournit quelques méthodes utilitaires pour créer un à MediaInfo partir des objets Vidéo et Source.

    Configuration du Cast MediaController

    Pour modifier la disposition du contrôleur qui apparaît dans votre vue vidéo Brightcove lorsqu'une session Cast a démarré, procédez comme suit.

    1. Étendre le BrightCoveCastMediaManager
    2. Définir le MediaControllerConfig
    3. Remplacer la configuration de la barre de contrôle

    Étendre le BrightCoveCastMediaManager

    Pour modifier le comportement par défaut du BrightcoveCastMediaManager, créez une sous-classe et remplacez certaines méthodes clés :

    • public void updateBrightcoveMediaController(boolean isRemote)

      Cette méthode est appelée par le GoogleCastComponent quand la session change, c'est-à-dire lorsque la session a démarré ou s'est terminée. Lorsque la session a démarré, le isRemote paramètre sera true et la setupRemoteController méthode est appelée. Sinon, le isRemote sera false et le resetToLocalController est appelé.

    • protected void setupRemoteController()

      Cette méthode émet l'événement EventType.SET_MEDIA_CONTROLLER_CONFIG avec l' MediaControllerConfig objet. Nous en parlerons plus MediaControllerConfig loin dans cette section. Cette méthode écoute également l' BrightcoveMediaController.CONTROL_BAR_CREATED événement et réagit en appelant la setupBrightcoveControlBar méthode.

    • protected void resetToLocalController()

      Cette méthode est responsable de la réinitialisation BrightcoveMediaController de la disposition du contrôleur d'origine en émettant le EventType.RESTORE_DEFAULT_MEDIA_CONTROLLER.

    • protected void setupBrightcoveControlBar(BrightcoveControlBar controlBar)

      Une fois que l' BrightcoveMediaController a été recréé avec la mise en page fournie dans le MediaControllerConfig , vous aurez accès à la BrightcoveControlBar vue. De là, vous pouvez accéder à vos vues d'interface utilisateur, comme les boutons, où vous pouvez les ajouter OnClickListener.

    Définir le MediaControllerConfig

    La MediaControllerConfig est une classe de configuration que vous pouvez utiliser pour modifier la disposition de contrôle par défaut de la BrightcoveMediaController classe. Dans cette classe, vous pouvez définir la mise en page et le OnTouchListener. Une fois créé et configuré, vous pouvez émettre cet objet comme indiqué ci-dessous :

    Map<String, Object> properties = new HashMap<>();
properties.put(Event.MEDIA_CONTROLLER_CONFIG, myMediaControllerConfig);
eventEmitter.emit(EventType.SET_MEDIA_CONTROLLER_CONFIG,properties);

    L' MediaControllerConfig objet par défaut définit la R.layout.cast_media_controller comme mise en page avec un seul bouton Lecture . Lorsque vous cliquez dessus, il ouvrira une boîte de dialogue avec deux options :

    • Lire maintenant - Lorsque cette option est sélectionnée, la loadMediaInfo() méthode est appelée et la vidéo sera chargée et lue dans Chromecast.
    • Ajouter à la file d'attente - Lorsque cette option est sélectionnée, la addMediaInfo() méthode est appelée et la vidéo est ajoutée à la fin de la file d'attente.
    Si vous souhaitez conserver la même interface utilisateur, mais souhaitez charger des informations supplémentaires dans la vidéo en cours de diffusion, vous pouvez remplacer le loadMediaInfo() et addMediaInfo() méthodes et personnalisez les informations que vous ajoutez MediaInfo objet.

    Remplacer la configuration de la barre de contrôle

    Lorsque la disposition de votre contrôleur de média est définie en émettant le MediaControllerConfig, la BrightcoveControlBar vue sera créée et la BrightcoveCastMediaManager.setupBrightcoveControlBar() méthode sera appelée. C'est ici que vous pouvez obtenir vos composants d'interface utilisateur par ID et ajouter les écouteurs appropriés.

protected void setupBrightcoveControlBar(BrightcoveControlBar controlBar) {
   Button playButton = controlBar.findViewById(R.id.cast_play);
   if (playButton != null) {
       playButton.setOnClickListener(new View.OnClickListener() {...});
   }
}

    problèmes connus

    Android 9

    Lorsque vous utilisez Chromecast avec Android 9 et versions ultérieures, vous devez inclure une FOREGROUND_SERVICE autorisation. Cela permet à l'application d'utiliser des notifications pour contrôler une session de casting lorsque l'application est background et ramenée au premier plan.

    La uses-permission balise doit être ajoutée au fichier AndroidManifest.xml de l'application, comme suit :

    <uses-permission android:name="android.permission.FOREGROUND_SERVICE"/>

    Services Google Play

    La connexion Casting ne peut pas être créée si la version des services Google Play de l'expéditeur n'est pas à jour. Lorsque les services Google Play de l'expéditeur, en particulier le framework Cast Services, sont obsolètes, vous risquez de ne pas réussir à créer une connexion Cast. Cette solution est corrigée en mettant à jour les services Google Play de l'expéditeur via le Google Play Store.
