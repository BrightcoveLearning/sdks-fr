    Android : Gestion des contrôles du joueur

    Dans cette rubrique, vous apprendrez comment utiliser les contrôles Brightcove Player lorsque vous utilisez le SDK Brightcove Player pour Android.

    Présentation

    Le SDK Native Player fournit des contrôles de lecteur similaires à Brightcove Web Player pour offrir une expérience utilisateur cohérente sur toutes les plates-formes. Vous pouvez utiliser les commandes du lecteur Brightcove prêtes à l'emploi ou les personnaliser avec vos propres couleurs, styles et boutons.

    Remarques :

    Pour en savoir plus sur ces fonctionnalités, consultez le document Brightcove Native Player Controls .

    Pour obtenir la liste complète des valeurs de propriétés dans le SDK natif pour Android, consultez la section Parcourir les valeurs des propriétés SDK de ce document.

    Contrôles du lecteur par défaut

    Lorsque vous créez une application à l'aide du SDK Brightcove Player pour Android, vous devez commencer par BrightcovePlayer ou la BrightcovePlayerFragment classe.

    Étendre le BrightCovePlayer

    Dans votre code d'application, étendez la MainActivity classe de l'une des classes ci-dessus. Par exemple :

    public class MainActivity extends BrightcovePlayer {
}

    Maintenant, vous êtes prêt à instancier la vue vidéo dans votre code.

    Instancier le BrightCoveVideoView

    Dans le MainActivity.java fichier et dans la onCreate() méthode, ajoutez le code suivant :

    @Override
protected void onCreate(Bundle savedInstanceState) {
    setContentView(R.layout.activity_main);
    brightcoveVideoView = (BrightcoveExoPlayerVideoView) findViewById(R.id.brightcove_video_view);
    super.onCreate(savedInstanceState);
}

    Vos commandes de joueur devraient ressembler à ceci :

    Personnaliser les contrôles du lecteur

    Les commandes du lecteur Brightcove prennent en charge les sous-titres, le rembobinage vidéo et le mode plein écran. Vous pouvez également sélectionner le style d'animation pour afficher et masquer les contrôles.

    Modifier les couleurs du contrôle

    Facilement personnalisable, vous pouvez modifier les couleurs de contrôle du lecteur Brightcove pour qu'elles correspondent au matériel de marque de votre entreprise.

    1. Utilisez le même code dans votre fichier MainActivity.java que ci-dessus.
    2. Dans votre fichier src/main/res/values/colors.xml ou res/values/strings.xml , incluez le code suivant : 
      <?xml version="1.0" encoding="utf-8"?>
<resources>

    <!-- Define the new colors. -->
    <color name="bc_orange">#F3951D</color>
    <color name="magenta_semi_trans">#4DFF00FF</color>

    <!-- Change the progress colors to magenta/black/orange, all transparent. -->
    <color name="bmc_seekbar_played">@color/magenta_semi_trans</color>
    <color name="bmc_seekbar_buffered">@color/black_semi_trans</color>
    <color name="bmc_seekbar_track">@color/bc_orange</color>

</resources>

      Vos commandes de joueur devraient ressembler à ceci :

    Modifier les icônes et les boutons dans les commandes du lecteur

    Pour personnaliser davantage les commandes du lecteur Brightcove, vous pouvez échanger des icônes et ajouter vos propres boutons pour correspondre aux styles de votre entreprise.

    1. Utilisez le même code dans votre fichier MainActivity.java que ci-dessus.
    2. Dans votre fichier src/main/res/values/strings.xml , incluez le code suivant : 
      <?xml version="1.0" encoding="utf-8"?>
<resources>
    <!-- Use the Font Awesome fa-stepbackward icon for the rewind
         button face. -->
    <string name="brightcove_controls_rewind">&#xf048;</string>
</resources>

      Vos commandes de joueur devraient ressembler à ceci :

    Pour obtenir des exemples de code complets, consultez les contrôles du lecteur Brightcove pour Android Samples.

    Afficher/masquer les contrôles du lecteur

    Il existe trois façons d'afficher et de masquer les commandes du lecteur pendant la lecture vidéo :

    1. L'utilisateur peut appuyer sur l'écran de l'appareil pour basculer les commandes du lecteur entre l'affichage et le masquage des commandes.

    2. Vous pouvez utiliser la BrightcoveMediaController classe. Voici un exemple de la façon d'afficher les contrôles du joueur pour une plage de position de tête de lecture définie.

      brightcoveVideoView.getEventEmitter().on(EventType.PROGRESS, new EventListener() {
    @Override
    public void processEvent(Event event) {
        int playheadPosition = brightcoveVideoView.getCurrentPosition();
        if(playheadPosition > 24000 && playheadPosition < 29000) {
            BrightcoveMediaController controller = brightcoveVideoView.getBrightcoveMediaController();
            controller.show();
        }
    }
});

      Vous obtiendrez une exception de pointeur null si vous essayez d'appeler la getBrightcoveMediaController() méthode avant le chargement de la vidéo. Vous pouvez également écouter l' DID_SET_VIDEO événement avant d'appeler cette méthode.

      Sachez que les commandes du joueur se masquent automatiquement 3 secondes après les avoir montrées, sauf si vous remplacez ce comportement par défaut.

    3. Vous pouvez utiliser les événements comme suit :

      import static com.brightcove.player.mediacontroller.ShowHideController.SHOW_MEDIA_CONTROLS;
import static com.brightcove.player.mediacontroller.ShowHideController.HIDE_MEDIA_CONTROLS;
...
eventEmitter.emit(HIDE_MEDIA_CONTROLS);
eventEmitter.emit(SHOW_MEDIA_CONTROLS);

    Désactiver les contrôles de

    Lors de la conception de la mise en page et des fonctionnalités de votre application, vous devrez peut-être désactiver les commandes du lecteur. Cela peut être fait en définissant MediaController la valeur null comme suit :

    @
Override
public View onCreateView(LayoutInflater inflater, ViewGroup container, Bundle savedInstanceState) {
   View rootView = inflater.inflate(R.layout.fragment_player, container, false);
   brightcoveVideoView = (BrightcoveExoPlayerVideoView) rootView.findViewById(R.id.brightcove_video_view);
   super.onCreateView(inflater, container, savedInstanceState); // Need to be called after brightcoveVideoView is set.
   brightcoveVideoView.setMediaController((MediaController) null); // Need to be called after onCreateView.
   return rootView;
}

    Cet exemple utilise la fonction BrightcovePlayerFragment.

    La BrightcoveMediaController classe fournit les commandes par défaut du lecteur Brightcove et vous permet de les gérer et de les personnaliser, y compris la barre de contrôle, la barre de recherche et un ensemble de boutons de contrôle multimédia. Vous pouvez également désactiver les contrôles Brightcove Player pour utiliser les contrôles natifs Android.

    Contrôler manuellement le mode plein écran

    Généralement, les utilisateurs contrôlent l'affichage en plein écran en appuyant sur le bouton plein écran dans la barre de contrôle. Il peut y avoir des moments où vous voulez contrôler cette fonctionnalité manuellement, dans le code.

    Pour définir le mode plein écran, utilisez le code suivant :

    brightcoveVideoView.getEventEmitter().emit(EventType.ENTER_FULL_SCREEN);

    Pour quitter le mode plein écran, utilisez le code suivant :

    brightcoveVideoView.getEventEmitter().emit(EventType.EXIT_FULL_SCREEN);

    Pour obtenir la liste complète des types d'événements, reportez-vous au Kit de développement logiciel (SDK) du lecteur pour Android Reference et accédez au document de classe EventType.

    Vous pouvez également utiliser le fullScreen() et normalScreen() méthodes. Vous pouvez trouver ces méthodes dans le kit SDK Player pour Android Reference et accéder au document de classe BrightCovePlayer.

    Parcourir les valeurs des propriétés SDK

    Pour obtenir une liste complète de valeurs, y compris les tableaux, les couleurs, les dimensions, les identifiants, les entiers, les chaînes et les styles, procédez comme suit :

    1. Localisez le fichier android-native-player-public-x.x.x.zip et décompressez-le.

    2. Dans le dossier sdk , ouvrez le fichier android-sdk-x.x.x.aar .

      Une façon d'ouvrir un fichier aar est de changer l'extension de aar à zip et de le décompresser.
    3. Ouvrez le fichier res/values/values.xml . Vous trouverez ici une liste de valeurs que vous pouvez référencer dans vos applications.
