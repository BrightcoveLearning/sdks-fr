    Android : Sélection des sources

    Dans cette rubrique, vous apprendrez comment le SDK Brightcove Native Player pour Android utilise un processus de sélection de source par défaut lors de la lecture d'une vidéo.

    Introduction

    Lorsque vous récupérez des vidéos du catalogue Brightcove, le SDK doit décider de la source à lire. Ceci est implémenté dans les composants DefaultSourceElectionController et ExoPlayerSourceElectionController .

    Processus de sélection par défaut

    Le SDK Brightcove Player pour Android utilise un processus de sélection de source par défaut. Pour simplifier, nous nous concentrerons sur la prise en charge par le SDK de l'API de lecture Brightcove et de l'ExoPlayer de Google.

    Le processus de sélection par défaut est défini comme suit :

    1. Avec le SDK Native Brightcove pour Android 6.4+, la livraison HTTPS pour tous les types de sources est sélectionnée si disponible.

      DefaultSourceSelectionController et ExoPlayerSourceSelectionController tentent de sélectionner la version HTTPS du deliveryType sélectionné. Si aucune source HTTPS n'existe, le contrôleur de sélection sélectionne le premier disponible.

      Si votre vidéo a à la fois des sources HTTP et HTTPS pour les sous-titres/sous-titres WebVTT, le SDK natif pour Android peut ne pas renvoyer la version HTTPS. Brightcove est au courant de cette situation et aura un correctif dans une prochaine version.
    2. Le BrightCoveExoPlayerVideoView recherche la première source qui a un deliveryType de MPEG-DASH , et un profil qui n'est ni urn:hbbtv ni urn:dvb. Le SDK ne prend pas en charge la lecture avec l'un ou l'autre de ces profils DASH, et ils seront donc filtrés hors des sources.
    3. Si ce qui précède n'est pas trouvé, la sélection de la source revient à la première source avec un deliveryType de HLS.
    4. Si aucune source de ce type n'est trouvée, elle revient à la source avec un deliveryType de MP4 et un débit binaire le plus proche de 256 kBps.

    Une fois qu'une source est trouvée, l'implémentation se poursuit comme suit :

    1. Si elle est trouvée, la source sélectionnée sera émise avec une réponse à l'événement SELECT_SOURCE .
    2. Par défaut, VideoPlaybackController gère la réponse SELECT_SOURCE , en émettant un événement SET_SOURCE .
    3. VideoDisplayComponent ou une sous-classe gère l'événement SET_SOURCE en chargeant l'URL dans le lecteur sous-jacent (ExoPlayer ou MediaPlayer).
    4. Lorsque la lecture commence, le joueur sous-jacent est responsable de l'adaptation du débit binaire avec le contenu HLS et DASH. Avec HLS, le joueur sous-jacent commence par le premier débit indiqué dans le manifeste principal.

    Lorsque vous utilisez les paramètres de transcodage par défaut, la première source HLS doit être la liste de lecture principale m3u8 qui pointe vers toutes vos sources individuelles (formats associés).

    Définition des sources HLS

    Il est possible de personnaliser la sélection de la source au niveau de l'application, pour filtrer les types de remise indésirables avant que la stratégie de sélection de source par défaut du SDK ne soit invoquée. Pour ce faire, vous devez modifier les collections source qui sont définies dans l'objet Video à partir de la réponse JSON de l'API Playback.

    Cet exemple utilise uniquement les sources HLS :

    catalog.findVideoByReferenceID(videoReferenceId, new com.brightcove.player.edge.VideoListener() {
	@Override
	public void onVideo(Video video) {
	 sourceSelectionFilter(DeliveryType.HLS, video);
	 brightcoveVideoView.add(video);
	 brightcoveVideoView.start();
	}
});

    Et la sourceSelectionFilter() méthode ressemblerait à ceci :

    public static void sourceSelectionFilter(@NonNull DeliveryType deliveryTypeFilter, @NonNull Video video) {
 if (video.getSourceCollections().containsKey(deliveryTypeFilter)) {
   // Only remove the other delivery types if the desired type is in our Source collections
   for (DeliveryType deliveryType : DeliveryType.values()) {
     if (!deliveryType.equals(deliveryTypeFilter)) {
            video.getSourceCollections().remove(deliveryType);
     }
   }
 }
}
