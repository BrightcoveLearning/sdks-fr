introduction

Testez vos applications vidéo avec les messages de journal par défaut pour faciliter le débogage. Lorsque vous êtes prêt à publier votre application, vous pouvez supprimer ces messages de journal.

Supprimer les messages du journal

Pour supprimer les messages de journal par défaut du SDK natif pour Android, procédez comme suit :

  1. Dans votre application, ouvrez le règles-proguard.pro déposer.
     
    Dans Android Studio, ce fichier est créé par défaut. Si vous vous demandez où se trouve ce fichier dans nos exemples Android Brightcove, nous avons renommé ce fichier en projet-proguard.txt.
  2. Ajoutez le bloc de code suivant pour supprimer le journal de votre application : 
    -assumenosideeffects class android.util.Log {
  public static *** v(...);
  public static *** d(...);
  public static *** i(...);
  public static *** w(...);
  public static *** e(...);
}
    Pour plus de détails sur les messages de journal, consultez le Classe de journal Android Documentation.
  3. Ensuite, ouvrez votre application build.gradle déposer.
  4. Dans le buildTypes section, ajoutez le minifyEnabled attribut avec une valeur de true.
  5. Sous le code précédent, ajoutez un proguardFiles attribut comme suit :
    • Utilisez le getDefaultProguardFile() méthode pour obtenir le proguard-android-optimize.txt fichier à partir du SDK Android.
    • Référencez le règles-proguard.pro déposer.

    Ton build.gradle le fichier devrait ressembler à ceci : 
    android {
    compileSdkVersion 25
    buildToolsVersion '25.0.0'

    defaultConfig {
        ...
    }

    buildTypes {
        release {
            minifyEnabled true
            proguardFiles getDefaultProguardFile('proguard-android-optimize.txt'),
            'proguard-rules.pro'
        }
    }
}
Android Studio désactive temporairement la bibliothèque de couverture de code Java (JaCoCo) et ProGuard lors de l'utilisation d'Instant Run. Étant donné que Instant Run ne fonctionne qu'avec les versions de débogage, cela n'affecte pas votre version de version (https://developer.android.com/studio/run/index.html).

Si vous souhaitez supprimer les journaux de votre type de build de débogage, vous devez désactiver l'exécution instantanée.
