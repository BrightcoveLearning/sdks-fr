    Plugin SSAI pour le SDK natif pour Android

    Dans cette rubrique, vous apprendrez comment utiliser le plugin d'insertion d'annonces côté serveur (SSAI) avec le SDK natif Brightcove pour Android. Il explique également comment migrer à partir de l'utilisation du plugin OnCeux.

    Utilisation du plugin SSAI

    Le plugin SSAI vous permet de lire du contenu avec Dynamic Delivery et la publicité côté serveur.

    Il existe deux options pour lire du contenu avec des annonces côté serveur :

    1. Fournir une URL pour le contenu SSAI
    2. Fournir une vidéo avec VMAP

    Dans les deux cas, le plugin SSAI demandera la source VMAP, l'analyse et l'ajoutera automatiquement au VideoView.

    Fournir une URL pour le contenu SSAI

    Avec une URL d'actifs SSAI, appelez la méthode suivante :

    SSAIComponent.processVideo(String url)

    Voici un exemple :

    String ssaiUrl = "https://onceux.host.com/path/to/content.once"
SSAIComponent ssaiComponent = new SSAIComponent(context, brightcoveVideoView);
ssaiComponent.processVideo(ssaiUrl);

    Fournir une vidéo avec VMAP

    Pour cette option, vous devez fournir une vidéo avec une source contenant une propriété VMAP avec l'URL de l'actif SSAI. Pour utiliser le plugin SSAI, procédez comme suit :

    1. Pour demander du contenu VideoCloud SSAI, votre compte doit d'abord être configuré pour utiliser Dynamic Delivery et SSAI. Pour plus de détails, consultez le document Video Cloud SSAI Overview .
    2. Ensuite, vous devrez créer un objet AdConfig (Ad Configuration). Pour plus d'informations, consultez le document Configuration des paramètres publicitaires côté serveur . Veuillez noter que le SDK natif Brightcove pour Android ne contient pas lui-même de fonctionnalités permettant de créer ou de gérer ADConfigs.
    3. Utilisez la id propriété de AdConfig en tant que paramètre URL (ad_config_id ) ajouté à l'appel de méthode Catalog : 
      findVideoById

      ou

      findVideoByReferenceId
    4. Après avoir récupéré l'objet Video avec la méthode FindVideo, appelez la méthode suivante dans le plugin SSAI : 
      SSAIComponent.processVideo(Video ssaiVideo)

    Voici un exemple :

    Catalog catalog = new Catalog(eventEmitter, myAccountId, myPolicyKey);
SSAIComponent plugin = new SSAIComponent(appContext, getBrightcoveVideoView());

// Set the ad_config_id URL parameter
HttpRequestConfig httpRequestConfig = new HttpRequestConfig.Builder()
        .addQueryParameter("ad_config_id", myAdConfig)
        .build();

// Find the Video by ID
catalog.findVideoByID(myVideoId, httpRequestConfig, new VideoListener() {
        @Override
        public void onVideo(Video video) {

        // Process the Video
        plugin.processVideo(video);
    }
});

    Travailler avec les annonces compagnons

    Si vos annonces contiennent des Compagnons, vous pouvez facilement les afficher dans votre application en fournissant un ViewGroup pour conserver l'image Compagnon, en appelant :

    SSAIComponent.addCompanionContainer(ViewGroup companionContainer)

    Voici un exemple. Tout d'abord, ajoutez le FrameLayout à votre disposition d'activité ou de fragment :

    // my_activity.xml
<RelativeLayout xmlns:android="http://schemas.android.com/apk/res/android"
                xmlns:app="http://schemas.android.com/apk/res-auto"
                android:layout_width="match_parent"
                android:layout_height="match_parent">
 ...
  <FrameLayout
        android:id="@+id/ad_frame"
        android:layout_width="300dp"
        android:layout_height="250dp"
        android:visibility="invisible" />
…
</RelativeLayout>

    Ensuite, utilisez-le FrameLayout pour le passer au plugin :

    ViewGroup viewGroup = findViewById(R.id.ad_frame);
plugin.addCompanionContainer(viewGroup);

// process your Video
plugin.processVideo(video);

    Gestion des événements

    Pour les événements spécifiques à SSAI, vous pouvez les écouter en utilisant le EventEmitter. Ces événements peuvent être trouvés dans la SSAIEventType classe.

    le START_AD_SEGMENT et END_AD_SEGMENT les événements ont été remplacés par EventType.AD_BREAK_STARTED et EventType.AD_BREAK_COMPLETED respectivement. Pour plus de détails, voir l'annexe B et l'annexe C.

    Pour obtenir des positions relatives et absolues, obtenez la com.brightcove.ssai.AdPod avec la SSAIEvent.AD_POD clé. Voici un exemple :

    private void registerEventHandlers() {
  EventEmitter eventEmitter = brightcoveVideoView.getEventEmitter();
  eventEmitter.on(EventType.AD_BREAK_STARTED, new EventListener() {
    @Override
    public void processEvent(Event event) {
      Object adPodObject = event.getProperties().get(SSAIEvent.AD_POD);
      if (adPodObject instanceof AdPod) {
        AdPod adPod = (AdPod) adPodObject;
        adPod.getRelativeStartPosition();
        adPod.getAbsoluteStartPosition();
        adPod.getAbsoluteEndPosition();
      }
    }
  });
}

    Gestion des erreurs

    Toutes les erreurs seront apparentes au développeur utilisant l' EventType.ERROR événement, comme indiqué ici :

    eventEmitter.on(EventType.ERROR, new EventListener() {
  @Override
  public void processEvent(Event event) {
    String errorMessage = event.getProperty(Event.ERROR_MESSAGE, String.class);
    Throwable error = event.getProperty(Event.ERROR, Throwable.class);
    Log.w(TAG, String.format("Received the error message: %s with error: %s", errorMessage, error));
  }
});

    Migration à partir du plugin ONCeux

    Si vous utilisez déjà le plugin ONCeux pour lire vos vidéos SSAI dans votre application, vous n'avez qu'à suivre quelques étapes pour migrer vers le nouveau plugin SSAI.

    1. Dans le fichier build.gradle de votre application, remplacez 
      implementation 'com.brightcove.player:android-onceux-plugin:6.5.1'

      avec ce qui suit : (utilisez la dernière version du plugin)

      implementation 'com.brightcove.player:android-ssai-plugin:6.7.0'
    2. Remplacez la OnceUxComponent classe par la SSAIComponent classe.
    3. Remplacer OnceUxEventType.NO_AD_DATA_URL par EventType.ERROR.
    4. Si votre code utilise l'une des constantes trouvées dans la OnceUxEventType classe, remplacez-les par SSAIEventType et SSAIEvent les constantes, en fonction de leur fonction :
      • Les constantes utilisées pour écouter un événement sont maintenant trouvées dans la SSAIEventType classe. Par exemple : 
        SSAIEventtType.AD_DATA_READY
      • Les constantes utilisées comme clés pour les propriétés d'événement se trouvent maintenant dans la classe SSaiEvent. Par exemple : 
        SSAIEvent.SSAI_AD
    5. Les plugins SSAI et ONCeux partagent une bibliothèque interne chargée de l'analyse de l'objet VMAP. Si votre application utilise directement des objets de cette bibliothèque (package com.brightcove.iab), il y a des modifications dans la nouvelle version que vous pourriez trouver utiles.

    6. Pour en savoir plus, consultez l'annexe A.
