    Personnalisation de l'interface utilisateur publicitaire Android avec insertion d'annonces côté serveur (SSAI)

    Dans cette rubrique, vous apprendrez comment gérer et personnaliser les éléments d'interface utilisateur affichés lors de la lecture d'annonces côté serveur, à l'aide du plugin SSAI pour le SDK Brightcove Native pour Android.

    Présentation

    Lorsque vous travaillez avec l'insertion d'annonces côté serveur (SSAI), le SDK natif Brightcove pour Android vous permet de personnaliser les éléments de l'interface utilisateur des annonces. Pour plus d'informations sur SSAI, consultez les sections suivantes :

    • Implémentation des annonces côté serveur avec les kits SDK du lecteur natif
    • Le plugin SSAI pour le SDK natif pour Android

    L'interface utilisateur de la publicité est composée de deux composantes principales :

    • Superposition d'annonce (A)
    • Contrôleur de média publicitaire (B)

    Superposition d'annonce

    La superposition d'annonce est composée des éléments suivants :

    • Compte à rebours de la durée totale du Pod publicitaire (A)
    • En savoir plus vue (B)
    • Ignorer la vue de l'annonce (C)
    Le bouton Ignorer la publicité est uniquement pris en charge par l'insertion d'annonces côté serveur VOD (SSAI). Il n'est actuellement pas pris en charge avec Live SSAI.

    Contrôleur média publicitaire

    Le contrôleur de média publicitaire est composé des composants suivants, de gauche à droite :

    • Bouton de lecture (A)
    • Compte à rebours du nombre d'annonces (B)
    • Compte à rebours de durée d'annonce unique (C)
    • Bouton plein écran (D)

    Gérer les composants de l'interface utilisateur

    Pour activer ou désactiver les composants de l'interface utilisateur d'annonces dans la vue, consultez les points suivants :

    • Modification du document VAST
    • Spécification de la configuration de l'annonce

    Modification du document VAST

    Les vues En savoir plus et Ignorer la publicité peuvent être affichées ou masquées en fonction de la définition de l'annonce dans le document VAST.

    L'affichage En savoir plus s'affiche uniquement lorsque la création linéaire contient une ClickThrough URL, comme illustré ici :

     <VAST xmlns:xsd="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
  <Ad id="blue-01-5s">
    <InLine>
      ...
      <Creatives>
        <Creative id="creative-01-5s">
          <Linear>
            ...
            <VideoClicks>
              <ClickThrough id="clickthrough">https://www.brightcove.com/en/</ClickThrough>
              <ClickTracking id="...”>...</ClickTracking>
            </VideoClicks>
          </Linear>
        </Creative>
      </Creatives>
    </InLine>
  </Ad>
</VAST>

    L'affichage Ignorer l'annonce apparaîtra uniquement lorsque skipoffset la valeur de création linéaire est valide :

     <VAST xmlns:xsd="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
  <Ad id="blue-01-5s">
    <InLine>
      ...
      <Creatives>
        <Creative id="creative-01-5s">
          <Linear skipoffset="00:00:03">
            ...
          </Linear>
        </Creative>
      </Creatives>
    </InLine>
  </Ad>
</VAST>

    Spécification de la configuration de l'annonce

    Lorsque vous utilisez Player Enhancements for Live SSAI, vous pouvez activer ou désactiver des composants dans la application_ad_configuration de la configuration de votre annonce. L' client_options objet vous permet d'activer ou de désactiver les composants de l'interface utilisateur des annonces.

    La modification de l'interface utilisateur de l'annonce à l'aide de la configuration de l'annonce s'applique uniquement à Live SSAI.
    {
  "application_ad_configuration": {
	"ad_configuration_description": "$YOUR_DESCRIPTION",
	"ad_configuration_expected_response_type": "Vast",
	"ad_configuration_headers_for_impressions": false,
	"ad_configuration_strategy": "SingleAdResponse",
	"ad_configuration_transforms": [],
	"ad_configuration_url_format": "$YOUR_AD_SERVER",
     "ad_configuration_client_sdk_enabled": true,
     "client_options": {
        "show_ad_break_remaining_time": true / false,
        "show_ad_remaining_time": true / false,
        "show_number_of_remaining_ads": true / false,
        "client_only_tracking": true / false
     }
  },
  "application_description": "$YOUR_DESCRIPTION"
}
    Si vous utilisez des client_options composants, vous devez avoir ad_configuration_client_sdk_enabled défini sur true.

    Utilisez l'objet client_options pour activer/désactiver les composants suivants :

    • Show_ad_break_remaining_time: Le compte à rebours de la durée totale du Pod d'annonces
      • Exemple : Votre vidéo reprendra dans 15 secondes
    • Show_ad_remaining_time: Le compte à rebours unique de la durée de l'annonce
      • Exemple : ( 0:15)
    • Show_number_of_remaining_ads: Le compte à rebours du numéro d'annonce
      • Exemple : Annonce 1 de 3, Annonce 2 de 3, etc.

    Personnalisation de la superposition d'annonces

    Vous pouvez personnaliser les vues et le texte par défaut associé à la superposition d'annonces.

    Modification de la vue

    Pour personnaliser la superposition d'annonces, vous pouvez remplacer votre version du fichier de mise en page ssai_ad_overlay.xml dans le dossier res de votre application. Ce fichier contient plusieurs mises en page indépendantes différentes, comme indiqué ici :

    Par exemple, vous pouvez remplacer la vue de durée du conteneur d'annonces en mettant à jour le fichier ssai_ad_pod_duration_countdown_view.xml . Vous pouvez également remplacer l'ensemble de la vue de superposition d'annonces en remplaçant le fichier ssai_ad_overlay.xml .

    Cette structure de mise en page vous permet de remplacer des composants particuliers tout en maintenant la compatibilité ascendante avec les anciennes versions de plugin SSAI.

    Explorez les fichiers de superposition d'annonces à l'aide d'Android Studio. À partir de la version 6.9.0 du SDK natif pour Android, le fichier ssai_ad_overlay.xml du plugin SSAI ressemble à ceci :

    <?xml version="1.0" encoding="utf-8"?>
<FrameLayout xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:tools="http://schemas.android.com/tools"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    tools:background="@color/white">

   <include layout="@layout/ssai_ad_countdown_and_learn_more_overlay"/>
   <include layout="@layout/ssai_ad_skip_overlay"/>
</FrameLayout>

    Lors du remplacement d'un fichier xml de composant, vous devez utiliser un TextView avec l'ID de vue d'origine spécifié pour éviter des résultats imprévus. Pour plus de détails, voir le tableau suivant

    Nom du composant ID du composant Vue des composants
    Vue de la durée du module d'annonces text_ad_countdown TextView
    En savoir plus text_ad_learn_more TextView
    Ignorer l'affichage des annonces text_ad_skip TextView

    Modification du texte par défaut

    Tous les composants Ad Overlay sont TextViewsdéfinis dans le fichier string.xml de votre application. Pour modifier le texte d'affichage, remplacez-le en créant un string ou plurals élément utilisant le même identifiant de chaîne que celui utilisé par le SDK natif pour Android.

    La vue de durée du conteneur d'annonces utilise les chaînes suivantes :

    <plurals name="ssai_message_ad_break_duration_countdown">
   <item quantity="one">Your video will resume in\n%1$d second</item>
   <item quantity="other">Your video will resume in\n%1$d seconds</item>
</plurals>
<string name="ad_info_now_text">Your video will resume now</string>
<string name="ad_buffering_text">Your ad is buffering …</string>

    La vue En savoir plus utilise la chaîne suivante :

    <string name="ssai_message_learn_more">Learn More >> </string>

    La vue Ignorer l'annonce utilise les chaînes suivantes :

    <plurals name="you_can_skip_text">
   <item quantity="one">You can skip this ad in\n%d second</item>
   <item quantity="other">You can skip this ad in\n%d seconds</item>
</plurals>
<string name="skip_text">Skip Ad</string>

    Personnalisation du contrôleur de média publicitaire

    Vous pouvez personnaliser les vues et le texte par défaut associés au contrôleur de média publicitaire.

    Modification de la vue

    Pour personnaliser le contrôleur Ad Media, vous pouvez remplacer votre version du fichier de mise en page default_ssai_ad_media_controller.xml dans le dossier res de votre application.

    Cela remplace la disposition par défaut utilisée dans le plugin SSAI. Chaque fois qu'une annonce est détectée, le plugin SSAI bascule vers cette mise en page, puis retourne au contrôleur par défaut lorsque l'annonce est terminée.

    Lorsque vous travaillez avec votre propre mise en page, utilisez le même type de vue et les mêmes identifiants de vue que ceux spécifiés à l'origine pour éviter des résultats imprévus. Explorez la mise en page du contrôleur multimédia publicitaire à l'aide d'Android Studio. Pour le plugin SSAI version 6.9.0, le default_ssai_ad_media_controller.xml ressemble à ceci :

    <com.brightcove.player.mediacontroller.BrightcoveControlBar
   xmlns:android="http://schemas.android.com/apk/res/android"
   xmlns:tools="http://schemas.android.com/tools"
   android:id="@+id/brightcove_control_bar"
   style="@style/SSAIAdControlBar"
   android:layout_width="match_parent"
   android:layout_height="wrap_content"
   android:layout_gravity="bottom"
   android:background="@color/bmc_background"
   android:orientation="vertical"
   android:padding="8dp">

   <LinearLayout
       android:layout_width="match_parent"
       android:layout_height="wrap_content"
       android:gravity="bottom"
       android:orientation="horizontal">

       <Button style="@style/BorderlessButton"
           android:id="@id/play"
           android:layout_width="wrap_content"
           android:layout_height="wrap_content"
           android:text="@string/brightcove_controls_play"
           android:visibility="gone"
           tools:visibility="visible"/>

       <include layout="@layout/ssai_ad_number_countdown_view"/>

       <include layout="@layout/ssai_single_ad_duration_countdown_view"/>

       <View
           android:id="@id/two_line_spacer"
           android:layout_width="0dp"
           android:layout_height="0dp"
           android:layout_weight="1"
           android:visibility="gone"
           tools:visibility="visible"/>

       <Button style="@style/BorderlessButton"
           android:id="@id/full_screen"
           android:layout_width="wrap_content"
           android:layout_height="wrap_content"
           android:padding="4dp"
           android:text="@string/brightcove_controls_enter_full_screen"
           android:visibility="gone"
           tools:visibility="visible"/>
   </LinearLayout>

</com.brightcove.player.mediacontroller.BrightcoveControlBar>

    Les types de vue et les identifiants de vue des composants SSAI Ad Media Controller sont décrits ici :

    Nom du composant ID du composant Vue des composants
    Bouton de lecture play Button
    Bouton Plein écran full_screen Button
    Compte à rebours du numéro d'annonce text_ad_number_countdown TextView
    Compte à rebours de durée d'annonce unique text_single_ad_duration_countdown TextView

    Consultez l' exemple d'application Contrôles personnalisés, où le contrôleur multimédia par défaut est personnalisé pour ajouter de nouveaux boutons. Vous pouvez appliquer la même approche pour personnaliser votre contrôleur média publicitaire SSAI.

    Modification du texte par défaut

    Tous les composants du contrôleur Ad Media sont TextViewsdéfinis dans le fichier string.xml de votre application. Pour modifier le texte d'affichage, remplacez-le en créant un string ou plurals élément utilisant le même identifiant de chaîne que celui utilisé par le SDK natif pour Android.

    La vue Compte à rebours du numéro d'annonce utilise la chaîne suivante :

    <string name="ssai_message_ad_number_countdown">Ad %1$d of %2$d</string>

    La vue Compte à rebours de la durée de l'annonce unique utilise la chaîne suivante :

    <string name="ssai_message_ad_duration_countdown">(%1$s)</string>

    Résumé

    Vous avez quelques options pour personnaliser les éléments de l'interface utilisateur SSAI. Le niveau de complexité dépendra de ce que vous voulez atteindre. De faible à haut niveau de complexité, vous pouvez :

    • Configure/masquer certains composants via votre configuration VAST
    • Afficher/masquer les composants à l'aide de client_options l'objet dans la configuration de votre annonce (actuellement Live SSAI uniquement)
    • Modifier la valeur de chaîne affichée
    • Remplacer la disposition de l'interface utilisateur

    Veuillez partager vos idées ou vos commentaires sur le forum des SDK Brightcove Native Player.
