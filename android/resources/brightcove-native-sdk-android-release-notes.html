    Notes de mise à jour du kit SDK natif de Brightcove pour Android

    Dans cette rubrique, vous trouverez les dernières informations sur les modifications et améliorations apportées au SDK natif Brightcove pour Android.

    Pour rejoindre la liste de diffusion et recevoir des annonces de sortie pour les SDK natifs, inscrivez-vous ici.

    Version actuelle

    Version 6.16.0

    14 oct. 2020

    Changements de plugin Cast :

    • Le plugin Cast a été repensé et prend désormais en charge le récepteur Brightcove Cast Receiver v2.0.

    • Pour plus d'informations et des exemples de code, reportez-vous à la documentation du SDK Android Native de Brightcove et aux exemples d'applications Brightcove Android Native SDK.

    Versions précédentes

    Version 6.15.1

    22 sept. 2020

    Modifications du SDK :

    • Le SDK natif Brightcove pour Android prend désormais en charge la langue arabe (RTL) dans les commandes des lecteurs et les étiquettes d'interface utilisateur.

    • Remarque : Bien que le texte en langue arabe (RTL) soit pris en charge, seules les dispositions de contrôle du lecteur de gauche à droite (LTR) sont actuellement prises en charge.

    • Correction d'un problème dans lequel certains paramètres d'événement video_view et video_engagement Analytics pouvaient inclure des chiffres arabes non occidentaux pour les paramètres suivants :

      • video_view:
        • start_time_ms
        • load_time_ms
      • video_engagement:
        • range
        • video_seconds_viewed

    Changements de plugin Cast :

    • Le plugin Cast prend désormais en charge le texte arabe (RTL) dans les contrôles de casting et les étiquettes d'interface utilisateur.

    • Veuillez noter que la prise en charge de la langue arabe ne s'étend pas aux écrans (durée et durée de la vidéo) dans le contrôleur étendu Cast.

    • L'affichage de ces composants est contrôlé par l'API Google Cast.

    Modifications du plug-in de lecture hors ligne :

    • Le plugin de lecture hors ligne prend désormais en charge le texte arabe (RTL) dans les étiquettes d'interface utilisateur.

    Modifications du plugin Pulse :

    • Le plugin Pulse prend désormais en charge le texte arabe (RTL) dans les contrôles publicitaires et les étiquettes d'interface utilisateur.

    Changements de plug-in SSAI :

    • Le plugin SSAI prend désormais en charge le texte arabe (RTL) dans les contrôles publicitaires et les étiquettes d'interface utilisateur.

    Version 6.15.0

    25 août 2020

    Modifications du SDK :

    • Mise en œuvre d'une vérification NULL supplémentaire pour les URI de légende

    Modifications ExoPlayer :

    • Correction d'une régression dans la ExoPlayerVideoDisplayComponent setPeakBitrate() méthode dans laquelle le débit spécifié était ignoré, et le débit le plus élevé était sélectionné à la place.

    Modifications du plugin IMA :

    • Le SDK Google IMA a été mis à jour vers la version 3.19.4. Voici quelques-uns des changements suivants :

      • SdkFactory.createAdDisplayContainer() est obsolète
      • AdDisplayContainer.setPlayer(VideoAdPlayer) est obsolète
      • AdDisplayContainer.setAdContainer(ViewGroup) est obsolète

      • AdsRequest.setAdDisplayContainer(AdDisplayContainer) est supprimé

        Pour obtenir la liste complète des modifications, reportez-vous à l'historique des versions du SDK Android de Google IMA.

    • Le utilise GoogleIMAComponent maintenant un Builder pour sa construction.

    • Le plugin IMA crée automatiquement l' AdDisplayContainer utilisant BrightcoveVideoView par défaut.

      Vous pouvez fournir un AdDisplayContainerFactory à la GoogleIMAComponent.Builder pour fournir votre propre AdDisplayContainer.

    • Les méthodes suivantes ont été obsolètes dans la GoogleIMAVideoAdPlayer classe :
      • playAd(). Use playAd(AdMediaInfo) à la place
      • loadAd(String). Use loadAd(AdMediaInfo, AdPodInfo) à la place
      • stopAd(). Use stopAd(AdMediaInfo) à la place
      • resumeAd(). Use playAd(AdMediaInfo) à la place

      • pauseAd(). Use pauseAd(AdMediaInfo) à la place

    • Correction d'un problème où le plugin IMA n'émet pas les AD_PAUSED/AD_RESUME événements lors de l'appel GoogleIMAVideoAdPlayer.pauseAd et GoogleIMAVideoAdPlayer.resumeAd , respectivement.

      Pour plus d'informations, consultez la présentation du SDK Android Brightcove sur le site de support Brightcove.

    Modifications du plugin VMAP :

    • Le plugin VMAP obsolète a été supprimé du SDK Brightcove Native pour Android. Pour la diffusion d'annonces VMAP, veuillez continuer à utiliser le plugin IMA.

    Modifications du plugin ONCeux :

    • Le plugin OnCeux obsolète a été supprimé du SDK Brightcove Native pour Android. Veuillez utiliser le plugin SSAI à la place.

    Version 6.14.1

    7 août 2020

    Changements de plug-in SSAI :

    • Résout un problème dans lequel l' CustomClick élément ne parvient pas à analyser.

    • Résout un problème lors de l'analyse de l'élément VAST Extension.

    Version 6.14.0

    8 juil. 2020

    Modifications du SDK :

    • Intégration améliorée avec Android TalkBack (et Samsung Voice Assistant).

    • Le VideoParser va maintenant ignorer toute source DRM qui ne contient pas de systèmes de clés Widevine.

    • Correction d'un problème qui ne VideoParser parvenait pas à analyser une source VTT miniature qui ne contient pas la propriété bande passante.

    • Correction d'un problème sur les plates-formes Android TV où le bouton Options du lecteur ne parvenait pas à s'afficher après la fin des pauses d'insertion d'annonces côté serveur.

    Modifications du plugin IMA :

    • ExoPlayer est maintenant utilisé pour la lecture d'annonces au lieu de MediaPlayer.

    Version 6.13.3

    21 mai 2020

    Modifications du SDK :

    • Amélioration du rendu du texte rubis dans les sous-titres codés sidecar. Une amélioration similaire est à venir pour les sous-titres dans le manifeste.

    Changements de plugin Cast :

    • Correction d'un problème où l'activité n'était pas libérée comme prévu lorsque le plugin Chromecast est utilisé.

    Modifications du plugin IMA :

    • Ajout de la prise en charge du ciblage par segment d'audience (publicité), en vertu duquel des paires clé/valeur sont ajoutées au paramètre cust_params de l'URL de la balise d'annonce.

    Changements de plug-in SSAI :

    • Ajout de la prise en charge du ciblage par segment d'audience (Ad), par lequel des paires clé/valeur sont ajoutées à l'URL source VMAP.

    • Correction d'un problème dans lequel une URL vide ou mal formée dans un élément de suivi VAST pouvait provoquer un plantage d'application

    Version 6.13.2

    24 avr. 2020

    Modifications de la lecture hors connexion :

    • Mise à niveau de la dépendance FileDownloader vers v1.7.7

      Cette mise à niveau résout un problème dans lequel le téléchargement alors que l'application est en arrière-plan pourrait s'interrompre de façon inattendue.

    Modifications du plugin miniature :

    • Correction d'un problème d'alignement entre le scrubber et l'aperçu des vignettes sur Android TV

    Version 6.13.1

    16 avr. 2020

    Modifications du SDK :

    • Ajout de la prise en charge des vidéos HEVC.

      Sur les appareils prenant en charge HEVC (avec API de niveau 21 et supérieur), la lecture de HEVC est désormais automatiquement disponible. Voir la documentation pour plus de détails techniques.

    Modifications du plugin Pulse :

    • Ajout de la recherche sans support publicitaire lors de adsDisabled la définition de VideoPlaybackController

    • Ajout du support pour lire les annonces Pulse passback

    • Ajout d'un message pour indiquer quand l'utilisateur est dans une annonce de pause

    Modifications du plugin miniature :

    • Augmentation de la taille de l'image miniature affichée dans les tablettes

    • Augmentation de la vitesse de déplacement des miniatures sur Android TV

    Changements de plug-in SSAI :

    • Correction d'un problème qui faisait que les sous-titres codés ne sont pas synchronisés avec la vidéo

    Version 6.12.0

    5 mars 2020

    Modifications du SDK :

    • Obsolète BrightcoveSurfaceView.isVideo360Supported(). Basé sur les versions actuelles prises en charge, il renvoie toujours true.

    Plugin Pulse :

    • Avec cette version, le SDK Native Brightcove pour Android implémente le Plugin Advertising Pulse.

      Veuillez noter que cette nouvelle fonctionnalité est disponible en disponibilité limitée. Contactez votre gestionnaire de compte pour plus d'informations.

    Version 6.11.0

    7 févr. 2020

    Modifications du SDK :

    • Ajout de la prise en charge du service de gestion des droits de lecture Brightcove.

      Veuillez noter que cette nouvelle fonctionnalité est disponible en disponibilité limitée.

    • Changement de rupture: L'objet Playlist est maintenant immuable.

    • L'objet Playlist peut désormais contenir un mélange de vidéos avec statut Status.OK et Status.ERROR

    • Playlist.getErrorVideos(): Cette nouvelle méthode ne retournera que les vidéos avec l'état Status.ERROR

    • Playlist.getAllVideosAndErrors(): Cette nouvelle méthode retournera toutes les vidéos avec les deux statuts Status.OK et Status.ERROR , suivant le même ordre que celui renvoyé à partir de la réponse du service de gestion des droits de lecture

    • Remarque : Playlist.getVideos() et Playlist.getCount() se comportent de la même manière que les versions précédentes, c'est-à-dire qu'ils ne considèrent que les vidéos avec statut Status.OK

    • Ajouté ErrorListener.onError(List) rappel.

    • ErrorListener.onError(String) Callback obsolète.

    • L'objet Vidéo a maintenant un code d'état, qui peut être Status.OK ou Status.ERROR. Vous pouvez récupérer le statut via Video.getStatus():Status.

    • Status.OK: La vidéo a toutes les propriétés attendues et n'a pas Video.Fields.ERRORS dedans.

    • Status.ERROR: La vidéo a Video.Fields.ERRORS dans ses propriétés sous la forme de List<CatalogError>.

    Plugin VMAP :

    • Le plugin VMAP a été obsolète. Pour la publicité côté client avec des documents VMAP et VAST, veuillez utiliser le plugin IMA.

    Version 6.10.1

    17 janv. 2020

    Modifications ExoPlayer :

    • Correction d'un plantage lors de la tentative de lecture d'un flux HLS Live.

    Modifications de la lecture hors connexion :

    • Correction d'un problème qui empêchait le téléchargement de certains fichiers de sous-titres codés de VTT.

    Plugin Cast :

    • Ajout de nouvelles CastMediaUtil.toMediaInfo méthodes sans le paramètre de contexte Android inutile.

    • CastMediaUtil.toMediaInfo Méthodes obsolètes avec le paramètre de contexte Android inutile.

    Version 6.10.0

    19 déc. 2019

    Modifications du SDK :

    • Changement de rupture: Le SDK natif Android Brightcove est maintenant construit à l'aide des bibliothèques AndroidX (JetPack). Reportez-vous aux pages Support SDK Android pour plus d'informations sur cette modification et les étapes requises pour migrer le code de votre application.

    • Correction d'un problème lié à la backgroundOpacity méthode dans BrightcoveCaptionStyle lequel cette méthode ne définissait pas l'opacité en arrière-plan des légendes comme prévu.

    • Correction d'un problème où la fonctionnalité plein écran ne fonctionnait pas comme prévu dans le BrightcovePlayerFragment.

    Modifications ExoPlayer :

    • La prise en charge d'ExoPlayer dans le SDK natif Android Brightcove a été mise à niveau vers la version 2.10.8.

    • Correction d'un problème de sélection des pistes audio qui empêcherait la sélection des rendus à débit supérieur après le changement de langue.

    Modifications du plugin IMA :

    • Ajout de la prise en charge du contenu en direct avec des publicités pré-lancées côté client.

    Changements de plug-in SSAI :

    • Correction d'un problème qui faisait que le bouton Légendes ne s'affichait pas systématiquement pendant la lecture du contenu. Veuillez noter que ce problème est actuellement résolu pour les appareils mobiles uniquement ; une solution pour les appareils Android TV suivra dans une prochaine version.

    Version 6.9.0

    22 oct. 2019

    Modifications du SDK :

    • Le SDK Android cible et prend désormais en charge Android 10 (TargetSDKVersion 29).

    • Corrige une exception RuntimeException : « Canvas : essayer d'utiliser un bitmap recyclé », lors de la tentative d'affichage de l'image de l'affiche.

    • Ajout des constantes de commodité suivantes en tant que clé de paramètres de requête HttpRequestConfig :

      • HttpRequestConfig.KEY_DELIVERY_RULE_CONFIG_ID Utilisé pour ajouter un ID de configuration des règles de livraison
      • HttpRequestConfig.KEY_AD_CONFIG_ID Utilisé pour ajouter un ID de configuration publicitaire SSAI

    ExoPlayer v1 Changements :

    • Toute la prise en charge d'ExoPlayer 1.x a été supprimée de The Brightcove Native SDK pour Android. Assurez-vous d'utiliser ExoPlayer v2. Pour plus d'informations sur le processus de migration, veuillez consulter le site https://support.brightcove.com/migrating-exoplayer-2-framework

    Changements de plug-in SSAI :

    • Les flux de DVR en direct et en direct avec l'insertion d'annonces côté serveur (SSAI Live) sont désormais pris en charge.

    • Changement de rupture: La Event.PLAYHEAD_POSITION propriété dans les deux événements, EventType.PROGRESS et EventType.AD_PROGRESS , contient maintenant la position absolue de la tête de lecture, qui est la même position de tête de lecture donnée par le joueur sous-jacent.
      Veuillez utiliser à la Event.PROGRESS_BAR_PLAYHEAD_POSITION place pour obtenir la position relative de la tête de lecture.

    • le Event.PROGRESS_BAR_PLAYHEAD_POSITION la propriété a été introduite pour les deux EventType.PROGRESS et EventType.AD_PROGRESS événements qui contiennent la position relative de la tête de lecture (comme Event.PLAYHEAD_POSITION fait dans les anciennes versions).

    • Ajout de contrôles publicitaires pour Android TV.

    • Correction d'un problème selon lequel le bouton de rembobinage ne se rembobinait pas correctement après des clics consécutifs.

    Version 6.8.1

    31 mai 2019

    Modifications ExoPlayer2 :

    • ExoPlayer2 a été rétrogradé à la version 2.9.3 en raison d'un problème avec HLS Multiple Audio Pistes dans la version 2.9.6.

    Modifications de la lecture hors connexion :

    • Ajoutez la prise en charge du service d'authentification de lecture (PAS).

    • Les méthodes suivantes sont ajoutées au com.brightcove.player.edge.AbstractOfflineCatalog:

      • findOfflineVideoById(String, @NonNull HttpRequestConfig): Video
      • findOfflineVideoById(String, @NonNull HttpRequestConfig, @NonNull OfflineCallback<Video>): void
      • requestPurchaseLicense(@NonNull Video, @NonNull EventListener, @NonNull HttpRequestConfig): void
      • requestRentalLicense(@NonNull Video, @NonNull Date, long, @NonNull EventListener, @NonNull HttpRequestConfig): void

    • La méthode suivante est supprimée de la com.brightcove.player.edge.AbstractOfflineCatalog

      • renewLicense(@NonNull Video, @NonNull EventListener): void

    • Résout un problème de chargement des images d'affiche pour les vidéos téléchargées. Une fois disponible, une image locale sera chargée.

    Modifications du plugin FreeWheel :

    • Ajout de la prise en charge du contenu en direct avec les annonces pré-roll côté client

    Changements de plug-in SSAI :

    • Résout un problème dans lequel une source de contenu pouvait être chargée directement à partir de l' src attribut dans la réponse de l'API de lecture, au lieu de l' contenturi attribut dans le VMAP.

    Version 6.8.0

    1 mai 2019

    Modifications du SDK :

    • La version minimale du SDK Android prise en charge par le SDK natif Brightcove pour Android est maintenant 19.

    • Le SDK natif Brightcove pour Android est maintenant conforme à Java 8. Veuillez noter que les applications Android développées à l'aide de cette version du SDK Android peuvent nécessiter l'ajout de la directive suivante au fichier build.gradle pour assurer la compatibilité du bytecode :

      compileOptions{
targetCompatibility JavaVersion.VERSION_1_8
sourceCompatibility JavaVersion.VERSION_1_8
}

    Modifications ExoPlayer2 :

    • ExoPlayer2 a été mis à niveau vers la version 2.9.6

    • Veuillez noter que cette mise à niveau a les modifications d'API suivantes :

    • Briser la modification supprimée :constructor TrackSelectorHelper(@NonNull ExoPlayer player, @NonNull MappingTrackSelector trackSelector)

      Utiliser constructor TrackSelectorHelper(@NonNull ExoPlayer player, @NonNull DefaultTrackSelector trackSelector)

    • Briser la modification supprimée : setTrackSelector(@NonNull MappingTrackSelector trackSelector)

      Utiliser setTrackSelector(@NonNull DefaultTrackSelector trackSelector)

    • Supprimé getTrackSelector(): MappingTrackSelector

      Utiliser getTrackSelector(): DefaultTrackSelector

    Modifications du plugin FreeWheel :

    • Le plugin FreeWheel a été mis à jour pour prendre en charge FreeWheel AdManager version 6.28.0.

    • Veuillez noter que FreeWheel emballe désormais la bibliothèque AdManager sous forme d'Archive Android (AAR).

    Modifications du plugin IMA :

    • Le plugin IMA a été mis à jour pour prendre en charge Google IMA SDK version 3.11.2.

    • Cette version du SDK Google IMA prend en charge les normes Open Measurement.

    • Veuillez vous référer à la documentation du SDK Android Brightcove pour plus d'informations.

    • Correction d'un problème qui empêchait les annonces midroll appelées après de longues périodes de lecture de contenu.

    Changements de plug-in SSAI :

    • Le plugin SSAI prend désormais en charge la diffusion de contenu via le service d'autorisation de lecture Brightcove (PAS).

    • Veuillez noter que PAS fonctionne uniquement avec le plugin SSAI. PAS avec le plug-in OnCeux obsolète n'est pas pris en charge.

    Version 6.7.0

    5 avr. 2019

    Plugin SSAI :

    • Avec cette version, le SDK natif Brightcove pour Android implémente le plugin SSAI. Pour plus d'informations sur ce nouveau plugin, veuillez visiter la présentation du SDK Android Brightcove sur le site de support Brightcove.

    Modifications du plugin ONCeux :

    • Avec cette version, le plugin OnCeux est obsolète. Veuillez utiliser le nouveau plugin SSAI à la place. Pour plus d'informations, consultez la présentation du SDK Android Brightcove sur le site de support Brightcove.

    Version 6.6.0

    29 mars 2019

    Modifications du SDK :

    • Une nouvelle classe, HttpRequestConfig a été ajoutée. Cette classe prend en charge l'ajout de paramètres URL de requête et/ou d'en-têtes de requête qui peuvent être requis par les services Brightcove (par exemple, l'API Brightcove Edge ou le service d'autorisation de lecture Brightcove (PAS)). Veuillez vous référer à la documentation du support Brightcove pour plus d'informations sur l'utilisation de cette classe.

    • Les méthodes FindPlaylist suivantes sont obsolètes dans com.BrightCove.player.edge.Catalog : FindPlayListByID (String, Map < String >, PlayListListener) : void findPlayListById (String, Map < String, String >, Map<String, String >, PlayListListener) : void findPlayListByReferenceId (String, Map < String, String >, PlayListListener) : void findPlayListByReferenceId (String, Map < String, String >, Map<Chaîne, String >, PlayListListener) : void
