    Comprendre les événements avec le SDK natif pour Android

    Dans cette rubrique, vous découvrirez les événements les plus courants déclenchés et écoutés par le SDK natif pour Android.

    Introduction

    Le SDK natif pour Android utilise un modèle piloté par les événements, qui écoute les actions de l'utilisateur avec le lecteur vidéo.

    Modèle axé sur l'événement

    Dans les modèles de programmation traditionnels, le flux est statique et prédéfini par le programmeur. Dans un modèle piloté par un événement, il n'y a pas de flux prédéfini. Au lieu de cela, le flux est déterminé par des événements, tels que les actions de l'utilisateur. Il y a généralement une boucle principale qui écoute les événements et déclenche des fonctions de rappel.

    Événements

    Lorsque vous utilisez le SDK Brightcove Player pour Android, vous pouvez profiter des événements prédéfinis pour contrôler les fonctionnalités de votre application. Parce que les événements ne sont que des littéraux de chaîne, vous pouvez facilement créer et utiliser vos propres événements.

    Les événements du SDK Brightcove Player pour Android représentent un état ou une action qui peut être transmis par une chaîne d'écouteurs. Tous les événements ont un type et une carte de propriétés qui leur sont associés. Les événements peuvent être empêchés de se propager vers le bas de la chaîne d'écoute ou peuvent être informés d'empêcher leurs écouteurs par défaut dans les classes de lecture vidéo.

    Les événements peuvent représenter un état ou une action se produisant. Par exemple, le type d'événement DID_SEEK_TO représente l'état de l'action SEEK_TO .

    EventEmitter

    Le Brightcove BaseVideoView est conçu pour remplacer le VideoViewnatif d'Android. Cette classe démarre également le EventEmitter qui lie tous les composants du SDK ensemble. Lorsqu'un utilisateur interagit avec la vue vidéo, l' EventEmitter émet un événement, tel que la lecture, la pause, etc. Les composants qui s'intéressent à chaque événement peuvent les écouter et les prendre en compte.

    Des plugins peuvent être ajoutés qui écoutent et émettent des événements lorsqu'ils veulent que le SDK fasse certaines choses.

    Le SDK natif fournit des événements prédéfinis pour contrôler votre application, mais vous pouvez facilement créer et utiliser vos propres événements.

    Il est également possible de créer votre propre émetteur d'événements et de demander au BrightCoveExoPlayerVideoView de l'utiliser. Ceci est utile si vous voulez contrôler une chaîne d'écouteurs.

    Composants et événements

    Pour chaque composant du SDK Brightcove pour Android, vous pouvez trouver tous les événements qui lui sont associés. Pour ce faire, procédez comme suit :

    1. Accédez au kit de référence Brightcove Player SDK pour Android .
    2. Dans le menu de gauche, sélectionnez un composant.
    3. Dans la section Détails de la classe de composants, vous trouverez une liste d'événements qui sont émis et écoutés par ce composant.

    Événements publicitaires courants

    Dans votre application, vous pouvez écouter et agir sur un ensemble d'événements publicitaires à partir du SDK Brightcove Player pour Android.

    Événements publicitaires IMA

    Événements publicitaires courants

    Voici une liste des événements publicitaires courants du SDK Brightcove Player pour Android que vous pouvez écouter et agir :

    • AD_BREAK_DÉMARRÉ
    • AD_BREAK_COMPLETED
    • AD_DÉMARRÉ
    • AD_PAUSE
    • AD_REPRIS
    • AD_PROGRESSION
    • AD_COMPLETED
    • AD_ERROR

    Exemple de code

    Lorsqu'un saut d'annonce démarre, l' EventType.AD_BREAK_STARTED événement est émis. Lorsqu'un saut d'annonce se termine, le EventType.AD_BREAK_COMPLETED est émis.

    Vous pouvez écouter et réagir aux événements comme indiqué ici :

    eventEmitter.on(EventType.AD_BREAK_STARTED, new EventListener() {
	@Override
	public void processEvent(Event event) {
		// React to the event
	}
});

eventEmitter.on(EventType.AD_BREAK_COMPLETED, new EventListener() {
	@Override
	public void processEvent(Event event) {
		// React to the event
	}
});

    Vous pouvez émettre des événements en procédant comme suit :

    eventEmitter.emit(EventType.STOP);
