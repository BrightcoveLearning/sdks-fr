Contenu de la page

    Étape par étape : Application vidéo simple utilisant le SDK natif pour Android

    Dans cette rubrique, vous allez utiliser Android Studio et le SDK Brightcove Player pour Android pour créer une application qui lit une vidéo à partir d'un compte Brightcove Video Cloud, ainsi que des vidéos d'une playlist Brightcove Video Cloud.

    Présentation

    Une fois cette brève introduction terminée, vous pourrez effectuer les tâches suivantes :

    • Créez un projet et ajoutez le SDK Brightcove Player pour Android à l'aide de Gradle.
    • Modifiez la mise en page pour inclure un BrightcoveExoPlayerVideoView.
    • Ajouter à la onCreate() méthode pour lire des vidéos. provenant de sources différentes.
    Remarques :
    • Cet exemple utilise le BrightcoveExoPlayerVideoView, qui est la vue recommandée à utiliser dans la plupart des cas. Si vous pensez que vous devez utiliser une vue différente, consultez la section Choix d'un document BaseVideoView .
    • Cet exemple utilise l' API de lecture Brightcove. Il s'agit de l'API la plus récente et recommandée pour récupérer du contenu de votre bibliothèque Video Cloud.

    Audience

    Les développeurs qui utilisent Android Studio pour le développement et qui sont intéressés par l'utilisation du SDK Brightcove Player pour Android dans une application Android.

    Conditions préalables

    Une connaissance minimale du développement d'applications Java et Android.

    Préparez-vous

    Préparez-vous pour le développement en installant Android Studio

    1. Téléchargez l'application Android Studio .
    2. Suivez les instructions sur les pages pour installer l'application.

    Créer un projet

    Créez un projet dans Android Studio, puis liez le SDK du lecteur à l'aide de Gradle.

    Créer un projet dans Android Studio

    1. Ouvrez Android Studio.
      • Si c'est la première fois que vous ouvrez Android Studio, sélectionnez Démarrer un nouveau projet Android Studio.
      • Si vous voyez les menus, sélectionnez Fichier > > Nouveau projet.
    2. Dans la fenêtre Choisir votre projet , sélectionnez Activité vide et cliquez sur Suivant.

    3. Fournissez des valeurs pour le nom, le nom du package, la langue et l'emplacement Enregistrer. Dans ce démarrage rapide, les valeurs indiquées ci-dessous sont utilisées :

    4. Cliquez sur Finish (Terminer).

    5. Android Studio fonctionnera pendant un certain temps et finira par afficher l'état initial du projet.

    Utilisez Gradle pour créer un lien vers le SDK Brightcove Player pour Android

    Gradle étant intégré à Android Studio, vous l'utiliserez pour ajouter le SDK natif pour Android à votre projet.

    1. Développez la section Gradle Scripts et ouvrez le deuxième build.gradle fichier, qui est associé au module actuel de votre projet PlayVideos.

    2. Dans le build.gradle fichier, procédez comme suit :

      1. Dans la android section, ajoutez la compileOptions valeur suivante :
        Pour ExoPlayer v2.9.0+, vous devez activer la prise en charge de Java 8. Pour ce faire, ajoutez ce qui suit à la android section du fichier build.gradle de votre application. Pour plus de détails, consultez le document ExoPlayer d'Android .
        android {
    compileSdkVersion 28
    defaultConfig {
        applicationId "com.brightcove.playvideos1"
        minSdkVersion 21
        targetSdkVersion 28
        versionCode 1
        versionName "1.0"
        testInstrumentationRunner "androidx.test.runner.AndroidJUnitRunner"
    }
    buildTypes {
        release {
            minifyEnabled false
            proguardFiles getDefaultProguardFile('proguard-android-optimize.txt'), 'proguard-rules.pro'
        }
    }
    compileOptions {
        targetCompatibility JavaVersion.VERSION_1_8
    }
}
      2. Sous la android section, ajoutez une repositories section pour le dépôt Brightcove Maven. 
        repositories {
      maven {
          url 'https://repo.brightcove.com/releases'
      }
    }
      3. Dans la dependencies section, incluez la dernière version du SDK natif pour Android. Remplacez la valeur 6.10.0 ci-dessous par la dernière version du SDK, qui se trouve dans la présentation : Brightcove Native SDK pour Android document. 
        dependencies {
  implementation fileTree(dir: 'libs', include: ['*.jar'])
  implementation 'com.android.support:appcompat-v7:26.1.0'
  implementation 'com.android.support.constraint:constraint-layout:1.0.2'
  testImplementation 'junit:junit:4.12'
  androidTestImplementation 'com.android.support.test:runner:1.0.1'
  androidTestImplementation 'com.android.support.test.espresso:espresso-core:3.0.1'
  implementation "com.brightcove.player:exoplayer2:6.10.0"
}
       

      L'utilisation d'une version de dépendance dynamique avec le caractère '+' est déconseillée dans la communauté Gradle. L'utilisation du versioning dynamique présente un risque important pour votre processus de génération, car les API plus récentes peuvent devenir incompatibles de façon inattendue avec le code source de votre application.

      Remarques :
      • Brightcove recommande toujours d'utiliser la dernière version avec une chaîne de version spécifique. Par exemple, passez en revue les fichiers build.gradle et gradle.properties dans le projet android-player-samples .
      • La dépendance ExoPlayer inclut le SDK natif pour Android, de sorte que vous n'avez pas à inclure le SDK séparément.
      • Google ExoPlayer est un lecteur multimédia open source, au niveau de l'application pour Android. Notez que nous utilisons le framework ExoPlayer 2. Pour plus d'informations, consultez le document Migration vers ExoPlayer 2 Framework .
    3. Synchronisez le projet pour récupérer les modifications du fichier Gradle.

    Codez l'application

    Ensuite, vous allez écrire le code pour mettre en page l'application, créer la liste des vidéos et lire les vidéos

    Définir la mise en page de l'application

    Même avec l'activité vide, vous obtiendrez une TextView mise en page simple. Remplacez ceci par le BrightcoveExoPlayerVideoView, qui définit la vue pour le joueur.

    1. Ouvrez le fichier app/res/layout/activity_main.xml.

    2. Cliquez sur Texte au bas de l'onglet du fichier pour afficher le code XML brut.

    3. Supprimez l' TextView élément existant et ajoutez une BrightcoveExoPlayerVideoView section afin que le XML résultant apparaisse comme suit. Remarque pour une utilisation ultérieure id de la vue est nommée brightcove_video_view.

      <?xml version="1.0" encoding="utf-8"?>
    <android.support.constraint.ConstraintLayout xmlns:android="http://schemas.android.com/apk/res/android"
        xmlns:app="http://schemas.android.com/apk/res-auto"
        xmlns:tools="http://schemas.android.com/tools"
        android:layout_width="match_parent"
        android:layout_height="match_parent"
        tools:context="com.brightcove.playvideos.MainActivity">

    <com.brightcove.player.view.BrightcoveExoPlayerVideoView
        android:id="@+id/brightcove_video_view"
        android:layout_width="match_parent"
        android:layout_height="280dp"
        android:layout_gravity="center_horizontal|top"/>
    </android.support.constraint.ConstraintLayout>

    Activer l'application pour utiliser Internet

    1. Ouvrez le app/manifests/AndroidManifest.xml fichier.

    2. Juste après le bloc de <application> code, mais au-dessus de la </manifest> balise de fin, insérez ce qui suit pour activer l'accès à Internet.

      <uses-permission android:name="android.permission.INTERNET"/>
    3. Synchronisez le projet pour le reconstruire et récupérez ces modifications.

    Créer la vue et lire une vidéo

    1. Retournez au MainActivity.java fichier. Pour la MainActivity classe, étendez la BrightcovePlayer classe. Cela permet de gérer le cycle de vie par défaut de votre application.

      public class MainActivity extends BrightcovePlayer {

    2. La BrightcovePlayer classe n'est pas importée automatiquement dans le projet. Appuyez sur l' option + touches de retour pour ajouter l' import instruction pour la classe.

      Avant :

      Après :

    3. Localisez la onCreate() fonction. En dessous de n'importe quel code existant dans la fonction, ajoutez ce qui suit :

      • Créez une instance de BrightcoveExoPlayerVideoView et associez-la à la mise en page. Dans la plupart des cas, vous utiliserez la vue exoplayer. Pour plus d'informations, consultez le document Choix d'une vue vidéo .

      public class MainActivity extends BrightcovePlayer {
  @Override
  protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.activity_main);

      N'oubliez pas d'importer la BrightcoveExoPlayerVideoView classe.

    4. Ensuite, vous devez envoyer votre ID de compte de joueur Brightcove à Brightcove en utilisant la setAccount() méthode d'analyse.

      Analytics analytics = brightcoveVideoView.getAnalytics();
analytics.setAccount("your account Id");

Facultatif : Si vous remplacez la BrightcoveExoPlayerVideoView classe ou si vous n'utilisez pas le lecteur et le catalogue Brightcove, vous devez envoyer votre ID de publication Video Cloud à Video Cloud Analytics. Vous pouvez le faire en utilisant la setAccount() méthode d'analyse. Cela vous permet d'afficher les données de cette application dans Video Cloud Analytics.

      Analytics analytics = brightcoveVideoView.getAnalytics();
analytics.setAccount("your account Id");

    6. Créez un objet vidéo à partir de votre vidéo hébergée sur un serveur distant. Définissez le DeliveryType pour qu'il corresponde au type de vidéo que vous avez.

      Video video = Video.createVideo("https://learning-services-media.brightcove.com/videos/hls/greatblueheron/greatblueheron.m3u8",
  DeliveryType.HLS);

    7. Chargez une image distante à utiliser comme image d'affiche avant le démarrage de la lecture vidéo.

      try {
  java.net.URI myposterImage = new java.net.URI("https://solutions.brightcove.com/bcls/assets/images/Great-Blue-Heron.png");
  video.getProperties().put(Video.Fields.STILL_IMAGE_URI, myposterImage);
} catch (URISyntaxException e) {
  e.printStackTrace();
}

    8. Ajoutez la vidéo à la vue et commencez la lecture vidéo.

      brightcoveVideoView.add(video);
brightcoveVideoView.start();

    9. Vérifiez que votre onCreate() méthode apparaît comme suit :

      @Override
  protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.activity_main);
    brightcoveVideoView = (BrightcoveExoPlayerVideoView) findViewById(R.id.brightcove_video_view);

    // Optional: For Brightcove Player customers to register their apps
    Analytics analytics = brightcoveVideoView.getAnalytics();
    analytics.setAccount("your account Id");

    // Define a video from a remote server
    Video video = Video.createVideo("https://learning-services-media.brightcove.com/videos/hls/greatblueheron/greatblueheron.m3u8", DeliveryType.HLS);

    // Load a remote poster image
    try {
        java.net.URI myposterImage = new java.net.URI("https://solutions.brightcove.com/bcls/assets/images/Great-Blue-Heron.png");
        video.getProperties().put(Video.Fields.STILL_IMAGE_URI, myposterImage);
    } catch (URISyntaxException e) {
        e.printStackTrace();
    }

    // Add video to the view
    brightcoveVideoView.add(video);
    // Start video playback
    brightcoveVideoView.start();
  }

    10. Exécutez ou déboguez l'application pour voir la vidéo en lecture.

      Lorsque vous exécutez l'application sur un appareil, assurez-vous qu'elle est connectée à Internet via votre forfait de données cellulaires ou wifi.

    Obtenir et lire une vidéo

    Dans cette section, vous allez utiliser la Catalog classe pour récupérer une seule vidéo du serveur Video Cloud, puis la lire.

    La com.brightcove.player.edge.Catalog classe fournit des méthodes asynchrones pour récupérer des vidéos et des listes de lecture à partir de l' API Brightcove Playback. Il s'agit de l'API la plus récente et recommandée pour récupérer du contenu de votre bibliothèque Video Cloud.

    Supprimer le code inutile

    1. Certains codes de l'application précédente n'est plus nécessaire. Dans la onCreate() méthode, supprimez tout le code après avoir instancié le brightcoveVideoView .

    2. Confirmez que votre onCreate() méthode apparaît comme suit :

      public class MainActivity extends BrightcovePlayer {

  @Override
  protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.activity_main);
    brightcoveVideoView = (BrightcoveExoPlayerVideoView) findViewById(R.id.brightcove_video_view);
  }
}

    Récupérer une vidéo à partir du catalogue

    1. À partir de Video Cloud Studio, collectez les informations suivantes :
      • ID de compte
      • ID de la vidéo
      • Clé de politique
       
      Remarques :
      • Pour accéder à l'API de lecture, vous aurez besoin d'une clé de stratégie. Si vous n'êtes pas familier avec les clés de stratégie, reportez-vous à la section Présentation de l'API de stratégie.
      • Étant donné que les kits SDK natifs n'envoient pas d'en-tête d'origine, vous ne pouvez pas utiliser une clé de stratégie restreinte au domaine. Si vous essayez cette option, vous obtiendrez une 403 Forbidden réponse lorsque vous tentez d'extraire du contenu de votre bibliothèque Video Cloud.

    2. Définissez vos valeurs personnalisées dans votre projet. Ouvrez le res/values/strings.xml fichier et mettez-le à jour comme suit :

      <?xml version="1.0" encoding="utf-8"?>
  <resources>

      <!-- Application name -->
      <string name="app_name">PlayVideos</string>

      <!-- A sample Brightcove Edge Account ID -->
      <string name="account">your account id</string>

      <!-- A sample Brightcove Edge Policy Key -->
      <string name="policy">your policy key</string>

      <!-- A sample Brightcove Video ID -->
      <string name="videoId">your video id</string>

  </resources>

    3. Retournez au MainActivity.java fichier et ajoutez le code suivant :

      • Obtenir l'émetteur d'événements à partir du SDK

      • Créez une instance de com.brightcove.player.edge.Catalog, en utilisant les valeurs définies pour l'ID de compte et la clé de stratégie de l'étape précédente.

      // Get the event emitter from the SDK and create a catalog request to fetch a video from the
// Brightcove Edge service, given a video id, an account id and a policy key.
EventEmitter eventEmitter = brightcoveVideoView.getEventEmitter();
Catalog catalog = new Catalog(eventEmitter, getString(R.string.account), getString(R.string.policy));

    4. Utilisez la findVideoByID() méthode du catalogue avec votre ID vidéo et un VideoListener pour le rappel.

      catalog.findVideoByID(getString(R.string.videoId), new VideoListener() {

  // Add the video found to the queue with add().
  // Start playback of the video with start().
  @Override
    public void onVideo(Video video) {
    }

  @Override
    public void onError(String s) {
    }
});
      Le VideoListener identifie les objets qui s'attendent à recevoir un objet Video à partir d'une opération asynchrone. Sur une opération réussie, une onVideo() méthode sera appelée. S'il y a une erreur dans l'opération, une onError() méthode sera appelée. Android Studio a inséré les signatures des deux méthodes nécessaires.

    5. Dans la onVideo() méthode, ajoutez la vidéo à brightcoveVideoView , puis démarrez la vidéo.

      // Start playback of the video with start().
  @Override
    public void onVideo(Video video) {
      brightcoveVideoView.add(video);
      brightcoveVideoView.start();
    }

    6. Dans la onError() méthode, lancez la chaîne d'erreur.

      @Override
    public void onError(String s) {
      throw new RuntimeException(s);
    }

    7. Le code complet de votre MainActivity classe devrait être similaire à ceci :

      public class MainActivity extends BrightcovePlayer {

  @Override
  protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.activity_main);
    brightcoveVideoView = (BrightcoveExoPlayerVideoView) findViewById(R.id.brightcove_video_view);

    EventEmitter eventEmitter = brightcoveVideoView.getEventEmitter();
    Catalog catalog = new Catalog(eventEmitter, getString(R.string.account), getString(R.string.policy));
    catalog.findVideoByID(getString(R.string.videoId), new VideoListener() {

      // Add the video found to the queue with add().
      // Start playback of the video with start().
      @Override
      public void onVideo(Video video) {
        brightcoveVideoView.add(video);
        brightcoveVideoView.start();
      }

      @Override
      public void onError(String s) {
        throw new RuntimeException(s);
      }
    });
  }
}
    8. Exécutez l'application pour confirmer la lecture de la vidéo.

    Obtenir et lire une liste de lecture

    Dans cette section, vous utiliserez la Catalog classe pour récupérer une playlist à partir du serveur Video Cloud, puis lire les vidéos dans la liste de lecture.

    La com.brightcove.player.edge.Catalog classe fournit des méthodes asynchrones pour récupérer des vidéos et des listes de lecture à partir de l' API Brightcove Playback. Il s'agit de l'API la plus récente et recommandée pour récupérer du contenu de votre bibliothèque Video Cloud.

    Supprimer le code inutile

    1. Certains codes de l'application précédente n'est plus nécessaire. Supprimez l'appel à la findVideoByID() méthode du catalogue et la fonction de rappel VideoListener anonyme associée.

      catalog.findVideoByID(getString(R.string.videoId), new VideoListener() {

    // Add the video found to the queue with add().
    // Start playback of the video with start().
    @Override
      public void onVideo(Video video) {
        brightcoveVideoView.add(video);
        brightcoveVideoView.start();
      }

    @Override
      public void onError(String s) {
        throw new RuntimeException(s);
      }
  });

    Récupérer une liste de lecture à partir du catalogue

    1. L'instance de catalogue existante fonctionnera pour récupérer une liste de lecture, donc aucune modification n'est nécessaire pour ces lignes de code :

      EventEmitter eventEmitter = brightcoveVideoView.getEventEmitter();
Catalog catalog = new Catalog(eventEmitter, getString(R.string.account), getString(R.string.policy));
    2. Dans le module Media de Video Cloud Studio, sélectionnez une liste de lecture et copiez l' ID de playlist.

    3. Utilisez la findPlaylistByID() méthode du catalogue en utilisant votre ID de liste de lecture et un PlaylistListener pour le rappel.

      catalog.findPlaylistByID("2764931905001", new PlaylistListener() {
  @Override
  public void onPlaylist(Playlist playlist) {
  }

  @Override
  public void onError(String s) {
  }
});
      Le PlaylistListener identifie les objets qui s'attendent à recevoir un objet Playlist à partir d'une opération asynchrone. Sur une opération réussie, une onPlaylist() méthode sera appelée. S'il y a une erreur dans l'opération, une onError() méthode sera appelée. Android Studio a inséré les signatures des deux méthodes nécessaires.

    4. Dans la onPlaylist() méthode, récupérez les vidéos de la liste de lecture, ajoutez toutes les vidéos à brightcoveVideoView , puis démarrez la première vidéo.

      @Override
    public void onPlaylist(Playlist playlist) {
      brightcoveVideoView.addAll(playlist.getVideos());
      brightcoveVideoView.start();
    }

    5. Dans la onError() méthode, lancez la chaîne d'erreur.

       @Override
    public void onError(String s) {
      throw new RuntimeException(s);
    }

    6. Le code complet de votre MainActivity classe devrait être similaire à ceci :

      public class MainActivity extends BrightcovePlayer {

  @Override
  protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.activity_main);
    brightcoveVideoView = (BrightcoveExoPlayerVideoView) findViewById(R.id.brightcove_video_view);

    EventEmitter eventEmitter = brightcoveVideoView.getEventEmitter();
    Catalog catalog = new Catalog(eventEmitter, getString(R.string.account), getString(R.string.policy));
    catalog.findPlaylistByID("4845949311001", new PlaylistListener() {
      @Override
      public void onPlaylist(Playlist playlist) {
        brightcoveVideoView.addAll(playlist.getVideos());
        brightcoveVideoView.start();
      }

      @Override
      public void onError(String s) {
        throw new RuntimeException(s);
      }
    });
  }
}
    7. Exécutez l'application pour confirmer plusieurs vidéos de la lecture de la liste de lecture.
