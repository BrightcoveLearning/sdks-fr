    Redimensionnement de la vue vidéo avec le SDK natif pour Android

    Dans cette rubrique, vous allez apprendre à redimensionner/étirer la vue vidéo interne pour remplir le BrightcoveVideoView lorsque vous travaillez avec le SDK natif pour Android.

    Présentation

    En tant que développeur, vous pouvez modifier le comportement par défaut pour l'affichage de la vidéo dans la vue vidéo. Vous pouvez redimensionner/étirer la vue vidéo interne (SurfaceView/TextureView) pour remplir le BrightcoveVideoView(FrameLayout). Pour ce faire, vous pouvez écouter l' EventType.VIDEO_SIZE_KNOWN événement, faire des calculs et définir les valeurs d'affichage.

    Il existe deux façons de modifier la vue vidéo :

    • Redimensionnement du conteneur (BaseVideoView/FrameLayout)
    • Redimensionnement de la vidéo (SurfaceView)

    Redimensionnement du conteneur

    Vous pouvez développer une application qui définit la vue vidéo pour qu'elle s'adapte à l'ensemble de l'écran de l'appareil en mode portrait. Vous pouvez le faire en redimensionnant le conteneur (BaseVideoView/FrameLayout).

    Étant donné que la Video hauteur est initialement inconnue, elle correspondra à sa taille de vue parent (l'écran entier dans BasicSampleApp ) jusqu'à ce que l' VIDEO_SIZE_KNOWN événement soit déclenché. Ensuite, le BaseVideoView va envelopper le SurfaceView. Lors du redimensionnement du conteneur, vous pouvez voir un scintillement initial.

    Définir la propriété du conteneur

    Si vous voyez des marges noires autour de votre vidéo, vous pouvez essayer de régler le BrightcoveVideoView layout-height.

    1. Ouvrez le fichier de mise en page XML.
    2. Réglez la valeur BrightcoveVideoView layout-height à wrap_content.
    3. Vous pouvez rencontrer un scintillement initial comme expliqué dans la note ci-dessus.

    Définir la taille de la vue vidéo

    Une autre façon de contrôler la vue vidéo consiste à définir la taille par programme.

    1. Pour minimiser le scintillement initial, essayez de layout_height définir la sur une valeur spécifique (par exemple 280dp). L'objectif est que la hauteur définie soit proche de la hauteur calculée.
    2. Écoutez l' EventType.VIDEO_SIZE_KNOWN événement.
    3. Obtenir le Event.VIDEO_WIDTH et Event.VIDEO_HEIGHT
    4. Avec ces valeurs, déterminez le rapport d'aspect.
    5. Utilisez le rapport d'aspect pour calculer la largeur et la hauteur sur l'écran.

    Cette solution ressemblera à ceci :

    //Event sent when we know the Video dimensions.
eventEmitter.on(EventType.VIDEO_SIZE_KNOWN, new EventListener() {
  @Override
  public void processEvent(Event event) {
    // Get the width and height
    float width = event.getIntegerProperty(Event.VIDEO_WIDTH);
    float height = event.getIntegerProperty(Event.VIDEO_HEIGHT);
    float aspectRatio = height/width;

    //Get the display metrics
    DisplayMetrics metrics = new DisplayMetrics();
    getWindowManager().getDefaultDisplay().getMetrics(metrics);
    int videoWidth = metrics.widthPixels; // We cover the entire display's width
    int videoHeight = (int) (videoWidth*aspectRatio); //Calculate the height based on the ratio

    // Set the layout params
    brightcoveVideoView.setLayoutParams(new FrameLayout.LayoutParams(videoWidth,videoHeight));
  }
});

    Lors de l'ajout d'une vue, vous devez remplacer les paramètres de mise en page. Pour plus de détails, consultez l' Android : Ajout d'un document Affichage/Gestion de la mise en page

    Redimensionnement de la vidéo

    Vous pouvez implémenter un double-clic qui remplit l'écran. Cela prendrait une vidéo 16:9 et remplissez l'écran sur un appareil 18:9 ou 19:9.

    Vous pouvez le faire en redimensionnant la vidéo réelle (SurfaceView).

    1. Si vous définissez la largeur et la hauteur pour correspondre renderView à la taille de l'écran de l'appareil, la vidéo sera redimensionnée pour correspondre à l'ensemble de l'écran.

      Cela n'étire pas la vidéo réelle. Nous le ferons à l'étape suivante.
      //Event sent when we know the Video dimensions.
eventEmitter.on(EventType.VIDEO_SIZE_KNOWN, new EventListener() {
  @Override
  public void processEvent(Event event) {
    // Get the width and height
    float width = event.getIntegerProperty(Event.VIDEO_WIDTH);
    float height = event.getIntegerProperty(Event.VIDEO_HEIGHT);
    float aspectRatio = height/width;

    //Get the display metrics
    DisplayMetrics metrics = new DisplayMetrics();
    getWindowManager().getDefaultDisplay().getMetrics(metrics);
    int videoWidth = metrics.widthPixels; // We cover the entire display's width
    int videoHeight = (int) (videoWidth*aspectRatio); //Calculate the height based on the ratio

    // Set the video size
    brightcoveVideoView.getRenderView().setVideoSize(videoWidth, videoHeight);
  }
});

    2. Le code ci-dessous met à jour la vue vidéo sous-jacente (par exemple SurfaceView) pour qu'elle corresponde à son parent BaseVideoView(FrameLayout). Il y a plusieurs choses à garder à l'esprit :

      • Vous devez écouter l' VIDEO_SIZE_KNOWN événement lors de l'entrée et de la sortie du plein écran, et lors de la rotation de l'appareil.
      • Vous devez décider quand mettre à jour votre vue pour les conditions ci-dessus.

      Un scénario peut être que vous définissez le BaseVideoView layout_height à, 280dp puis redimensionnez le SurfaceView pour correspondre 280dp. Le rapport d'aspect sera légèrement modifié. Si un utilisateur passe au plein écran en mode portrait et updateVideoViewMatchToParent() est appelé, alors le BaseVideoView et SurfaceView remplira l'écran de l'appareil et le rapport d'aspect vidéo sera légèrement modifié.

      public void updateVideoViewMatchToParent(@NonNull BaseVideoView brightcoveVideoView) {
  RenderView renderView = brightcoveVideoView.getRenderView();
  if (renderView != null) {
      renderView.setVideoSize(brightcoveVideoView.getWidth(), brightcoveVideoView.getHeight());
  }
}

      Pour détecter le double robinet, vous pouvez utiliser l'interface GestureDetector.OnDoubleTAListener . Pour plus d'informations, consultez le document Détecter les gestes communs .

    Si vous voulez étirer la vidéo sans prendre en compte le rapport d'aspect (sans recadrer la vidéo), vous pouvez essayer ceci :

    eventEmitter.on(EventType.VIDEO_SIZE_KNOWN, new EventListener() {
  @Override
  public void processEvent(Event event) {
      //Get the display metrics
      DisplayMetrics metrics = new DisplayMetrics();
      getWindowManager().getDefaultDisplay().getMetrics(metrics);

      int videoHeight = metrics.heightPixels;
      int videoWidth = metrics.widthPixels;
      brightcoveVideoView.getRenderView().setVideoSize(videoWidth, videoHeight);
  }
});
    Il est important que le BaseVideoView les usages match_parent pour les deux layout_width et layout_height pour le SurfaceView pour remplir l'écran de l'appareil.

    Gardez à l'esprit que les valeurs que vous obtenez sont affectées par l'orientation du périphérique. Pour étirer la vidéo pour le portrait et le paysage, vous devez effectuer les opérations suivantes :

    1. Remplacer ce qui suit Activity: 

      public void onConfigurationChanged(Configuration configuration)
    2. Réinitialiser la taille de la vidéo.
