Aperçu

Le SDK natif pour tvOS vous permet de créer des applications centrées sur la vidéo pour l'Apple TV.

  • Les clients Video Cloud peuvent lire le contenu de la bibliothèque Video Cloud
  • Les clients Brightcove Player peuvent lire des vidéos avec des URL accessibles sur Internet

Lors de l'ingestion de votre contenu, nous vous recommandons d'utiliser un profil haute résolution pour les meilleurs rendus pour Apple TV. Pour plus de détails, voir Apple Formats vidéo pour Apple TV Documentation.

Versions prises en charge

Soutien actif

Brightcove fournit un support actif pour le dernier SDK tvOS sur la dernière version publique des versions tvOS suivantes :

  • tvOS 14
  • tvOS 13.4.8
  • tvOS 12.4.1

Assistance passive

Brightcove fournit une prise en charge passive des versions de tvOS suivantes :

  • tvOS 11.4.1

Comprendre les niveaux d'assistance

Le soutien est défini en termes de [version majeure].[version mineure].[version corrigée] d'un OS (par exemple : tvOS 11.4.1). Afin de profiter des dernières améliorations technologiques d'Apple, nous proposons deux niveaux de support :

  • actif: Brightcove testera et corrigera les bogues sur ces versions du système d'exploitation.
  • Passif: Brightcove effectuera des tests sur ces versions du système d'exploitation, mais les corrections de bogues seront traitées au cas par cas.

En règle générale, Brightcove fournit une prise en charge active des versions du système d'exploitation utilisées par 80 % de la base d'utilisateurs et une prise en charge passive des autres versions utilisées par au moins 5 % de la base d'utilisateurs.

Commencer

Avant de commencer le développement avec le Player SDK pour tvOS, vous devez connaître quelques concepts :

  1. Médias

    La version actuelle du Native Player SDK prend en charge la lecture de contenu à partir de votre compte Video Cloud via l'API de lecture, ou en tant qu'actifs distants avec des URL accessibles sur Internet.

  2. Analytique

    Avec le kit de développement logiciel Native Player, les clients de Video Cloud obtiennent automatiquement des données d'analyse robustes rapportées dans Video Cloud.

  3. SDK principal et dépendances

    Le Brightcove Player SDK pour iOS et ses plugins peuvent être ajoutés à votre projet en utilisant le CocoaPods gestionnaire de dépendances. Les podspecs de tous les composants du Brightcove Player SDK sont hébergés dans le Référentiel BrightcoveSpecs.

    CocoaPods version 1.0 ou plus récente est requis.
  4. Référence du SDK natif

    Pour vous aider à comprendre le SDK, vous pouvez trouver une ventilation détaillée de l'architecture dans le SDK du lecteur. Découvrez comment les composants critiques interagissent pour gérer les complexités du contrôle du lecteur vidéo natif. Installation du SDK du lecteur, démarrage rapide et architecture

    Ici, vous trouverez également la dernière version bêta du SDK, s'il en existe une.

  5. Questions et réponses

    Rejoignez-nous dans le Forum des développeurs de SDK où les questions liées au SDK sont répondues par les ingénieurs de Brightcove et la communauté. Recherchez les questions auxquelles vous avez déjà répondu ou postez-en une nouvelle. En tant que groupe public, vous pouvez y accéder en fournissant votre adresse e-mail.

Remarques:

  • Pour améliorer la conformité CocoaPods, le SDK natif pour iOS 6.7.7 a renommé certains fichiers Podspec. Pour plus de détails, consultez le Référence du SDK natif.
  • La cible de déploiement minimale pour tous les frameworks Brightcove Native Player pour iOS est désormais iOS 11.0 et tvOS 11.0.
  • Le Brightcove Native SDK pour iOS 6.5.0+ nécessite Xcode 11.0 ou supérieur.
  • Le comptage automatique de références (ARC) est requis dans votre projet. C'est la valeur par défaut actuelle pour les projets Xcode.
  • Le Brightcove Native SDK pour tvOS est écrit pour vous permettre de créer des applications personnalisées à l'aide d'Obj-C et de Swift, mais il ne prend pas en charge le Television Markup Language (TVML) d'Apple.
  • Un problème Apple connu avec le contenu en streaming FairPlay est qu'il ne sera pas rendu en mode de mise en miroir AirPlay. Pour plus de détails, consultez le Présentation de la WWDC : Protection du contenu pour HLS (faites défiler environ à mi-chemin de la transcription).

Intégrations

Le tableau suivant contient des liens pour intégrer des plug-ins à votre implémentation SDK. Les plugins peuvent être téléchargés à partir de leurs dépôts Github respectifs, et chaque dépôt contient un README détaillé avec des instructions sur la façon de configurer le plugin.

Les exemples d'applications illustrent les fonctionnalités de base de chaque plug-in intégré au SDK. Ces exemples d'applications sont un bon moyen d'en savoir plus sur chaque plugin et servent de point de référence lors du débogage.

La version actuelle du SDK natif pour iOS inclut tous les composants avec le même numéro de version. Lors de l'intégration de composants, le SDK et les numéros de version des composants doivent correspondre. Par exemple, lorsque vous utilisez le composant de plug-in IMA 6.7.0, vous devez utiliser le kit de développement logiciel natif 6.7.0.
