    Présentation : Kit de développement logiciel (SDK) Brightcove Player pour TVOS

    Bienvenue dans la documentation du développeur de Brightcove Player SDK pour TVOS. Vous trouverez ici des informations sur la lecture de contenu vidéo sur Apple TV.

    Présentation

    Le SDK natif pour TVOS vous permet de créer des applications centrées sur la vidéo pour l'Apple TV.

    • Les clients Video Cloud peuvent lire le contenu de la bibliothèque Video Cloud
    • Les clients de Brightcove Player peuvent lire des vidéos avec des URL accessibles à Internet

    Lors de l'ingestion de votre contenu, nous vous recommandons d'utiliser un profil haute résolution pour obtenir les meilleurs formats associés pour Apple TV. Pour plus de détails, consultez la documentation Formats vidéo d'Apple pour Apple TV .

    Versions supportées

    Le SDK Native Player pour tvOS prend en charge les versions suivantes de tvOS :

    Prise en charge active

    Brightcove fournit une prise en charge active du dernier SDK iOS sur les versions suivantes de tvOS :

    • TVOS 14
    • TVO 13
    • TVO 12.4.1

    Prise en charge passive

    Brightcove prend en charge passivement les versions de TVOS suivantes :

    • TVO 11.4.1

    Comprendre les niveaux de support

    Le support est défini en termes de [version majeure].[ version mineure].[ version correctif] d'un système d'exploitation (par exemple : TVOS 11.4.1). Afin de tirer parti des dernières améliorations technologiques d'Apple, nous offrons deux niveaux de support :

    • Actif: Brightcove testera et corrigera les bogues sur ces versions du système d'exploitation.
    • Passif: Brightcove effectuera des tests sur ces versions du système d'exploitation, mais les corrections de bogues seront traitées au cas par cas.

    De manière générale, Brightcove fournira une prise en charge active pour les versions du système d'exploitation qui sont utilisées par 80 % de la base d'utilisateurs, et une prise en charge passive pour les autres versions, utilisées par au moins 5 % de la base d'utilisateurs.

    Premiers pas

    Avant de commencer le développement avec le SDK Player pour TVOS, il y a quelques concepts que vous devriez apprendre :

    1. Médias

      La version actuelle du Kit de développement logiciel (SDK) Native Player prend en charge la lecture de contenu depuis votre compte Video Cloud via l'API de lecture ou en tant que ressources distantes avec des URL accessibles à Internet.

    2. Analytique

      Avec le SDK Native Player, les clients Video Cloud obtiennent automatiquement des données analytiques robustes signalées dans Video Cloud.

    3. SDK principal et dépendances

      Le SDK Brightcove Player pour iOS et ses plugins peuvent être ajoutés à votre projet en utilisant le gestionnaire de dépendances CocoaPods . Les podspecs de tous les composants du SDK Brightcove Player sont hébergés dans le référentiel BrightCoveSpecs.

      CocoaPods version 1.0 ou ultérieure est requise.
    4. Référence du SDK natif

      Pour vous aider à comprendre le SDK, vous trouverez une ventilation détaillée de l'architecture dans le SDK Player. Découvrez comment les composants critiques interagissent pour gérer les complexités du contrôle du lecteur vidéo natif. Installation, démarrage rapide et architecture du Kit de développement logiciel (SDK)

      Ici, vous trouverez également la dernière version bêta du SDK, s'il en existe une.

    5. Questions et réponses

      Rejoignez-nous sur le forum des développeurs SDK où les ingénieurs de Brightcove et la communauté répondent aux questions relatives au SDK. Recherchez des questions déjà répondues ou publiez une nouvelle. En tant que groupe public, vous pouvez y accéder en fournissant votre adresse e-mail.

    Téléchargement du SDK

    Le SDK Brightcove Player pour tvOS est contenu dans le kit SDK Player pour iOS. Ici, vous trouverez un framework de bibliothèque dynamique qui prend en charge les tvOS.

    Obtenez la dernière version du SDK natif Brightcove pour iOS :

    Pour afficher les mises à jour de produits pour tous les kits SDK natifs et les plugins associés, consultez les notes de version du document Brightcove Native SDK .

    Le composant Sidecar Subtitles prend en charge la lecture sur AirPlay avec des légendes. Pour plus d'informations sur la synchronisation de vos sous-titres, consultez le document Synchronisation des légendes WebVTT .

    Remarques :

    • Pour améliorer la conformité CocoaPods, le SDK natif pour iOS 6.7.7 a renommé certains fichiers Podspec. Pour plus d'informations, reportez-vous à la référence du SDK natif.
    • La cible minimale de déploiement pour tous les frameworks Brightcove Native Player pour iOS est désormais iOS 11.0 et tvOS 11.0.
    • Le SDK natif Brightcove pour iOS 6.5.0+ nécessite Xcode 11.0 ou supérieur.
    • Le comptage automatique des références (ARC) est requis dans votre projet. Il s'agit de la valeur par défaut actuelle pour les projets Xcode.
    • Le SDK natif Brightcove pour TVOS est écrit pour vous permettre de créer des applications personnalisées en utilisant Obj-C et Swift, mais il ne prend pas en charge le TVML (Television Markup Language) d'Apple.
    • Un problème Apple connu avec le contenu en continu FairPlay est qu'il ne sera pas rendu en mode Mirroring AirPlay. Pour plus de détails, consultez la présentation du WWDC : Protection du contenu pour HLS (faites défiler vers la moitié de la transcription).

    Intégrations

    Le tableau suivant contient des liens pour intégrer les plugins à votre implémentation SDK. Les plugins peuvent être téléchargés à partir de leurs Repos Github respectifs, et chaque repo contient un README détaillé avec des instructions sur la façon de configurer le plugin.

    Les exemples d'applications démontrent les fonctionnalités de base de chaque plugin intégré au SDK. Ces exemples d'applications sont un bon moyen d'en apprendre davantage sur chaque plugin, et servent de point de référence lors du débogage.

    La version actuelle du kit SDK natif pour iOS inclut tous les composants ayant le même numéro de version. Lors de l'intégration de composants, les numéros de version du SDK et des composants doivent correspondre. Par exemple, lorsque vous utilisez le composant du plugin IMA 6.7.0, vous devez utiliser le SDK natif 6.7.0.

    Les notes suivantes se réfèrent aux intégrations du tableau ci-dessus.

    [ 1] La version actuelle du plugin IMA prend en charge le SDK v3 iOS IMA de Google, qui prend en charge les versions 9.0+ d'iOS. Pour plus de détails, consultez la documentation de Google.

    Les kits SDK natifs Brightcove prennent en charge les modèles XML VMAP qui contiennent le protocole VAST, y compris VAST 3.0 , et la prise en charge limitée de VAST 4.0. Pour plus d'informations sur la compatibilité des versions VAST, consultez le document Support et compatibilité de Google pour le SDK IMA pour iOS. Les kits SDK natifs prennent également en charge le SDK IAB Open Measurement.

    [ 2] Si vous utilisez Swift, vous devez ajouter manuellement les bibliothèques IMA de Google, au lieu d'utiliser CocoaPods. En effet, la bibliothèque IMA de Google ne prend pas en charge les frameworks dynamiques, et Swift ne prend pas en charge la construction de bibliothèques statiques. Pour plus de détails, reportez-vous à la section Manuel du module IMA Plugin for Brightcove Player SDK pour iOS . Même si le plugin IMA de Brightcove utilise un framework dynamique, la bibliothèque Google IMA n'est fournie qu'en tant que framework statique.

    Meilleures pratiques

    Voici des informations supplémentaires que vous pouvez trouver utiles lorsque vous travaillez avec le SDK natif Brightcove.

    Advertising

    Les annonces complémentaires et les annonces de clics ne sont pas prises en charge sur l'Apple TV. Comme il n'y a pas de prise en charge du navigateur Web sur l'Apple TV, il n'y a pas de destination pour une URL de click-through.
