Configuration requise pour l'App Store

Pour tirer parti des nouvelles fonctionnalités fournies par le SDK iOS, Apple exigera que toutes les nouvelles applications soumises à l'App Store soient créées avec le SDK iOS 12.1 ou une version ultérieure. Cette exigence prend effet à compter du Mars 2019.

Cela signifie que pour soumettre de nouvelles applications, vous devrez utiliser Xcode 10.1, qui fournit les outils pour créer des applications pour les derniers SDK, notamment iOS 12, tvOS 12 et Swift 4.2.

Pour plus de détails, consultez le Soumettre des applications iOS à l'App Store Documentation.

Prise en charge native du SDK

Suivant l'exemple d'Apple, le Brightcove Native SDK pour iOS/tvOS v6.3.10+ prend en charge la construction avec Xcode 10.1. Si vous ne l'avez pas déjà fait, veuillez migrer vers Xcode 10.1 dès que possible.

Le Brightcove Native SDK pour iOS/tvOS v6.3.5+ ne prend plus en charge la construction avec Xcode 8.
