Vous trouverez ci-dessous une liste de questions fréquemment posées associées aux SDK Brightcove Native Player. Vous avez une question que vous ne voyez pas ici ? Faites le nous savoir!

Questions générales

Les sous-titres codés sont-ils pris en charge ?

Les SDK Player prennent en charge les sous-titres WebVTT (HLS) in-manifest ainsi que WebVTT side-car et 608-over-708. Pour plus de détails sur l'assistance, consultez le Formats vidéo et sous-titrage document.


Puis-je personnaliser la barre de recherche ?

Oui. Si vous utilisez les commandes Brightcove Native Player, vous pouvez personnaliser les éléments, y compris la barre de recherche. Pour plus d'informations, consultez le Commandes natives du lecteur Brightcove document.


Puis-je superposer la vidéo avec des commandes personnalisées ?

Les SDK Player ne vous obligent pas à utiliser ses commandes. Vous pouvez choisir de les utiliser ou non. Si vous utilisez les commandes Brightcove Native Player, vous pouvez personnaliser les éléments. Pour plus d'informations, consultez le Commandes natives du lecteur Brightcove document.


Puis-je ajouter une fonctionnalité vidéo ff/rew de 10 secondes ?

Les SDK Player prennent en charge la recherche, vous pouvez donc implémenter cette fonctionnalité.


Le Chromecast est-il compatible ?

Les kits de développement logiciel Brightcove Native prennent en charge la diffusion de vidéos sur des téléviseurs connectés à Chromecast.

  • Pour les appareils Android, utilisez le Plugin Cast pour le SDK natif pour Android.
  • Pour les appareils iOS, utilisez le Plugin Cast pour le SDK natif pour iOS.

Les listes de lecture sont-elles prises en charge ?

Oui, les kits SDK natifs de Brightcove fournissent des méthodes pour récupérer une vidéo ou une liste de lecture à partir de l'API de lecture. Pour plus de détails, voir ce qui suit :

  • VideoListSampleApp pour le SDK natif pour Android
  • Récupération des ressources Brightcove à l'aide du service de lecture pour le SDK natif pour iOS

Spécifique à Android

Comment puis-je faire fonctionner HLS sur Android ?

Le Player SDK de Brightcove pour Android fournit une fonctionnalité de lecture HLS pour Android 4.1 et plus récent.


Et HLSe ?

Le Player SDK de Brightcove pour Android prend en charge la lecture avec HLSe.

Spécifique à iOS

Est-ce qu'AirPlay est pris en charge ?

Le SDK du lecteur pour iOS prend en charge AirPlay tant que vous n'avez pas de publicités. AirPlay d'Apple vous permet de diffuser sans fil du contenu à partir de vos appareils Apple.


L'Apple TV est-elle prise en charge ?

La dernière version du Player SDK pour iOS prend en charge le système d'exploitation tvOS d'Apple TV.
