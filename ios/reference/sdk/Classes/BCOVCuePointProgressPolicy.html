Aperçu

Cette politique fournit aux clients de BCOVPlayerSDK les moyens de spécifier le comportement du code qui agit en réponse aux événements de point de repère. L'application la plus courante consiste à créer des règles publicitaires : la tête de lecture atteint un point de repère qui indique qu'une annonce doit être diffusée, et une instance de cette règle est fournie par le code client pour spécifier comment le lecteur doit se comporter, en ce qui concerne :

  1. Quels points de repère doivent être traités par le code consultant la politique.
  2. L'heure (position de la tête de lecture) à laquelle la lecture doit reprendre une fois que les points de repère ont été traités. Ceci est spécifié à l'aide de l'un des points de repère qui a été transmis à la stratégie, ou néant (ce qui indique que le contenu doit reprendre à partir de sa position actuelle de tête de lecture).

– appliquer à l'événement :

Renvoie un BCOVCuePointProgressPolicyResult qui spécifie quels points de repère doivent être traités par le code consultant cette politique, ainsi que la position à laquelle la lecture doit reprendre une fois que tous les points de repère ont été traités.

- (BCOVCuePointProgressPolicyResult *)applyToEvent:(NSDictionary *)cuePointEvent

Paramètres

cuePointEvent

Un dictionnaire contenant des informations sur les événements de point de repère, comme spécifié dans le Contrôleur de lecture BCOV méthode déléguée de point de repère.

Valeur de retour

Un résultat de stratégie pour l'événement de point de repère spécifié.

Discussion

Renvoie un BCOVCuePointProgressPolicyResult qui spécifie quels points de repère doivent être traités par le code consultant cette politique, ainsi que la position à laquelle la lecture doit reprendre une fois que tous les points de repère ont été traités.

Cette méthode est appelée par le consommateur de la politique de progression du point de repère. Dans la plupart des cas, ce consommateur est un plug-in publicitaire pour le Brightcove Player SDK, et non le client (application) du Player SDK. Une stratégie de point de repère personnalisée doit remplacer cette méthode si une stratégie ne peut pas être produite à l'aide d'une méthode de classe de fabrique pratique.
