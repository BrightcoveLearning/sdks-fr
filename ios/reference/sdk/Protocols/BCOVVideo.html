Aperçu

Une vidéo dans l'écosystème Brightcove Player SDK. Brightcove organise le contenu en entités appelées vidéos, qui peuvent avoir zéro ou plus sources contenant des informations spécifiques à l'accès au contenu. La vidéo fournit un conteneur pour référencer ces sources dans l'ensemble, et un endroit pour les métadonnées sur la vidéo ou la collection de sources sous la forme de Propriétés. En outre, une vidéo peut avoir une collection de points de repère, qui est un ensemble de points de repère qui décrivent des emplacements spécifiques dans la vidéo’chronologie du contenu.

  points de repère méthode requise

Les points de repère associés à cette vidéo. Ces points de repère s'appliquent également au contenu de chacune de cette vidéo’s sources.

Les points de repère associés à cette vidéo. Ces points de repère s'appliquent également au contenu de chacune de cette vidéo’s sources.

– mettre à jour: méthode requise

Renvoie une version modifiée de cette source. Étant donné que les objets BCOVVideo sont immuables, un tout nouveau BCOVVideo doit être créé même si une seule modification est nécessaire. Par conséquent, cette méthode fournit un moyen pratique d'obtenir une copie modifiable temporaire de cette instance et de la modifier (dans updateBlock).

- (instancetype)update :(void ( ^ ) ( id<BCOVMutableVideo> mutableVideo ))updateBlock

Paramètres

updateBlock

Un bloc qui est passé une copie mutable de cette vidéo.

Valeur de retour

La copie de cette vidéo modifiée par updateBlock.

Discussion

Renvoie une version modifiée de cette source. Étant donné que les objets BCOVVideo sont immuables, un tout nouveau BCOVVideo doit être créé même si une seule modification est nécessaire. Par conséquent, cette méthode fournit un moyen pratique d'obtenir une copie modifiable temporaire de cette instance et de la modifier (dans updateBlock).

N'essayez pas d'enregistrer une référence à la vidéo mutable transmise à updateBlock , car le comportement de cette opération n'est pas défini. Au lieu de cela, apportez les modifications nécessaires dans le bloc, puis capturez l'objet vidéo renvoyé par cette méthode.
