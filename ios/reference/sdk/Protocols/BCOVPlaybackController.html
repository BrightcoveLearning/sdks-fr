Conforme à NSObject
Déclaré en BCOVPlaybackController.h

Aperçu

Protocole adopté par les objets qui fournissent une fonctionnalité de lecture.

Les implémentations de ce protocole formel doivent prendre en charge ses opérations de lecture standard, mais peuvent étendre l'API pour exécuter des fonctions supplémentaires spécifiques à leur ensemble de fonctionnalités.

  annoncesDésactivé méthode requise

SSAI uniquement : Si vous utilisez d'autres plug-ins publicitaires, utilisez seekWithoutAds:completionHandler : au lieu.

@property (non atomique, readwrite) BOOL adsDisabled

Discussion

SSAI uniquement : Si vous utilisez d'autres plug-ins publicitaires, utilisez seekWithoutAds:completionHandler : au lieu.

Désactive la lecture d'annonces qui pourrait autrement se produire après un forward -seekTo :. L'utilisation prévue est, par exemple, de reprendre la lecture vidéo lorsqu'une application a été fermée et relancée ; l'application peut chercher à gagner du temps et les annonces qui ont déjà été vues ne seront pas jouer de nouveau. Définissez adsDisabled sur YES, puis appelez l'une des méthodes -seekToTime:, et enfin définissez adsDisabled sur NO dans le gestionnaire d'achèvement -seekToTime:. La valeur par défaut de adsDisabled est NO.

Notez qu'à moins que le BCOVPlaybackSessionLifecycleEvent kBCOVPlaybackSessionLifecycleEventReady a été reçu, le gestionnaire d'achèvement -seekTo: ne sera jamais appelé. Pour cette raison, il est recommandé d'appeler -seekTo: depuis le -playbackController:playbackSession:didReceiveLifecycleEvent déléguer pour l'événement kBCOVPlaybackSessionLifecycleEventReady.

– resumeVideoAtTime:withAutoPlay: méthode requise

Demande à cette instance de réinitialiser la session en cours. S'il n'y a pas de session en cours, cette méthode n'a aucun effet. Une fois cette méthode appelée, elle enverra l'événement kBCOVPlaybackSessionLifecycleEventResumeBegin pour indiquer que la réinitialisation a commencé. Si une erreur se produit n'importe où pendant la réinitialisation, cette méthode enverra l'événement kBCOVPlaybackSessionLifecycleEventResumeFail. Si la réinitialisation réussit, le kBCOVPlaybackSessionLifecycleEventResumeComplete sera envoyé.

- (void)reprendreVideoAtTime :(CMTime)temps avec lecture automatique :(BOOL)lecture automatique

Paramètres

temps

Cherchez le temps de réglage après la réinitialisation.
lecture automatique

Automatiquement jouer nouvelle session une fois qu'il est redémarré.

Discussion

Demande à cette instance de réinitialiser la session en cours. S'il n'y a pas de session en cours, cette méthode n'a aucun effet. Une fois cette méthode appelée, elle enverra l'événement kBCOVPlaybackSessionLifecycleEventResumeBegin pour indiquer que la réinitialisation a commencé. Si une erreur se produit n'importe où pendant la réinitialisation, cette méthode enverra l'événement kBCOVPlaybackSessionLifecycleEventResumeFail. Si la réinitialisation réussit, le kBCOVPlaybackSessionLifecycleEventResumeComplete sera envoyé.

N'appelez pas cette méthode une deuxième fois tant que vous n'avez pas reçu kBCOVPlaybackSessionLifecycleEventResumeFail ou kBCOVPlaybackSessionLifecycleEventResumeComplete du premier appel.

– addFairPlayApplicationCertificate:identifiant : méthode requise

Cette méthode est utilisée pour ajouter un certificat d'application FairPlay au contrôleur de lecture’s liste des certificats d'application FairPlay disponibles. Les certificats seront conservés pendant toute la durée de vie du contrôleur de lecture.

- (void) addFairPlayApplicationCertificate :(NSData *)applicationCertificatDonnées identifiant :(NSString *)identifiant

Discussion

Cette méthode est utilisée pour ajouter un certificat d'application FairPlay au contrôleur de lecture’s liste des certificats d'application FairPlay disponibles. Les certificats seront conservés pendant toute la durée de vie du contrôleur de lecture.

Si vous utilisez Dynamic Delivery, les certificats d'application sont récupérés automatiquement par le plug-in FairPlay, cette méthode n'est donc pas nécessaire. Vous pouvez toutefois utiliser cette méthode pour précharger votre certificat d'application afin d'accélérer la lecture de la première vidéo cryptée FairPlay. Les certificats sont stockés et réutilisés pour les vidéos suivantes.

@param applicationCertificateData Le certificat d'application FairPlay dans un objet NSData. S'il est défini sur nil, tout certificat d'application existant pour l'identifiant donné sera supprimé. @param identifier Une chaîne utilisée pour localiser le certificat d'application. Cette chaîne ne peut pas être nulle. - Pour la livraison dynamique, l'identifiant doit être l'URL qui a été utilisée pour récupérer ce certificat. - Pour les anciens comptes Video Cloud, ce paramètre doit être défini sur kBCOVDefaultFairPlayApplicationCertificateIdentifier pour s'appliquer à tous les comptes. - Si vous utilisez plusieurs comptes Video Cloud hérités, définissez ce paramètre sur l'ID de compte.
