Référence du protocole BCOVOfflineVideoManagerDelegate

Conforme à NSObject
Déclaré en BCOVOfflineVideoManagerDelegate.h

Aperçu

Conformez-vous à ce protocole pour recevoir des informations sur chaque téléchargement de vidéo et de piste vidéo.

– didCreateSharedBackgroundSesssionConfiguration :

Cette méthode est appelée après la création de l'objet d'arrière-plan NSURLSessionConfiguration et avant’s utilisé pour créer l'objet partagé AVAssetDownloadURLSession qui’s utilisé pour télécharger des vidéos.

- (void)didCreateSharedBackgroundSesssionConfiguration :(NSURLSessionConfiguration *)backgroundSessionConfiguration

Discussion

Vous pouvez utiliser cet objet de configuration pour définir diverses options spécifiées pour NSURLSessionConfiguration dans NSURLSession.h, telles que le discrétionnaire drapeau, ou le HTTPMaximumConnectionsParHost réglage.

Vous devriez ne pas met le autoriseAccès Cellulaire biens; qui est défini dans les tâches de téléchargement individuelles.

– didDownloadStaticImagesWithOfflineVideoToken :

Cette méthode est appelée lorsque les images statiques (vignette et affiche) associées à une vidéo sont téléchargées sur le stockage.

- (void) a faitDownloadStaticImagesWithOfflineVideoToken :(BCOVOfflineVideoToken)hors ligneVideoToken

Paramètres

hors ligneVideoToken

Jeton vidéo hors ligne utilisé pour identifier la vidéo téléchargée.

Discussion

Les URL des fichiers des images téléchargées sont stockées en tant que propriétés sur un BCOVVidéo créé à partir du jeton vidéo hors ligne à l'aide de -[videoObjectFromOfflineVideoToken:] Les propriétés sont récupérées à l'aide des clés kBCOVOfflineVideoThumbnailFilePath et kBCOVOfflineVideoPosterFilePath.

Cette méthode sera appelée même si aucune image n'a pu être récupérée, alors assurez-vous de vérifier l'UIImage que vous créez à partir des chemins des affiches et des vignettes. Les chemins de fichiers seront présents, mais il se peut qu'aucune image ne soit stockée à ces emplacements.

– shouldDeleteVideoPackage :

Cette méthode est appelée lors du nettoyage des téléchargements bloqués.

- (BOOL)devraitDeleteVideoPackage :(NSString *)videoPackagePath

Paramètres

videoPackagePath

Chemin d'accès au répertoire du package vidéo qui est signalé comme bloqué.

Discussion

Les téléchargements peuvent être bloqués si vous forcez la fermeture d'une application pendant un téléchargement ou si vous redémarrez pendant que l'application est suspendue. Vous n'avez pas besoin d'implémenter cette méthode à moins que vous ne commenciez vos propres téléchargements de vidéos en dehors du SDK Brightcove et que vous ne’Je ne veux pas que vos vidéos soient supprimées. Cette méthode est appelée lorsqu'un package vidéo est marqué pour suppression, ce qui signifie qu'il n'y a aucune trace de celui-ci dans le Brightcove SDK.

S'il s'agit d'un fichier que vous reconnaissez, vous devez retourner NON. Sinon, le retour de YES entraînera la suppression du package vidéo. Si vous n'implémentez pas cette méthode, les téléchargements échoués seront supprimés par défaut.

– offlineVideoStorageDidChange

Cette méthode est appelée lorsque l'application est activée et que le stockage vidéo hors ligne est modifié alors que l'application n'était pas en cours d'exécution ou en arrière-plan, par exemple lorsque l'utilisateur supprime des vidéos téléchargées via les paramètres de stockage de l'appareil dans les paramètres généraux de l'application Paramètres.

- (vide) hors ligneVideoStorageDidChange

Discussion

Lorsqu'il est appelé, vous devez mettre à jour votre interface utilisateur pour refléter les modifications apportées au stockage hors ligne.

La détection des modifications apportées au stockage vidéo hors ligne se produit chaque fois que l'application devient active.

Cette méthode est appelée sur le thread principal.
