Aperçu

Implémentez ce protocole lors de l'interaction avec un serveur de licences FPS personnalisé.

– contentIdentifierFromLoadingRequest :

Méthode facultative. Implémentez cette méthode pour gérer la récupération de l'identifiant de contenu à partir de la demande de chargement. Ceci est appelé pour chaque actif. Aucune garantie n'est donnée quant au fil auquel cela est appelé. Vous devez renvoyer l'identifiant de contenu correspondant à cette demande de chargement. Celui-ci est généralement extrait du “skd://…” URL dans loadingRequest.request.URL. L'identifiant de contenu sera transmis à [AVAssetResourceLoadingRequest streamingContentKeyRequestDataForApp:contentIdentifier:options:error:] pour récupérer le SPC.

- (Nullable NSData *)contentIdentifierFromLoadingRequest:(nonnull AVAssetResourceLoadingRequest *)chargementDemande

Paramètres

chargementDemande

La demande de chargement skd:// effectuée pour l'actif.

Discussion

Méthode facultative. Implémentez cette méthode pour gérer la récupération de l'identifiant de contenu à partir de la demande de chargement. Ceci est appelé pour chaque actif. Aucune garantie n'est donnée quant au fil auquel cela est appelé. Vous devez renvoyer l'identifiant de contenu correspondant à cette demande de chargement. Celui-ci est généralement extrait du “skd://…” URL dans loadingRequest.request.URL. L'identifiant de contenu sera transmis à [AVAssetResourceLoadingRequest streamingContentKeyRequestDataForApp:contentIdentifier:options:error:] pour récupérer le SPC.

Si cette méthode n'est pas fournie, la méthode par défaut sera utilisée. La méthode par défaut supprime “skd://” de loadingRequest.request.URL pour générer l'identifiant de contenu.
