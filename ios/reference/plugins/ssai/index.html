Installation

Le plug-in SSAI pour Brightcove Player SDK pour iOS fournit deux packages d'installation ; un framework de bibliothèque statique et un framework de bibliothèque dynamique.

CocoaPods

Vous pouvez utiliser CocoaPods pour ajouter le plug-in SSAI pour Brightcove Player SDK à votre projet. Vous pouvez trouver les dernières Brightcove-Player-SSAI podspec ici.

Exemple de cadre dynamique :

```source bash ‘https://github.com/brightcove/BrightcoveSpecs.git'

use_frameworks ! plateforme :ios, ‘11,0’

cible ‘MonLecteurVidéo’ faire la cosse ‘Brightcove-Player-SSAI’ fin ```

Exemple de cadre statique :

```source bash ‘https://github.com/brightcove/BrightcoveSpecs.git’

use_frameworks ! plateforme :ios, 11.0'

cible ‘MonLecteurVidéo’ faire la cosse ‘Brightcove-Player-SSAI-statique’ fin ```

Lors de la mise à jour de votre installation, il’C'est une bonne idée d'actualiser la copie locale de votre référentiel BrightcoveSpecs afin que vous disposiez des derniers podspecs localement, tout comme vous mettriez à jour votre référentiel maître CococaPods.

Généralement si vous utilisez mise à jour du pod dans Terminal, cela se produira automatiquement, ou vous pouvez également mettre à jour explicitement avec pod repo mise à jour brightcove. (Votre référentiel BrightcoveSpecs peut avoir un nom différent si vous l'avez explicitement ajouté à votre liste de référentiels podspecs.)

Manuel

Pour ajouter manuellement le plug-in SSAI pour Brightcove Player SDK à votre projet :

  1. Installez la dernière version du Kit de développement logiciel Brightcove Player.
  2. Téléchargez la dernière version zippée du plugin BrightcoveSSAI depuis notre page de publication.
  3. Ajouter BrightcoveSSAI.framework à votre projet.
  4. Sur le “Paramètres de construction” de l'onglet cible de votre application, assurez-vous que le “Chemins de recherche du framework” inclure le chemin d'accès au cadre. Cela aurait dû être fait automatiquement à moins que le framework ne soit stocké dans un répertoire racine différent de celui de votre projet.
  5. (Dynamic Framework uniquement) Sur le “Général” onglet de votre application cible, ajoutez ‘BrightcoveSSAI.framework’ à la “Frameworks, bibliothèques, contenu intégré” section.
  6. (Dynamic Framework uniquement) Sur le “Phases de construction” onglet, ajoutez un “Script de lancement” phase avec la commande coup ${BUILT_PRODUCTS_DIR}/${FRAMEWORKS_FOLDER_PATH}/BrightcoveSSAI.framework/strip-frameworks.sh. Vérifier “Exécuter le script uniquement lors de l'installation”. Cela supprimera les architectures inutiles de la construction, ce qui est important pour la soumission App Store.
  7. (Cadre statique uniquement) Sur le “Paramètres de construction” onglet de votre application cible, ajoutez -ObjC à la “Autres indicateurs de liens” paramètre de construction.

Importations

Le plug-in SSAI pour Brightcove Player SDK pour iOS peut être importé dans le code de différentes manières ; @import BrightcoveSSAI; , #import <BrightcoveSSAI/BrightcoveSSAI.h> ou #import <BrightcoveSSAI/[classe spécifique].h>.

Obtention d'informations sur la lecture des annonces

BrightcoveSSAI fournit des informations sur la lecture des publicités via le BCOVPlaybackControllerAnnoncesDélégué. Par exemple, si vous souhaitez masquer vos contrôles pendant une annonce, vous pouvez implémenter -[BCOVPlaybackControllerAdsDelegate playController:playbackSession:didEnterAdSequence:] pour les cacher.

Pour plus d'informations sur l'utilisation de ces méthodes déléguées, veuillez consulter Kit de développement logiciel Brightcove Player pour iOS.

En cherchant

Le plugin BrightcoveSSAI fournit une fonction de recherche qui doit être utilisée lors de la mise en œuvre des contrôles. Cette fonction de recherche est exposée sur la session via le fournisseurExtension biens. Voici comment il est utilisé :

```objc
CMTime contentTimeToSeekTo = <calcul-from-scrub-bar>; [self.currentSession.providerExtension ssai_seekToTime:contentTimeToSeekToachèvementHandler:^(BOOL terminé) {

si (fini) { [self.currentSession.player play]; }

}] ; ```

Les achèvementGestionnaire s'exécutera à la fin d'une recherche réussie. Il ne s'exécutera pas si une recherche a déjà été initiée par un appel précédent à [BCOVSessionProviderExtension ssai_seekToTime:completionHandler :] ou si une annonce est en cours de lecture. Pour tester si une tentative de recherche peut être effectuée, vérifiez la [BCOVSessionProviderExtension ssai_canSeek] biens. Pour plus d'informations sur ces deux méthodes, assurez-vous de lire le doc d'en-tête.

Rechercher sans publicité

Le BrightcovePlayerSDK fournit le annoncesDésactivé Propriété BOOL pour désactiver les publicités lors de la recherche, permettant à une application de reprendre la lecture sans obliger l'utilisateur final à afficher les publicités précédemment jouées.

En vue de la recherche, désactiver lecture automatique lors de la configuration du Contrôleur de lecture BCOV.

Une logique de désactivation des publicités doit être ajoutée au kBCOVPlaybackSessionLifecycleEventReady gestionnaire de la -playbackController:playbackSession:didReceiveLifecycleEvent : méthode de votre Contrôleur de lecture BCOV déléguer.

```objc - (void)playbackController:(id)contrôleur playbackSession:(id)session didReceiveLifecycleEvent:(BCOVPlaybackSessionLifecycleEvent *)lifecycleEvent { if ([kBCOVPlaybackSessionLifecycleEventReady isEqualToString:lifecycleEvent.eventType]) { // désactiver les publicités. _playbackController.adsDisabled = OUI;

// recherche quelque part dans le contenu vidéo. [session.providerExtension ssai_seekToTime:resumeTimeachèvementHandler:^(BOOL terminé) { // re-enable ads. _playbackController.adsDisabled = NO; // open the shutter. _playbackController.shutterFadeTime = 0.25; _playbackController.shutter = NO; } ;

} ```

Lors de l'appel ssai_seekToTime:completionHandler : pour reprendre la lecture à un moment donné, la première image de la vidéo peut être visible jusqu'à la fin de la recherche. Pour une présentation plus nette, couvrez temporairement la vue vidéo pendant la recherche en réglant le obturateur propriété de BCOVplabackController à OUI avant d'appeler -setVidéos :. Lorsque la recherche est terminée, fermez l'obturateur en réglant le obturateur propriété à NON. Les obturateurFadeTime La propriété définit la durée de l'animation de fondu de l'obturateur.

```objc self.playbackController = [sdkManager createSSAIPlaybackController];

// activer l'obturateur avant de charger la vidéo. self.playbackController.shutterFadeTime = 0.0; self.playbackController.shutter = OUI;

// charge la vidéo. ```

Notez que pour des raisons de performances, de petites tolérances sont intégrées à la recherche vidéo. Une recherche vers une position de tête de lecture de contenu proche du début d'une séquence publicitaire peut entraîner une recherche vers le début de la séquence publicitaire. Si la lecture automatique est NON dans ce cas, AVPlayer sera mis en pause sur la première image de la séquence publicitaire.

Accès aux données de réponse VMAP ========================== Si vous souhaitez accéder aux données de réponse VMAP, vous pouvez vous abonner à la notification `kBCOVSSAIVMAPResponseReceivedNotification`. Une fois reçu, le dictionnaire userInfo de la notification contiendra la réponse VMAP en tant que NSData. Vous pouvez utiliser la constante `kBCOVSSAIVMAPResponseReceivedNotificationDataUserInfoKey` pour y accéder depuis userInfo. Étant donné que vous pouvez avoir plusieurs contrôleurs de lecture, et donc plusieurs réponses VMAP, vous pouvez vérifier l'objet de la notification, `id playbackController = notification.object` pour vérifier à quelle vidéo les données VMAP sont destinées. De plus, lorsque vous vous abonnez à la notification, vous pouvez définir un contrôleur de lecture comme objet afin que seules les notifications de données VMAP concernant ce contrôleur de lecture soient reçues. Par exemple : ```objc [[NSNotificationCenter defaultCenter] addObserver:self selector:@selector(vmapResponseReceived:) name:kBCOVSSAIVMAPResponseReceivedNotification object:self.playbackController];

Problèmes connus

  • Étant donné que tvOS ne prend pas en charge la navigation Web, les annonces associées, En savoir plus et tous les clics publicitaires sont ignorés sur cette plate-forme.

  • Vous ne pouvez pas utiliser BrightcoveSSAI avec d'autres plug-ins Brightcove, à l'exception du plug-in BrightcoveFairPlay ou du plug-in BrightcoveIMA (pour les annonces pré-roll uniquement).

  • Si vous souhaitez utiliser le plug-in BrightcoveSSAI avec le plug-in BrightcoveFairPlay, le plug-in BrightcoveFairPlay doit être le fournisseur de session en amont du fournisseur de session SSAI.

  • Le plugin SSAI ne prend pas en charge la lecture de contenu non-SSAI. Si vous devez lire du contenu non-SSAI, vous devez créer un autre contrôleur de lecture qui n'utilise pas le plug-in SSAI.
