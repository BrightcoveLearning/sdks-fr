Référence BCOVPlayerUI

Plugin BCOVPlayerUI pour Brightcove Player SDK pour iOS, version 1.0.4.143

Plateformes prises en charge

iOS 7.0 et supérieur.

Installation

Le plugin Brightcove PlayerUI fournit deux packages d'installation pour iOS, un framework de bibliothèque statique et un framework dynamique. La cible de bibliothèque statique prend en charge le déploiement sur iOS 7 tandis que le cadre dynamique ne prend en charge que iOS 8 et versions ultérieures.

CocoaPods

Vous pouvez utiliser CocoaPods pour ajouter le plug-in PlayerUI pour Brightcove Player SDK à votre projet. Vous pouvez trouver les dernières Brightcove-Player-SDK-Player-UI podspec ici.

Exemple de cadre statique :

pod 'Brightcove-Player-SDK-Player-UI'

Exemple de cadre dynamique :

pod 'Brightcove-Player-SDK-Player-UI/dynamic'

Manuel

Pour ajouter manuellement le plug-in PlayerUI pour Brightcove Player SDK à votre projet :

  1. Installez la dernière version du Kit de développement logiciel Brightcove Player.
  2. Téléchargez la dernière version zippée du plugin depuis notre page de publication.
  3. Ajouter le ‘BrightcovePlayerUI.framework’ à votre projet.
  4. Sur le “Paramètres de construction” de l'onglet cible de votre application, assurez-vous que le “Chemins de recherche du framework” inclure le chemin d'accès au cadre. Cela aurait dû être fait automatiquement à moins que le framework ne soit stocké dans un répertoire racine différent de celui de votre projet.
  5. Sur le “Phases de construction” de l'onglet cible de votre application, ajoutez ce qui suit à la “Lier le binaire avec les bibliothèques” phase:
    • BrightcovePlayerUI.framework
  6. (Dynamic Framework uniquement) Sur le “Général” onglet de votre application cible, ajoutez ‘BrightcovePlayerUI.framework’ à la “Binaire intégré” section.
  7. (Dynamic Framework uniquement) Sur le “Phases de construction” onglet, ajoutez un “Script de lancement” phase avec la commande coup ${BUILT_PRODUCTS_DIR}/${FRAMEWORKS_FOLDER_PATH}/BrightcovePlayerUI.framework/strip-frameworks.sh. Vérifier “Exécuter le script uniquement lors de l'installation”. Cela supprimera les architectures inutiles de la construction, ce qui est important pour la soumission App Store. ([rdar://19209161][19209161])
  8. (Cadre statique uniquement) Sur le “Paramètres de construction” onglet de votre application cible, ajoutez -ObjC à la “Autres indicateurs de liens” paramètre de construction.
  9. (Frame statique uniquement) Ajouter le fichier bcovpuiiconfont.ttf du BrightcovePlayerUI.framework bundle directement dans votre liste de projets afin que le fichier de police soit copié dans le bundle d'applications. En fin de compte, le fichier de police devrait se retrouver au même niveau que l'application’s Info.plist déposer. Le fichier de police fournit certains des éléments de l'interface BrightcovePlayerUI, mais il n'a pas besoin d'être répertorié dans le plist lui-même.

Importations

Le Brightcove Player SDK pour iOS peut être importé dans le code de différentes manières ; @import BrightcovePlayerUI; , #import <BrightcovePlayerUI/BrightcovePlayerUI.h> ou #import <BrightcovePlayerUI/[classe spécifique].h>.

Démarrage rapide

Son utilisation est relativement simple. Cet exemple crée une mise en page de base adaptée aux flux vidéo à la demande généraux :

 BCOVPlayerSDKManager *manager = [BCOVPlayerSDKManagersharedManager] ; [1] id<BCOVPlaybackController> contrôleur = [gestionnaire createPlaybackController] ; [2] BCOVPUIPlayerView *playerView = [[BCOVPUIPlayerView alloc] initWithPlaybackController:playbackController]; playerView.frame = self.videoContainer.bounds; playerView.autoresizingMask = UIViewAutoresizingFlexibleHeight | UIViewAutoresizingFlexibleWidth; [self.videoContainer addSubview:playerView] ; [3] NSString *jeton ; // (Jeton API Brightcove Media avec accès URL) NSString *playlistID; // (ID de la liste de lecture que vous souhaitez utiliser) BCOVCatalogService *catalog = [[BCOVCatalogService alloc] initWithToken:token]; [catalogue findPlaylistWithPlaylistID:playlistID parameters:nil achèvement :^(BCOVPlaylist *playlist, NSDictionary *jsonResponse, NSError *error) { [controller setVideos:playlist]; [controller play]; } ;

Laisser’s décomposez ce code en étapes, pour le rendre un peu plus simple à digérer :

  1. Créez un contrôleur de lecture avec ses paramètres par défaut.
  2. Créez une vue du lecteur à l'aide du contrôleur de lecture. Cela instanciera une vue du joueur avec ses paramètres par défaut. Réglez la vue du lecteur pour qu'elle corresponde à la taille de la vue de votre conteneur vidéo.
  3. Récupérer et lire une vidéo.

Personnalisation du comportement du plugin

Le plugin BCOVPlayerUI offre une expérience simple et prête à l'emploi. Cependant, il’s possible de modifier le comportement par défaut.

BCOVPUIPlayerViewOptions

Les BCOVPUIPlayerViewOptions La classe vous permet de personnaliser certains comportements de BCOVPlayerUI lors de l'initialisation. Vous pouvez personnaliser :

  • jumpBackInterval Le temps en secondes pendant lequel le joueur reviendra en arrière lorsque le bouton de retour en arrière est enfoncé.
  • masquerIntervalleContrôles Le temps en secondes après le dernier événement tactile, avant que les commandes ne soient masquées.
  • masquerContrôlesAnimationDurée Le temps en secondes qu'il faut pour que les contrôles s'animent jusqu'à ce qu'ils soient masqués.
  • showControlsAnimationDurée Le temps en secondes nécessaire pour que les contrôles s'animent et deviennent visibles.
  • présentationViewController Le contrôleur de vue à utiliser pour présenter des choses comme les modaux et le plein écran.

Les options peuvent être définies à l'aide de la méthode suivante :

 BCOVPlayerSDKManager *manager = [BCOVPlayerSDKManagersharedManager] ; id<BCOVPlaybackController> contrôleur = [gestionnaire createPlaybackController]; BCOVPUIPlayerViewOptions *options = [[BCOVPUIPlayerViewOptions alloc] init]; options.jumpBackInterval = 5; BCOVPUIPlayerView *playerView = [[BCOVPUIPlayerView alloc] initWithPlaybackController:playbackController options:options];

Dispositions fournies

Trois dispositions sont fournies pour prendre en charge différents types de vidéo :

  • BCOVPUIControlLayout basicVODControlLayout est une disposition de base pour les flux vidéo à la demande généraux.

  • BCOVPUIControlLayout basicLiveControlLayout est une mise en page ciblée vers la vidéo en direct.

  • BCOVPUIControlLayout basicLiveDVRControlLayout est destiné aux flux vidéo en direct avec des commandes DVR.

Vous définissez généralement une nouvelle mise en page immédiatement après votre BCOVPUIPlayerView est créé, mais vous pouvez également définir une nouvelle mise en page à tout moment. Par exemple, vous pouvez définir une nouvelle mise en page VOD comme celle-ci :

playerview.controlView.layout = [BCOVPUIControlLayout basicVODControlLayout]

Dispositions personnalisées

En plus des mises en page par défaut, vous pouvez créer vos propres mises en page hautement personnalisées en instanciant un nouveau BCOVPUIControlLayout avec votre propre conception.

Tout d'abord, créez les contrôles qui iront dans votre mise en page en utilisant BCOVPUIBasicControlView layoutViewWithControlFromTag:width:elasticity:. Chaque contrôle est conditionné dans un BCOVPUILayoutView qui détermine l'espacement des commandes. Vous pouvez définir le largeur de chaque vue de mise en page à la largeur par défaut (qui est basée sur le type de contrôle), ou vous pouvez spécifier votre propre largeur.

Les élasticité L'argument détermine de combien la vue de disposition contenant le contrôle redimensionne sa largeur pour remplir la barre de contrôle. Une élasticité de zéro signifie que la taille de la vue de mise en page sera fixe. Une valeur d'élasticité supérieure à zéro détermine de combien la vue de mise en page s'agrandira pour remplir l'espace disponible par rapport à toutes les autres vues élastiques dans cette barre de contrôle. Une vue de mise en page avec une valeur d'élasticité de 2,0 croîtra deux fois plus vite qu'une vue de mise en page avec une valeur d'élasticité de 1,0. En règle générale, une rangée de vues de mise en page aura au moins un contrôle avec une élasticité supérieure à zéro.

Voici des exemples de création de divers contrôles de base.

BCOVPUILayoutView *playbackLayoutView = [BCOVPUIBasicControlView layoutViewWithControlFromTag:BCOVPUIViewTagButtonPlayback width:kBCOVPUILayoutUseDefaultValue elasticity:0.0] ; BCOVPUILayoutView *jumpBackButtonLayoutView = [BCOVPUIBasicControlView layoutViewWithControlFromTag:BCOVPUIViewTagButtonJumpBack width:kBCOVPUILayoutUseDefaultValue elasticity:0.0] ; BCOVPUILayoutView *currentTimeLayoutView = [BCOVPUIBasicControlView layoutViewWithControlFromTag:BCOVPUIViewTagLabelCurrentTime width:kBCOVPUILayoutUseDefaultValue elasticity:0.0] ; BCOVPUILayoutView *timeSeparatorLayoutView = [BCOVPUIBasicControlView layoutViewWithControlFromTag:BCOVPUIViewTagLabelTimeSeparator width:kBCOVPUILayoutUseDefaultValue elasticity:0.0] ; BCOVPUILayoutView *durationLayoutView = [BCOVPUIBasicControlView layoutViewWithControlFromTag:BCOVPUIViewTagLabelDuration width:kBCOVPUILayoutUseDefaultValue elasticity:0.0] ; BCOVPUILayoutView *progressLayoutView = [BCOVPUIBasicControlView layoutViewWithControlFromTag:BCOVPUIViewTagSliderProgress width:kBCOVPUILayoutUseDefaultValue elasticity:1.0] ; BCOVPUILayoutView *closedCaptionLayoutView = [BCOVPUIBasicControlView layoutViewWithControlFromTag:BCOVPUIViewTagButtonClosedCaption width:kBCOVPUILayoutUseDefaultValue elasticity:0.0] ; BCOVPUILayoutView *screenModeLayoutView = [BCOVPUIBasicControlView layoutViewWithControlFromTag:BCOVPUIViewTagButtonScreenMode width:kBCOVPUILayoutUseDefaultValue elasticity:0.0] ; BCOVPUILayoutView *externalRouteLayoutView = [BCOVPUIBasicControlView layoutViewWithControlFromTag:BCOVPUIViewTagViewExternalRoute width:kBCOVPUILayoutUseDefaultValue elasticity:0.0] ; BCOVPUILayoutView *spacerLayoutView1 = [BCOVPUIBasicControlView layoutViewWithControlFromTag:BCOVPUIViewTagViewEmpty width:1.0 élasticité:1.0]; BCOVPUILayoutView *spacerLayoutView2 = [BCOVPUIBasicControlView layoutViewWithControlFromTag:BCOVPUIViewTagViewEmpty width:1.0 élasticité:1.0]; BCOVPUILayoutView *logoLayoutView1 = [BCOVPUIBasicControlView layoutViewWithControlFromTag:BCOVPUIViewTagViewEmpty width:80.0 elasticity:1.0]; BCOVPUILayoutView *logoLayoutView2 = [BCOVPUIBasicControlView layoutViewWithControlFromTag:BCOVPUIViewTagViewEmpty width:36.0 elasticity:0.0]; // Masquez les sous-titres codés et les contrôles AirPlay jusqu'à ce que vous en ayez explicitement besoin. closedCaptionLayoutView.removed = OUI; externalRouteLayoutView.removed = OUI;

Notez que vous pouvez également créer une vue de mise en page vide, dans laquelle vous pouvez placer votre propre vue (logo, champ, rien, etc.). Ce code montre comment placer un logo UIImage dans le logoLayoutView1 que nous avons créé ci-dessus.

// Crée une image de logo dans une vue d'image pour l'affichage dans la barre de contrôle. UIImage *logoImage1 = [UIImage imageNamed:@"myLogo"]; UIImageView *logoImageView1 = [[UIImageView alloc] initWithImage:logoImage1]; logoImageView1.autoresizingMask = UIViewAutoresizingFlexibleWidth | UIViewAutoresizingFlexibleHeight; logoImageView1.contentMode = UIViewContentModeScaleAspectFit; logoImageView1.frame = logoLayoutView1.frame; // Ajouter une vue d'image à notre vue de mise en page vide. [logoLayoutView1 addSubview:logoImageView1] ;

Maintenant que nous avons une collection de contrôles regroupés dans des vues de mise en page, nous pouvons les ordonner dans des tableaux, chacun représentant une seule ligne de contrôles, c'est-à-dire une barre de contrôle. Notez que vous pouvez avoir différentes dispositions pour les orientations portrait et paysage, vous devrez donc généralement configurer deux tableaux différents de barres de contrôle.

Dans cette disposition standard pour l'orientation paysage, nous organiserons nos contrôles sur une seule ligne, puis placerons ce tableau dans un autre tableau représentant l'ensemble de la configuration.

 NSArray *standardLayoutLine1 = @[ playbackLayoutView, jumpBackButtonLayoutView, currentTimeLayoutView, timeSeparatorLayoutView, durationLayoutView, progressLayoutView, spacerLayoutView1, logoLayoutView1, spacerLayoutView2, closedCaptionLayoutView, screenModeLayoutView, externalRouteLayout; NSArray *standardLayoutLines = @[ standardLayoutLine1 ];

Dans la disposition compacte, pour l'orientation portrait, nous créons deux tableaux de contrôles, puis ces deux tableaux sont regroupés dans un autre tableau représentant la disposition compacte.

Notez que nous avons utilisé exactement les mêmes objets pour la plupart des contrôles dans chaque disposition. Lorsque cela est fait et que vous basculez entre les orientations portrait et paysage, l'objet sera déplacé vers sa nouvelle position à l'aide d'une animation fluide.

 NSArray *compactLayoutLine1 = @[ currentTimeLayoutView, progressLayoutView, durationLayoutView ]; NSArray *compactLayoutLine2 = @[ playbackLayoutView, jumpBackButtonLayoutView, spacerLayoutView1, closedCaptionLayoutView, screenModeLayoutView, externalRouteLayoutView, logoLayoutView2 ]; NSArray *compactLayoutLines = @[ compactLayoutLine1, compactLayoutLine2 ];

Enfin, maintenant que nous avons deux configurations de mise en page, une pour la pleine largeur et une pour la largeur compacte, nous pouvons créer un nouveau BCOVPUIControlLayout objet, et placez-le dans le lecteur’s vue de contrôle.

 BCOVPUIControlLayout *customLayout = [[BCOVPUIControlLayout alloc] initWithStandardControls:standardLayoutLines compactControls:compactLayoutLines]; playerView.controlView.layout = customLayout;

Si vous avez des contrôles que vous devez afficher ou masquer fréquemment, vous pouvez définir le supprimé propriété sur ce contrôle’s vue mise en page. Lorsque vous avez modifié vos commandes, appelez setNeedsLayout sur le lecteurVoir’s controlView :

 logoLayoutView1.removed = OUI; [playerView.controlsView setNeedsLayout] ;

Vous pouvez également personnaliser plusieurs BCOVPUIControlLayout Propriétés:

  • controlBarHeight définit la taille de chaque ligne de contrôles.
  • horizontalItemSpacing définit l'espacement entre chaque BCOVPUILayoutView dans chaque barre de contrôle.
  • compactLayoutLargeurMaximum détermine quel ensemble de contrôles est utilisé. Si la vue de contrôle est plus petite que compactLayoutLargeurMaximum , le jeu de commandes compact sera utilisé, sinon les commandes standard seront utilisées.

Pour modifier l'ensemble des contrôles affichés, vous devez créer et installer un nouveau BCOVPUIControlLayout.

Questions fréquemment posées

Cela fonctionne-t-il avec tous les plug-ins Brightcove Player SDK ?

Cette version prend en charge la lecture VOD, Live et Live DVR de base sans publicité. Nous ajouterons la prise en charge d'autres plugins dans les prochaines versions.

Comment personnaliser des propriétés autres que celles mentionnées ci-dessus ?

Notre objectif est de créer quelque chose qui’est facile à personnaliser, tout en offrant de nombreuses fonctionnalités et comportements prêts à l'emploi. Dans les prochaines versions, nous couvrirons plus de personnalisation avec de nouvelles API et plus de documentation. S'il y a une fonctionnalité que vous recherchez, faites-le nous savoir dans notre forum.

Y aura-t-il des modifications majeures de l'API dans les futures versions ?

Nous ferons de notre mieux pour ne pas changer les choses, sauf si nous en avons vraiment besoin. Cependant, si nous devons apporter des modifications à l'API, nous fournirons des étapes de migration claires dans le CHANGELOG.
