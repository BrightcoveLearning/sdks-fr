Référence de catégorie BCOVCatalogService(BCOVWidevineImperativeCallbacks)

Déclaré en BCOVCatalogService+BCOVWidevineAdditions.h

Aperçu

La fonctionnalité des méthodes de recherche ajoutées dans ces méthodes de catégorie est similaire aux méthodes de recherche de base dans BCOVCatalogService, sauf que plutôt que de renvoyer un signal qui fournit les résultats (ou une erreur), ces méthodes renvoient void mais prennent un bloc qui est appelé avec les résultats (ou une erreur). Si vous êtes à l'aise avec ReactiveCocoa’s API de signal, les méthodes de signal proposées par BCOVCatalogService offrent une meilleure flexibilité. Ces méthodes de catégorie sont destinées aux clients qui ne souhaitent pas utiliser directement les signaux.

– findWidevinePlaylistWithPlaylistID:paramètres:achèvement :

Récupère une BCOVPlaylist du service Media API par son ID de liste de lecture.

- (void) findWidevinePlaylistWithPlaylistID:(NSString *)ID de la liste de lecture paramètres :(NSDictionary *)paramètres achèvement:(void ( ^ ) ( BCOVPlaylist *playlist , NSDictionary *jsonResponse , NSError *error ))achèvementGestionnaire

Paramètres

ID de la liste de lecture

L'ID de la liste de lecture à rechercher.
paramètres

Paramètres de requête NSString supplémentaires à ajouter aux requêtes de l'API Media. Ces valeurs remplaceront les valeurs par défaut en cas de conflit.
achèvementGestionnaire

bloc qui sera invoqué à la fin de la requête. L'exécution de l'achèvementHandler se produira sur le thread principal.

Discussion

Récupère une BCOVPlaylist du service Media API par son ID de liste de lecture.

Si l'API Media renvoie une réponse d'erreur au format JSON standard, la réponse désérialisée sera le rappel’erreur’s informations utilisateur.

Si une erreur d'analyse JSON se produit, la réponse NSData brute sera incluse dans le rappel’erreur’s informations utilisateur , saisi par kBCOVCatalogJSONDeserializationErrorRawDataKey.
