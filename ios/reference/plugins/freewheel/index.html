Installation

Le plugin FreeWheel pour Brightcove Player SDK fournit un cadre de bibliothèque statique pour l'installation. Un cadre dynamique sera ajouté à l'avenir lorsque FreeWheel publiera une version dylib.

Le plugin FreeWheel pour Brightcove Player SDK a été intégré à la version 6.38.0 du framework FreeWheel AdManager qui est disponible à partir du Téléchargements d'intégration du SDK AdManager page.

CocoaPods

Vous pouvez utiliser CocoaPods pour ajouter le plug-in FreeWheel pour Brightcove Player SDK à votre projet. Vous pouvez trouver les dernières Brightcove-Player-FreeWheel podspec ici. Pour utiliser cette spécification, ajoutez ce qui suit en haut de Podfile : la source 'https://github.com/brightcove/BrightcoveSpecs.git'. CocoaPods version 1.0 ou supérieure est requis.

Le SDK FreeWheel n'est pas inclus dans ce module. Tu doit ajoutez manuellement le FreeWheel SDK AdManager.framework à votre projet. Le pod ajoutera cependant toutes les bibliothèques requises par le framework AdManager.framework.

Exemple de cadre statique :

source 'https://github.com/brightcove/BrightcoveSpecs.git' pod 'Brightcove-Player-FreeWheel'

Lors de la mise à jour de votre installation, il’C'est une bonne idée d'actualiser la copie locale de votre référentiel BrightcoveSpecs afin que vous disposiez des derniers podspecs localement, tout comme vous mettriez à jour votre référentiel maître CococaPods.

Généralement si vous utilisez mise à jour du pod dans Terminal, cela se produira automatiquement, ou vous pouvez également mettre à jour explicitement avec pod repo mise à jour brightcove. (Votre référentiel BrightcoveSpecs peut avoir un nom différent si vous l'avez explicitement ajouté à votre liste de référentiels podspecs.)

Manuel

Pour ajouter manuellement le plug-in FreeWheel pour Brightcove Player SDK à votre projet :

  1. Installez la dernière version du Kit de développement logiciel Brightcove Player et suivez les instructions d'installation du [Cadre statique][https://github.com/brightcove/brightcove-player-sdk-ios/#InstallationManuelle] .
  2. Téléchargez la dernière version zippée du plugin depuis notre page de publication.
  3. Ajouter le ‘BrightcoveFW.framework’ à votre projet. Vous pouvez le faire en cliquant avec le bouton droit sur le dossier Frameworks et choisissez “Ajouter des fichiers à” et sélectionnez le BrightcoveFW.framework à partir du chemin où il est stocké.
  4. Sur le “Paramètres de construction” de l'onglet cible de votre application, assurez-vous que le “Chemins de recherche du framework” inclure le chemin d'accès au cadre. Cela aurait dû être fait automatiquement à moins que le framework ne soit stocké dans un répertoire racine différent de celui de votre projet.
  5. Sur le “Phases de construction” de l'onglet cible de votre application, ajoutez ce qui suit à la “Lier le binaire avec les bibliothèques” phase:
    • BrightcoveFW.framework
  6. Sur le “Paramètres de construction” onglet de votre application cible :
    • Veiller à ce que -ObjC a été ajouté au “Autres indicateurs de liens” paramètre de construction.
  7. Installez la bibliothèque FreeWheel, qui doit être récupérée à partir de votre compte FreeWheel.

Importations

Le plugin FreeWheel pour Brightcove Player SDK peut être importé dans le code de différentes manières ; @import BrightcoveFW; , #import <BrightcoveFW/BrightcoveFW.h> ou #import <BrightcoveFW/[classe spécifique].h>.

Lecture et pause

Le plugin Brightcove FreeWheel implémente une logique de lecture et de pause personnalisée pour garantir l'expérience publicitaire la plus fluide possible. Par conséquent, vous devrez vous assurer que vous utilisez la méthode de lecture sur le Contrôleur de lecture BCOV ou la [BCOVSessionProviderExtension fw_play] ou [BCOVSessionProviderExtension fw_pause] (BCOVSessionProviderExtension), et non l'AVPlayer.

Par exemple, appeler play pour la première fois sur Contrôleur de lecture BCOV permet au plugin Brightcove FreeWheel de traiter les annonces de pré-roll sans qu'aucun contenu ne soit lu avant le pré-roll.

Utilisation du PlayerUI intégré

Si vous utilisez la version 5.1 ou une version ultérieure du Brightcove Player SDK, vous pouvez profiter des commandes publicitaires intégrées avec le plug-in Brightcove FreeWheel.

Pour utiliser PlayerUI, créez votre contrôleur de lecture comme dans le code de démarrage rapide ci-dessus.

BCOVPlayerSDKManager *sdkManager = [BCOVPlayerSDKManagersharedManager] ; id<BCOVPlaybackController> playbackController = [sdkManager createFWPlaybackControllerWithAdContextPolicy:[self adContextPolicy] viewStrategy:nil];

Créez ensuite votre vue joueur. Cette vue contient à la fois la vue du lecteur vidéo et la vue qui affiche les commandes de lecture et de publicité. Cette configuration est la même quel que soit le plugin que vous utilisez.

Noter: Les BrightcovePlayerUI module n'est plus nécessaire et a été supprimé. (Avant la version 5.1 du Brightcove Player SDK, le plug-in Brightcove PlayerUI était un framework et un module distincts.) Vous pouvez supprimer toutes les importations faisant référence au module Brightcove PlayerUI. Tous les en-têtes PlayerUI se trouvent maintenant dans le BrightcovePlayerSDK module.

Créez une propriété dans votre UIViewController pour garder une trace de BCOVPUIPlayerView. Le BCOVPUIPlayerView contiendra à la fois le contrôleur de lecture’la vue s et la vue des commandes.

// Vue du joueur de PlayerUI @property (non atomique) BCOVPUIPlayerView *playerView;

Créez le BCOVPUIBasicControlView, puis le BCOVPUIPlayerView. C'est là que nous associons le contrôleur de lecture (et donc toutes les vidéos qu'il lit) aux commandes.

Personnalisation du comportement du plugin

Vous pouvez personnaliser le comportement du plugin par défaut en créant une instance de BCOVFWSessionProviderOptions et en remplaçant les propriétés par défaut. Pour utiliser un BCOVFWSessionProviderOptions l'instance d'options, vous devez créer le BCOVFWSessionProvider à l'aide de [BCOVSDKManager createFWSessionProviderWithAdContextPolicy:upstreamSessionProvider:options:].

 BCOVFWSessionProviderOptions *options = [[BCOVFWSessionProviderOptions alloc] init]; options.cuePointProgressPolicy = [BCOVCuePointProgressPolicy progressPolicyProcessingCuePoints:BCOVProgressPolicyProcessFinalCuePoint resumingPlaybackFrom:BCOVProgressPolicyResumeFromContentPlayhead ignoringPreviouslyProcessedCuePoints:YES]; id<BCOVPlaybackSessionProvider> sessionProvider = [sdkManager createFWSessionProviderWithAdContextPolicy:[self adContextPolicy] amontSessionProvider:nil options:options]; id<BCOVPlaybackController> playController = [sdkManager createPlaybackControllerWithSessionProvider:sessionProvider viewStrategy:nil];

Emplacements de préchargement

Selon les conditions du réseau, il peut y avoir un délai dans le temps nécessaire pour que les annonces soient diffusées, une fois que la position de l'annonce a été atteinte. Afin d'améliorer les performances des annonces, il est possible de précharger les emplacements FreeWheel. Pour activer le préchargement, activez preloadSlots au BCOVFWSessionProviderOptions.

Lorsqu'il est activé, les emplacements de pré-roll seront chargés dès que possible. Pour les midrolls, les emplacements seront préchargés pendant un nombre de secondes configurable, déterminé par timeBeforeAdToPreload au BCOVFWSessionProviderOptions.

Utiliser une durée précise

En fonction de votre configuration FreeWheel, fournir une durée précise des appels vers setVideoId : est la différence entre recevoir le nombre correct de midrolls ou non.

Par défaut, le SDK attendra pour appeler le BCOVFWSessionProviderAdContextPolicy jusqu'à ce que l'AVPlayerItem ait une durée précise, et cette valeur sera fournie à l'appel de bloc. Si vous ne voulez pas attendre que la durée d'AVPlayerItem soit prête et que vous ayez ces informations disponibles à partir d'une autre source (comme les propriétés BCOVVideo/BCOVSource ou votre propre CMS), waitForAVPlayerDurée peut être utilisé pour désactiver l'attente.

Traitement des recherches

Lors de la recherche sur plusieurs modules publicitaires (comme deux emplacements midrolls, à des positions différentes), le SDK fournit un mécanisme pratique pour déterminer quels modules sont lus. Cela peut être modifié en changeant la valeur par défaut BCOVCuePointProgressPolicy au BCOVFWSessionProviderOptions. Pour plus d'informations sur le BCOVCuePointProgressPolicy , veuillez consulter le BCOVFWSessionProvider.h déposer.

Suivi des erreurs FreeWheel

Événements d'erreur FreeWheel (FWAdErrorEvent ) rencontrés via le FWAdEventNotification sera transmis au playbackController:playbackSession:didReceiveLifecycleEvent : méthode de délégation avec un type d'événement de kBCOVFWLifecycleEventAdError. L'intégralité de l'objet userInfo que le FreeWheel SDK envoie avec la notification est inclus dans les propriétés des événements et est accessible avec le kBCOVFWLifecycleEventPropertyKeyAdError clé.

Voici un exemple de la façon de détecter ces erreurs :

- (void)playbackController:(id<BCOVPlaybackController>)contrôleur playbackSession:(id<BCOVPlaybackSession>)session didReceiveLifecycleEvent:(BCOVPlaybackSessionLifecycleEvent *)lifecycleEvent { if ([kBCOVFWLifecycleEventAdError isEqualToString:lifecycleEvent.eventType]) { NSDictionary *errorInfo = lifecycleEvent.properties[kBCOVFWLifecycleEventPropertyKeyAdError]; NSLog(@"FreeWheel Error Encountered: %@", errorInfo[FWInfoKeyErrorInfo]); } }
