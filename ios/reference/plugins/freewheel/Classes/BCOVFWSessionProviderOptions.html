  waitForAVPlayerDurée

Attendre ou non que l'élément AVPlayer signale une durée avant d'appeler adContextPolicy. Lorsqu'elle est définie sur NON, la durée est passée dans le BCOVFWSessionProviderAdContextPolicy bloc peut ne pas être précis. La valeur par défaut est Oui.

@property (nonatomique, assign, getter=shouldWaitForAVPlayerDuration) BOOL waitForAVPlayerDuration

Discussion

Attendre ou non que l'élément AVPlayer signale une durée avant d'appeler adContextPolicy. Lorsqu'elle est définie sur NON, la durée est passée dans le BCOVFWSessionProviderAdContextPolicy bloc peut ne pas être précis. La valeur par défaut est Oui.

  adTimeoutInterval

Le nombre de secondes pendant lesquelles le fournisseur de session attendra une demande d'annonce avant de livrer une session sans utiliser d'annonces. La valeur par défaut est de 3 secondes.

Discussion

  cuePointProgressPolicy

Une règle qui détermine si une publicité FreeWheel doit être lue lorsqu'un événement de point de repère donné se produit et, le cas échéant, où la lecture doit reprendre. La valeur par défaut est BCOVProgressPolicyProcessFinalCuePoint, BCOVProgressPolicyResumeFromContentPlayhead, YES.

@property (non atomique, fort) BCOVCuePointProgressPolicy *cuePointProgressPolicy

Discussion

Une règle qui détermine si une publicité FreeWheel doit être lue lorsqu'un événement de point de repère donné se produit et, le cas échéant, où la lecture doit reprendre. La valeur par défaut est BCOVProgressPolicyProcessFinalCuePoint, BCOVProgressPolicyResumeFromContentPlayhead, YES.

  preloadSlots

Détermine si les emplacements sont préchargés ou non afin qu'ils soient prêts à jouer lorsqu'ils sont touchés. Lorsqu'il est défini sur OUI, les emplacements de pré-roll se chargeront dès que possible. Les slots Midrolls se chargeront X secondes avant que le slot ne soit joué, déterminé par timeBeforeAdToPreload. La valeur par défaut est NON.

@property (nonatomic, assign, getter=shouldPreloadSlots) BOOL preloadSlots

Discussion

Détermine si les emplacements sont préchargés ou non afin qu'ils soient prêts à jouer lorsqu'ils sont touchés. Lorsqu'il est défini sur OUI, les emplacements de pré-roll se chargeront dès que possible. Les slots Midrolls se chargeront X secondes avant que le slot ne soit joué, déterminé par timeBeforeAdToPreload. La valeur par défaut est NON.

  timeBeforeAdToPreload

Si le préchargement de l'espace publicitaire est activé, il s'agit du nombre de secondes avant que l'annonce ne la précharge. La valeur par défaut est 10 secondes.

@property (non atomique, assigner) NSTimeInterval timeBeforeAdToPreload

Discussion

Si le préchargement de l'espace publicitaire est activé, il s'agit du nombre de secondes avant que l'annonce ne la précharge. La valeur par défaut est 10 secondes.
