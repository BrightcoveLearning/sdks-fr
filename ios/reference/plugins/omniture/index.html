Référence Brightcove Native Player pour iOS

Plugin Omniture (Adobe Marketing Cloud - Adobe Analytics) pour Brightcove Player SDK pour iOS, version 6.9.0.1697

Conditions

  • Xcode 11.0+
  • ARC

Plateformes prises en charge

iOS 11.0 et supérieur.

Installation

Le plugin Omniture est un framework de bibliothèque statique qui prend en charge Adobe Marketing Cloud v4.14.1 et Adobe Video Heartbeat v2.0.1. Les bibliothèques Marketing Cloud et Video Heartbeat ne sont pas inclus avec ce SDK et doit être ajouté manuellement à votre projet. Les instructions pour télécharger les bibliothèques sont fournies ci-dessous.

CocoaPods

Vous pouvez utiliser CocoaPods version 1.0 ou supérieure pour ajouter le plugin Omniture à votre projet. Vous pouvez trouver les dernières Brightcove-Player-Omniture podspec ici. Pour utiliser cette spécification, ajoutez ce qui suit en haut de votre fichier Podfile : la source 'https://github.com/brightcove/BCOVSpecs.git'.

Exemple de cadre statique :

```source bash ‘https://github.com/brightcove/BrightcoveSpecs.git'

cible ‘MonOmniturePlayer’ faire la cosse ‘Brightcove-Player-Omniture’ fin ```

Lors de la mise à jour de votre installation, il’C'est une bonne idée d'actualiser la copie locale de votre référentiel BrightcoveSpecs afin que vous disposiez des derniers podspecs localement, tout comme vous mettriez à jour votre référentiel maître CococaPods.

Généralement si vous utilisez mise à jour du pod dans Terminal, cela se produira automatiquement, ou vous pouvez également mettre à jour explicitement avec mise à jour du dépôt de pod.

Manuel

Pour ajouter manuellement le plug-in Omniture pour Brightcove Player SDK à votre projet :

  1. Installez la dernière version du Kit de développement logiciel Brightcove Player.
  2. Téléchargez la dernière version zippée du plugin à partir du page de publication.
  3. Ajouter le BrightcoveAMC.framework projet. Vous pouvez le faire en cliquant avec le bouton droit sur le dossier Frameworks et choisissez “Ajouter des fichiers à” option et sélectionnez le BrightcoveAMC.framework du chemin où il est stocké.
  4. Sur le “Paramètres de construction” de l'onglet cible de votre application, assurez-vous que le “Chemins de recherche du framework” inclure le chemin d'accès au cadre. Cela aurait dû être fait automatiquement à moins que le framework ne soit stocké dans un répertoire racine différent de celui de votre projet.
  5. Sur le “Phases de construction” de l'onglet cible de votre application, ajoutez ce qui suit à la “Lier le binaire avec les bibliothèques” phase:
    • BrightcoveAMC.framework
  6. Sur le “Paramètres de construction” onglet de votre application cible :
    • Veiller à ce que -ObjC a été ajouté au “Autres indicateurs de liens” paramètre de construction.
  7. Téléchargez Adobe Marketing Cloud en suivant Adobe’s instructions et ajouter AdobeMobileLibrary.a à votre projet.
  8. Téléchargez Adobe Video Heartbeat en suivant Adobe’s instructions et ajouter VideoHeartbeat.a à votre projet.
  9. Ajoutez les dossiers d'en-tête Marketing Cloud et Video Heartbeat aux paramètres du chemin de recherche d'en-tête de votre projet.

Démarrage rapide

BrightcoveAMC est un pont entre le Kit de développement logiciel Brightcove Player pour iOS et le suivi vidéo Adobe, y compris Adobe Video Heartbeat v2 et Suivi des médias Adobe v4. BrightcoveAMC suit les comportements vidéo de base : vidéo prête, vidéo terminée, lecture vidéo et pause vidéo.

Implémentation client

Pour configurer Adobe Video Heartbeat, les clients Omniture Plugin doivent implémenter des instances de BCOVAMCVideoHeartbeatConfigurationPolicy. Pour configurer Adobe Media Tracking, une instance de BCOVAMCMediaSettingPolicy est requise. Ces Stratégies permettre la personnalisation des objets ADBMediaHeartbeatConfig ou ADBMediaSettings pour chaque session de lecture.

Vidéo Heartbeat v2

Cet exemple utilise le suivi des pulsations vidéo.

[1] // Créer l'objet de stratégie de configuration VHB. BCOVAMCVideoHeartbeatConfigurationPolicy videoHeartbeatConfigurationPolicy = ^ADBMediaHeartbeatConfig *(id<BCOVPlaybackSession> session) { ADBMediaHeartbeatConfig *configData = [[ADBMediaHeartbeatConfig alloc] init]; configData.trackingServer = <adobe-assigned_tracking_server>"; configData.channel = <cutomize_sample_channel>; configData.appVersion = <app_version>; configData.ovp = <online_video_platform>; configData.playerName = <player_name>; configData.ssl = <YES | NO>; // Set debugLogging to true to activate debug tracing. Remove it in production. configData.debugLogging = YES; return configData; } ; [2] // Créez l'objet de politique d'analyse Brightcove AMC à partir de w1w l'objet de politique de configuration VHB. BCOVAMCAnalyticsPolicy *heartbeatPolicy = [[BCOVAMCAnalyticsPolicy alloc] initWithHeartbeatConfigurationPolicy:videoHeartbeatConfigPolicy]; BCOVAMCSessionConsumer *sessionConsumer = [BCOVAMCSessionConsumer heartbeatAnalyticsConsumerWithPolicy:heartbeatPolicy délégué:self]; BCOVPlayerSDKManager *manager = [BCOVPlayerSDKManagersharedManager] ; id<BCOVPlaybackController> contrôleur = [gestionnaire createPlaybackController]; contrôleur.delegate = self; [self.view addSubview:controller.view] ; [3] // Ajoutez le consommateur de session Brightcove AMC au contrôleur de lecture. [contrôleur addSessionConsumer:sessionConsumer] ; [4] // Recherche et lecture d'une vidéo. NSString *policyKey = <votre-clé-politique> ; NSString *accountId = <your-account-id>; NSString *videoID = <votre-vidéo-id> ; BCOVPlaybackService *service = [[BCOVPlaybackService alloc] initWithAccountId:accountId policyKey:policyKey]; [service findVideoWithVideoID:paramètres videoID:nil achèvement :^(BCOVVideo *video, NSDictionary *jsonResponse, NSError *error) { [controller setVideos:@[ video ]]; [controller play]; } ;
  1. Créez le bloc de politique de configuration de pulsation vidéo qui sera appelé au début de chaque session de lecture. La stratégie permet la personnalisation des données de configuration en fonction de la session en cours.
  2. Utilisez le bloc de stratégie de configuration pour créer et initialiser une instance BCOVAMCAnalyticsPolicy avec -[initWithHeartbeatConfigurationPolicy :]. L'objet BCOVAMCAnalyticsPolicy est utilisé pour créer le BCOVAMCSessionConsumer.
  3. Une fois le contrôleur de lecture créé, appelez la méthode consommateur d'ajout de session, -[addSessionConsumer :] , pour ajouter le consommateur de session AMC.

Nuage de marketing

Cet exemple utilise le suivi des médias.

[1] BCOVAMCMediaSettingPolicy mediaSettingPolicy = ^ADBMediaSettings *(id<BCOVPlaybackSession> session) { ADBMediaSettings *settings = [ADBMobile mediaCreateSettingsWithName:<cutomize_setting_name> [2] length:0 playerName:<cutomize_player_name> playerID:<cutomize_player_ID>]; [3] settings.milestones = @"25,50,75"; // a customization. return settings; } ; [4] BBCOVAMCAnalyticsPolicy *mediaPolicy = [[BCOVAMCAnalyticsPolicy alloc] initWithMediaSettingsPolicy:mediaSettingPolicy] ; BCOVAMCSessionConsumer *sessionConsumer = [BCOVAMCSessionConsumer mediaAnalyticsConsumerWithPolicy:mediaPolicy délégué:self]; BCOVPlayerSDKManager *manager = [BCOVPlayerSDKManagersharedManager] ; id<BCOVPlaybackController> contrôleur = [gestionnaire createPlaybackController]; contrôleur.delegate = self; [self.view addSubview:controller.view] ; [5] [contrôleur addSessionConsumer:sessionConsumer] ; NSString *policyKey = <votre-clé-politique> ; NSString *accountId = <your-account-id>; NSString *videoID = <votre-vidéo-id> ; BCOVPlaybackService *service = [[BCOVPlaybackService alloc] initWithAccountId:accountId policyKey:policyKey]; [service findVideoWithVideoID:paramètres videoID:nil achèvement :^(BCOVVideo *video, NSDictionary *jsonResponse, NSError *error) { [controller setVideos:@[ video ]]; [controller play]; } ;
  1. Créez le bloc de stratégie des paramètres multimédias qui sera appelé au début de chaque session de lecture. La stratégie permet la personnalisation des paramètres multimédias en fonction de la session en cours.
  2. Lors de la création de l'instance ADBMediaSettings avec une méthode de classe ADBMobile +[mediaCreateSettingsWithName:length:playerName:playerID:] , vous pouvez définir durée de la vidéo à 0. Le plugin Omniture le mettra à jour plus tard.
  3. Ajoutez des paramètres facultatifs tels que des jalons.
  4. Utilisez le bloc de stratégie des paramètres multimédias pour créer et initialiser une instance BCOVAMCAnalyticsPolicy avec -[initWithMediaSettingsPolicy :]. L'objet de stratégie est utilisé pour créer un consommateur de session AMC pour le suivi des médias Adobe en tant que +[mediaAnalyticsConsumerWithPolicy:delegate:].
  5. Une fois le contrôleur de lecture créé, appelez la méthode consommateur d'ajout de session, -[addSessionConsumer :] , pour ajouter le consommateur de session AMC.
