iOS 11.0 et supérieur.

Installation

Le plug-in Google Cast pour Brightcove Player SDK fournit un cadre de bibliothèque statique pour l'installation.

Le plug-in Google Cast prend en charge la version 4.5.0 du SDK Google Cast pour iOS. Les notes de version de Google Cast peuvent être trouvées ici.

CocoaPods

Vous pouvez utiliser CocoaPods pour ajouter le plug-in Google Cast pour Brightcove Player SDK à votre projet. Vous pouvez trouver les dernières spécifications de pod Brightcove-Player-GoogleCast ici. Le pod incorporera automatiquement la version correcte de GoogleCast. CocoaPods 1.0 ou plus récent est requis.

Exemple de Podfile CocoaPod :

```source bash ‘https://github.com/CocoaPods/Specs' la source ’https://github.com/brightcove/BrightcoveSpecs.git'

plateforme :ios, ‘11,0’

use_frameworks !

cible ‘Exemple d'application’ faire la cosse ‘Brightcove-Player-GoogleCast’ fin ```

L'installation de Brightcove-Player-GoogleCast installe automatiquement la dépendance Brightcove-Player-Core. En fonction de vos besoins, vous pouvez choisir d'inclure soit la structure dynamique Brightcove-Player-Core, soit la structure statique. Par défaut, le framework Core dynamique est installé. Pour choisir le framework Core statique, votre Podfile ressemblerait à ceci :

```source bash ‘https://github.com/CocoaPods/Specs’ la source ‘https://github.com/brightcove/BrightcoveSpecs.git’

plateforme :ios, ‘11,0’

use_frameworks !

cible ‘Exemple d'application’ faire la cosse ‘Brightcove-Player-GoogleCast-statique’ fin ```

Lors de la mise à jour de votre installation, il’C'est une bonne idée d'actualiser la copie locale de votre référentiel BrightcoveSpecs pour vous assurer que vous disposez des derniers podspecs localement, tout comme vous le feriez pour mettre à jour votre référentiel maître CococaPods. Utilisation mise à jour du dépôt de pod faire cela.

Manuel

Pour ajouter manuellement le plug-in Google Cast pour Brightcove Player SDK à votre projet :

  1. Suivre la Configuration manuelle du SDK Google Cast guider.
  2. Suivre la Installation manuelle du kit de développement logiciel Brightcove Player guider.
  3. Téléchargez le Plugin Google Cast pour le SDK Brightcove Player cadre.
  4. Sur le “Général” onglet de votre application cible, ajoutez BrightcoveGoogleCast.framework à partir du téléchargement du plug-in Google Cast pour Brightcove Player SDK à la liste des Frameworks, bibliothèques, contenu intégré.
  5. Sur le “Paramètres de construction” de l'onglet cible de votre application, assurez-vous que le “Chemins de recherche du framework” inclure les chemins vers les cadres. Cela aurait dû être fait automatiquement à moins que le framework ne soit stocké dans un répertoire racine différent de celui de votre projet.

Importations

Le plug-in Google Cast pour Brightcove Player SDK peut être importé dans le code de différentes manières ; @import BrightcoveGoogleCast ; , #import <BrightcoveGoogleCast/BrightcoveGoogleCast.h> ou #import <BrightcoveGoogleCast/[classe spécifique].h>. Vous pouvez importer le Google Cast et BrightcovePlayerSDK modules de manière similaire.

Avant que tu commences

Avant d'essayer d'utiliser ce plugin, vous devez déjà être familiarisé avec les éléments suivants :

  • Enregistrez votre candidature. Cela vous guidera tout au long de la création de votre application destinataire et vous accordera un ID d'application. Pour commencer, vous pouvez utiliser l'ID de l'application de démonstration ‘4F8B3483’.
  • Configuration pour le développement avec Cast Application Framework (CAF) pour iOS . Cela vous guidera tout au long de la configuration des droits nécessaires pour votre application.
  • Initialiser le contexte de diffusion. Cela vous guidera tout au long de l'initialisation du contexte de diffusion avec l'ID de votre application réceptrice.
  • Ajouter des mini-contrôleurs. Cela vous expliquera comment tirer parti du SDK Google Cast’s mini commandes multimédia.

Démarrage rapide

Le plugin BrightcoveGoogleCast est un pont entre SDK Google Cast pour iOS et le Kit de développement logiciel Brightcove Player pour iOS.

Cet extrait montre son utilisation de base.

[1] self.googleCastManager = [BCOVGoogleCastManager nouveau] ; [2] id<BCOVPlaybackController> playbackController = [BCOVPlayerSDKManager.sharedManager createPlaybackController]; [3] [playbackController addSessionConsumer:self.googleCastManager] ;

Décomposer le code en étapes :

  1. Créer une référence au BCOVGoogleCastManager singleton.
  2. Créez une instance de BCOVPlaybackController à utiliser.
  3. Ajoutez votre BCOVGoogleCastManager référence en tant que consommateur de session à votre instance BCOVPlaybackController.

Récepteur Brightcove CAF

L'ID d'application pour le récepteur Brightcove CAF est 341387A3 et est affecté à la constante kBCOVCAFReceiverApplicationID. Vous pouvez vérifier l'ID de l'application en vérifiant le Récepteur CAF config.json.

Si vous utilisez le récepteur Brightcove CAF, vous’ll besoin d'initialiser ces variables comme ceci :

[1] BCOVReceiverAppConfig *receiverAppConfig = [BCOVReceiverAppConfig nouveau] ; receiverAppConfig.accountId = kAccountID; receiverAppConfig.policyKey = kServicePolicyKey; [2] self.googleCastManager = [[BCOVGoogleCastManager alloc] initForBrightcoveReceiverApp:receiverAppConfig] ;

Les propriétés suivantes peuvent également être définies sur BCOVReceiverAppConfig comme requis:

  • splashScreen (pour personnaliser l'image de l'écran de démarrage)
  • playerUrl (pour utiliser un lecteur personnalisé)
  • authToken (à utiliser avec PAS/EPA)
  • watermarkingToken (à utiliser avec Forensic Watermaking)
  • adConfigId (à utiliser avec SSAI)
  • userId (à utiliser avec le suivi analytique)
  • applicationId (à utiliser avec le suivi analytique)

Déléguer les méthodes

BCOVGoogleCastManagerDélégué a des méthodes déléguées que vous pouvez utiliser pour être averti lorsque des événements majeurs liés à la distribution se sont produits ou sont sur le point de se produire, par exemple

  • - (void)switchedToLocalPlayback :(NSTimeInterval)lastKnownStreamPosition ;
  • - (vide)basculé vers la lecture à distance ;
  • - (void)currentCastedVideoDidComplete ;
  • - (void)castedVideoFailedToPlay ;
  • - (void)appropriéSourceNotFound;
  • - (void)willBuildMediaInformationBuilder:(GCKMediaInformationBuilder *_Nonnull);
  • - (void)willSendMediaLoadOptions:(GCKMediaLoadOptions *_Nonnull);

Pour une liste complète avec des descriptions, reportez-vous au Référence du protocole BCOVGoogeCastManagerDelegate.

Pour profiter de ces événements, placez un délégué sur le BCOVGoogleCastManager singleton.

 self.googleCastManager.delegate = self;

Sélection de la source

Les BCOVGoogleCastManager tentera de trouver une source appropriée à utiliser. Il recherchera les sources dans cet ordre :

  1. HTTPS HLS v3
  2. TABLEAU DE BORD HTTPS
  3. HTTPS MP4
  4. HTTP HLS v3
  5. DASH HTTP
  6. HTTP MP4

Si aucun de ces éléments n'est trouvé, la méthode déléguée appropriéSourceNotFound sera appelé.

Votre compte Video Cloud devra être configuré pour prendre en charge HLS v3, assurez-vous que DASH est activé ou qu'une source MP4 soit disponible pour chaque vidéo.

Personnalisations

Il existe deux propriétés, en plus de la propriété de délégué, que vous pouvez définir sur le BCOVGoogleCastManager classer. Ceux-ci sont:

  • GCKImage *fallbackPosterImage: Le GCKImage qui sera utilisé lorsqu'il n'y a pas d'image d'affiche disponible pour une vidéo.
  • CGSize posterImageSize: La hauteur et la largeur que vous souhaitez utiliser pour l'image d'objet GCKImage qui est créée. La valeur par défaut est 480h x 720w.

Problèmes connus / Limitations

Lors de l'utilisation d'un récepteur par défaut non modifié (y compris le récepteur de démonstration), les limitations suivantes s'appliquent :

  • Les DRM ne sont pas pris en charge.
  • Les pistes audio multiples ne sont pas prises en charge.
  • Les publicités côté client et côté serveur ne sont pas prises en charge.
  • Les flux DVR en direct et en direct ne sont pas pris en charge.

Lors de l'utilisation du récepteur Brightcove CAF, les limitations suivantes s'appliquent :

  • La publicité côté client n'est pas prise en charge.
