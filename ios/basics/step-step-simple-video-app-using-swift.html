    Étape par étape : Application vidéo simple utilisant Swift

    Dans cette rubrique, vous apprendrez comment créer une application de lecture vidéo simple à l'aide du SDK Brightcove Player pour iOS. Vous allez le construire en utilisant le langage de programmation Swift.

    Présentation

    Cet exemple montre comment lire une vidéo renvoyée à partir de l' API Lecture Brightcove, qui est l'API la plus récente et recommandée pour récupérer du contenu de votre bibliothèque Video Cloud.

    Pour cette option, vous aurez besoin d'une clé de stratégie. Si vous ne connaissez pas les clés de stratégie, reportez-vous à la section Présentation de l'API de stratégie.

    Cet exemple est écrit dans Swift, mais vous pouvez également expérimenter avec Objective-C. Voir ce qui suit :
    • Étape par étape። Application vidéo simple utilisant Obj-C
    • Version Obj-C du VideoCloudBasicPlayer

    Démarrer

    Cet exemple lit des vidéos à partir d'un tableau interne de fichiers vidéo.

    Suivez les étapes ci-dessous pour vous familiariser avec la configuration d'un projet d'application qui utilise le SDK Brightcove Player pour iOS. Vous pouvez afficher le code complet pour chacun des éléments suivants :

    • AppDelegate

    • Contrôleur View

    Il y a deux façons d'essayer cet exemple :

    Télécharger l'exemple

    Téléchargez l'intégralité du projet Xcode pour l'expérimentation.

    1. Clonez ou téléchargez les exemples de SDK natif pour iOS sur votre système local.
    2. Accédez à l'exemple d'application Player/VideoCloudBasicPlayer/SWIFT .
    3. Exécutez la pod install commande.
    4. Ouvrez et exécutez l'espace de travail nouvellement créé.

    Pour plus d'informations, consultez le document Exécuter des exemples d'applications pour le SDK natif pour iOS/tVOS .

    Créez l'application en suivant les étapes décrites dans ce guide

    Suivez les étapes ci-dessous pour vous familiariser avec la configuration d'un projet d'application qui utilise le SDK Brightcove Player pour tvOS. Vous pouvez afficher le code complet pour chacun des éléments suivants :

    1. AppDelegate

    2. ViewController

    Créer le projet

    Configurez un projet dans Xcode. Ensuite, ajoutez le SDK avec toutes les dépendances au projet.

    Configurer le projet Xcode

    Créez un nouveau projet Xcode pour l'application.

    1. Commencez la création d'un nouveau projet iOS dans Xcode. Sélectionnez Créer un nouveau projet Xcode.

    2. Sélectionnez iOS et Application Single View pour votre nouveau projet. Sélectionnez Suivant.

    3. Définissez les informations du projet comme suit :

      • Nom du produit : Lecture vidéo simple
      • Équipe : aucune

        Le champ Équipe est facultatif. Si vous souhaitez distribuer votre application sur l'App Store, vous devrez sélectionner une équipe à laquelle vous appartenez via le programme Apple Developer. Le nom de l'équipe est requis pour la signature de code. Dans cet exemple, nous allons sélectionner aucun.

      • Nom de l'organisation : nom de votre entreprise
      • Identificateur d'organisation : com.example-company
        Cela rend le nom du produit unique pour l'App Store.
      • Langue : Rapide
      • Interface utilisateur : Storyboard

        Dans cet exemple, nous utilisons l'interface storyboard.

    4. Choisissez l'emplacement où vous souhaitez enregistrer votre projet et le créer.

    5. Maintenant, fermez le projet (oui, fermez-le — c'est important !)

    Ajouter le SDK et ses dépendances à votre projet

    Le moyen le plus simple d'ajouter le SDK et ses dépendances à votre projet est d'utiliser CocoaPods.

    CocoaPods est un gestionnaire de dépendances qui ajoute des bibliothèques à votre projet. Ce n'est pas nécessaire, mais facilite l'installation. Pour installer CocoaPods, consultez les instructions sur le site CocoaPods .

    • CocoaPods version 1.0 ou ultérieure est requise.
    • L'utilisation de CocoaPods (l' pod outil) sur un projet Xcode existant modifiera le fichier de projet. Vous voudrez peut-être faire une sauvegarde avant de le faire.

    1. Dans votre dossier de projet, créez un fichier texte brut appelé Podfile (pas d'extension de fichier).

    2. À l'aide d'un éditeur de texte, ajoutez les lignes de code suivantes au fichier Podfile et enregistrez-le. Ce code effectue les opérations suivantes :

      • Ligne 1 : Pointe vers l'emplacement GitHub pour les spécifications du module Brightcove.
      • Ligne 3 : Définit Pods pour utiliser des frameworks au lieu des bibliothèques statiques.
      • Ligne 5 : Définit la version de la plate-forme iOS.

      • Lignes 7 à 9 : Installez le SDK Brightcove Native Player.

        1. Obtenez la version actuelle du SDK natif à partir du document Présentation de Brightcove Native SDK pour iOS .
        2. Remplacez la valeur '6.7.10' par le numéro de version actuel du SDK.

        3. Pour plus d'informations sur la gestion des dépendances, consultez l'article How To Manage Dependencies with CocoaPods .

    3. Ouvrez une session Terminal et accédez à votre dossier de projet Xcode Simple Video-Lecture .

    4. Dans la session Terminal, tapez la commande

      pod install

      et appuyez sur Retour pour exécuter le it.

      Si CocoaPods ne trouve pas de version compatible pour le module, ou si ce n'est pas la dernière version, essayez d'exécuter ce qui suit dans l'application Terminal :

      pod update

      Vous devriez voir une série de messages dans le terminal indiquant que le SDK Brightcove Player a été ajouté à votre projet.

    5. La pod install commande crée un .xcworkspace fichier pour votre projet.

      À partir de ce moment, vous devez ouvrir le fichier Simple Video-Playback.xcWorkspace dans Xcode, pas le fichier Simple Video-Playback.xcodeProj .

    • Si vous créez et modifiez régulièrement des fichiers podfiles, vous pouvez essayer d'utiliser l' application CocoaPods pour effectuer les étapes précédentes.
    • Si vous ne souhaitez pas utiliser CocoaPods, vous pouvez télécharger la dernière version du SDK et l'ajouter manuellement à votre projet.

    Définir le comportement audio de l'application

    La session audio gère le comportement audio au niveau de l'application. En savoir plus sur la classe AVAudioSession .

    Vous pouvez choisir parmi plusieurs catégories et paramètres de session audio pour personnaliser le comportement audio de votre application. Choisissez la catégorie de session audio la plus adaptée à votre application.

    Pour cet exemple, nous utiliserons la catégorie de lecture . Cela permet de lire l'audio même lorsque l'écran est verrouillé et avec le commutateur Ring/Silencieux réglé sur silencieux. Puisque nous jouons des vidéos, nous allons utiliser le mode MoviePlayback . Pour rester simple, nous allons mettre ce code dans le délégué de l'application.

    1. Dans votre projet, ouvrez le fichier App Delegate (AppDelegate.swift).

    2. Dans la didFinishLaunchingWithOptions fonction, ajoutez du code pour définir la catégorie de session audio. Assurez-vous d'importer le AVFoundation framework.

      • Nous avons besoin du code ci-dessous afin de garantir que l'audio est lu quand nous l'attendons. Par exemple, sans définir ce code, nous n'entendrons pas la vidéo lorsque le bouton muet est allumé.

      //
//  AppDelegate.swift
//  Simple-Video-Playback
//
//  Copyright © 2018 Brightcove. All rights reserved.
//

import UIKit
import AVFoundation

@UIApplicationMain
class AppDelegate: UIResponder, UIApplicationDelegate {

  var window: UIWindow?

  func application(_ application: UIApplication, didFinishLaunchingWithOptions launchOptions: [UIApplicationLaunchOptionsKey: Any]?) -> Bool {
  	// Override point for customization after application launch.

    var categoryError :NSError?
  	var success: Bool
  	do {
  		try AVAudioSession.sharedInstance().setCategory(.playback, mode: .moviePlayback, options: .duckOthers)
  		success = true
  	} catch let error as NSError {
  		categoryError = error
  		success = false
  	}

  	if !success {
  		print("AppDelegate Debug - Error setting AVAudioSession category.  Because of this, there may be no sound. \(categoryError!)")
  	}

  	return true
  }

    Voir le code

    Le contrôleur de vue est maintenant terminé. Voici le code complet :

    //
//  ViewController.swift
//  VideoCloudBasicPlayer
//
//  Copyright © 2018 Brightcove, Inc. All rights reserved.
//

import UIKit
import BrightcovePlayerSDK

let kViewControllerPlaybackServicePolicyKey = "your policy key"
let kViewControllerAccountID = "your account id"
let kViewControllerVideoID = "your video id"

class ViewController: UIViewController, BCOVPlaybackControllerDelegate {

  let sharedSDKManager = BCOVPlayerSDKManager.shared()
  let playbackService = BCOVPlaybackService(accountId: kViewControllerAccountID, policyKey: kViewControllerPlaybackServicePolicyKey)
  let playbackController :BCOVPlaybackController
  @IBOutlet weak var videoContainerView: UIView!

  required init?(coder aDecoder: NSCoder) {
    playbackController = (sharedSDKManager?.createPlaybackController())!

    super.init(coder: aDecoder)

    playbackController.analytics.account = kViewControllerAccountID // Optional

    playbackController.delegate = self
    playbackController.isAutoAdvance = true
    playbackController.isAutoPlay = true
  }

  override func viewDidLoad() {
    super.viewDidLoad()
    // Do any additional setup after loading the view, typically from a nib.

    // Set up our player view. Create with a standard VOD layout.
    guard let playerView = BCOVPUIPlayerView(playbackController: self.playbackController, options: nil, controlsView: BCOVPUIBasicControlView.withVODLayout()) else {
        return
    }

    // Install in the container view and match its size.
    self.videoContainerView.addSubview(playerView)
    playerView.translatesAutoresizingMaskIntoConstraints = false
    NSLayoutConstraint.activate([
      playerView.topAnchor.constraint(equalTo: self.videoContainerView.topAnchor),
      playerView.rightAnchor.constraint(equalTo: self.videoContainerView.rightAnchor),
      playerView.leftAnchor.constraint(equalTo: self.videoContainerView.leftAnchor),
      playerView.bottomAnchor.constraint(equalTo: self.videoContainerView.bottomAnchor)
    ])

    // Associate the playerView with the playback controller.
    playerView.playbackController = playbackController

    requestContentFromPlaybackService()
  }

  func requestContentFromPlaybackService() {
    playbackService?.findVideo(withVideoID: kViewControllerVideoID, parameters: nil) { (video: BCOVVideo?, jsonResponse: [AnyHashable: Any]?, error: Error?) -> Void in

      if let v = video {
        self.playbackController.setVideos([v] as NSArray)
      } else {
        print("ViewController Debug - Error retrieving video: \(error?.localizedDescription ?? "unknown error")")
      }
    }
  }
}

    Gérer la sécurité des médias

    App Transport Security (ATS) applique des connexions sécurisées entre votre application et les services Web. Avec la sortie d'Apple du SDK iOS 9, une nouvelle fonctionnalité appelée App Transport Security (ATS) a été ajoutée.

    1. L'une des situations suivantes s'appliquera à votre application :

      • Par défaut, le SDK natif Brightcove pour iOS utilise une stratégie de sélection de source pour choisir HTTPS sur les sources HTTP, de sorte que vous pouvez créer vos applications avec ATS activé.

        C'est tout, vous êtes prêt à exécuter votre application.

         

      • Si vos ressources distantes utilisent HTTPS au lieu de sources HTTP, vous pouvez créer vos applications avec ATS activé.

        C'est tout, vous êtes prêt à exécuter votre application.

         

      • Si vous utilisez des sources HTTP ou si vous avez d'autres appels HTTP dans votre application, vous pouvez rencontrer le message d'erreur suivant :

        App Transport Security has blocked a cleartext HTTP (http://) resource load since it is insecure.
Temporary exceptions can be configured via your app's Info.plist file.

        Cela signifie que ATS est activé, mais que votre système n'est pas configuré pour répondre aux exigences ATS. Pour résoudre ce problème, consultez le document Utilisation de la sécurité du transport d'applications (ATS) .
