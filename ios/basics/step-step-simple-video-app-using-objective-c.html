    Étape par étape : Application vidéo simple utilisant Objective-C

    Dans cette rubrique, vous apprendrez comment créer une application de lecture vidéo simple à l'aide du SDK Brightcove Player pour iOS. Vous le construirez en utilisant le langage de programmation Objective-C.

    Présentation

    Cet exemple montre comment lire une vidéo renvoyée à partir de l' API Lecture Brightcove, qui est l'API la plus récente et recommandée pour récupérer du contenu de votre bibliothèque Video Cloud.

    Pour cette option, vous aurez besoin d'une clé de stratégie. Si vous ne connaissez pas les clés de stratégie, reportez-vous à la section Présentation de l'API de stratégie.

    Cet exemple est écrit en Objective-C, mais vous pouvez également expérimenter avec Swift. Voir ce qui suit :
    • Étape par étape። Application vidéo simple utilisant Swift
    • Version rapide du VideoCloudBasicPlayer

    Démarrer

    Cet exemple lit des vidéos à partir d'un tableau interne de fichiers vidéo.

    Suivez les étapes ci-dessous pour vous familiariser avec la configuration d'un projet d'application qui utilise le SDK Brightcove Player pour iOS. Vous pouvez afficher le code complet pour chacun des éléments suivants :

    • Implémentation AppDelegate
    • En-tête View Controller

    • Mise en œuvre View Controller

    Il y a deux façons d'essayer cet exemple :

    Télécharger l'exemple

    Téléchargez l'intégralité du projet Xcode pour l'expérimentation.

    1. Clonez ou téléchargez les exemples de SDK natif pour iOS sur votre système local.
    2. Accédez à l'exemple d'application Player/VideoCloudBasicPlayer/Objc .
    3. Exécutez la pod install commande.
    4. Ouvrez et exécutez l'espace de travail nouvellement créé.

    Pour plus d'informations, consultez le document Exécuter des exemples d'applications pour le SDK natif pour iOS/tVOS .

    Créez l'application en suivant les étapes décrites dans ce guide

    Suivez les étapes ci-dessous pour vous familiariser avec la configuration d'un projet d'application qui utilise le SDK Brightcove Player pour tvOS. Vous pouvez afficher le code complet pour chacun des éléments suivants :

    1. Implémentation AppDelegate
    2. En-tête View Controller
    3. Mise en œuvre View Controller

    Créer le projet

    Configurez un projet dans Xcode. Ajoutez ensuite le SDK Brightcove Player avec toutes les dépendances au projet.

    Configurer le projet Xcode

    Créez un nouveau projet Xcode pour l'application.

    1. Commencez la création d'un nouveau projet iOS dans Xcode. Sélectionnez Créer un nouveau projet Xcode. Alternativement, dans le menu supérieur Xcode, vous pouvez sélectionner Fichier, Nouveau, Projet.

    2. Choisissez Application à vue unique pour le modèle et sélectionnez Suivant.

    3. Définissez les informations du projet comme suit :

      • Nom du produit : Lecture vidéo simple
      • Équipe : aucune

        Le champ Équipe est facultatif. Si vous souhaitez distribuer votre application sur l'App Store, vous devrez sélectionner une équipe à laquelle vous appartenez via le programme Apple Developer. Le nom de l'équipe est requis pour la signature de code. Dans cet exemple, nous allons sélectionner aucun.

      • Nom de l'organisation : nom de votre entreprise
      • Identificateur d'organisation : com.example-company
        Cela rend le nom du produit unique pour l'App Store.

    4. Choisissez l'emplacement où vous souhaitez enregistrer votre projet et le créer.

    5. Maintenant, fermez le projet (oui, fermez-le — c'est important !)

    Ajouter le SDK et ses dépendances à votre projet

    Le moyen le plus simple d'ajouter le SDK et ses dépendances à votre projet est d'utiliser CocoaPods.

    CocoaPods est un gestionnaire de dépendances qui ajoute des bibliothèques à votre projet. Ce n'est pas nécessaire, mais facilite l'installation. Pour installer CocoaPods, consultez les instructions sur le site CocoaPods .

    • CocoaPods version 1.0 ou ultérieure est requise.
    • L'utilisation de CocoaPods (l' pod outil) sur un projet Xcode existant modifiera le fichier de projet. Vous voudrez peut-être faire une sauvegarde avant de le faire.

    1. Dans votre dossier de projet, créez un fichier texte brut appelé Podfile (pas d'extension de fichier).

    2. À l'aide d'un éditeur de texte, ajoutez les lignes de code suivantes au fichier Podfile et enregistrez-le. Ce code effectue les opérations suivantes :

      • Ligne 1 : Pointe vers l'emplacement GitHub pour les spécifications du module Brightcove.
      • Ligne 3 : Définit Pods pour utiliser des frameworks au lieu des bibliothèques statiques.
      • Ligne 5 : Définit la version de la plate-forme iOS.

      • Lignes 7 à 9 : Installez le SDK Brightcove Native Player.

        1. Obtenez la version actuelle du SDK natif à partir du document Présentation de Brightcove Native SDK pour iOS .
        2. Remplacez la valeur '6.3.11' par le numéro de version actuel du SDK.

        3. Pour plus d'informations sur la gestion des dépendances, consultez l'article How To Manage Dependencies with CocoaPods .

         
        La target propriété est utilisée pour CocoaPods version 1.0 et ultérieure.
      source 'https://github.com/brightcove/BrightcoveSpecs.git'

use_frameworks!

platform :ios, '11.0' # (or whatever version you want)

target 'Simple-Video-Playback' do
  pod 'Brightcove-Player-Core', '6.3.11'
end

    3. Ouvrez une session Terminal et accédez à votre dossier de projet Xcode Simple Video-Lecture .

    4. Dans la session Terminal, tapez la commande suivante :

      pod install

      et appuyez sur Retour pour exécuter le it. Vous devriez voir une série de messages dans le terminal indiquant que le SDK Brightcove Player a été ajouté à votre projet.

    5. Notez la dernière ligne, ce qui est important - à partir de ce moment, vous devez ouvrir le fichier Simple Video-Playback.xcWorkspace dans Xcode, pas le fichier Simple Video-Playback.xcodeproj. La pod install commande crée ce .xcworkspace fichier pour votre projet.

    • Si vous créez et modifiez régulièrement des fichiers podfiles, vous pouvez essayer d'utiliser l' application CocoaPods pour effectuer les étapes précédentes.
    • Si vous ne souhaitez pas utiliser CocoaPods, vous pouvez télécharger la dernière version du SDK et l'ajouter manuellement à votre projet.

    Code l'application vidéo

    Créez le code pour une application de lecture vidéo simple.

    1. Dans Xcode, ouvrez le fichier Simple Video-Playback.xcWorkspace .

    Définir le comportement audio de l'application

    La session audio gère le comportement audio au niveau de l'application. En savoir plus sur la classe AVAudioSession .

    Vous pouvez choisir parmi plusieurs catégories et paramètres de session audio pour personnaliser le comportement audio de votre application. Choisissez la catégorie de session audio la plus adaptée à votre application.

    Pour cet exemple, nous allons utiliser AVAudioSessionCategoryPlayback. Cela permet de lire l'audio même lorsque l'écran est verrouillé et avec le commutateur Ring/Silencieux réglé sur silencieux. Pour le garder simple, nous allons mettre le code pour cela dans le délégué de l'application.

    1. Dans votre projet, ouvrez le fichier d'implémentation App Delegate (AppDelegate.m).

    2. Dans la didFinishLaunchingWithOptions méthode, ajoutez du code pour définir la catégorie de session audio. Assurez-vous d'importer le AVFoundation framework.

      • Nous avons besoin du code ci-dessous afin de garantir que l'audio est lu quand nous l'attendons. Par exemple, sans définir ce code, nous n'entendrons pas la vidéo lorsque le bouton muet est allumé.
      • Pour plus de simplicité dans l'échantillon, nous mettons ceci dans le délégué de l'application. Pour plus d'informations sur l'utilisation de cette option dans votre propre application, consultez le Guide de programmation de session audio.
      //  AppDelegate.m
//  Simple-Video-Playback
//
//  Copyright © 2018 Brightcove. All rights reserved.
//

#import "AppDelegate.h"
#import <AVFoundation/AVFoundation.h>

@interface AppDelegate ()

@end

@implementation AppDelegate


- (BOOL)application:(UIApplication *)application didFinishLaunchingWithOptions:(NSDictionary *)launchOptions {
// Override point for customization after application launch.

NSError *categoryError = nil;
BOOL success = [[AVAudioSession sharedInstance] setCategory:AVAudioSessionCategoryPlayback error:&categoryError];

if (!success)
{
		NSLog(@"AppDelegate Debug - Error setting AVAudioSession category.  Because of this, there may be no sound. `%@`", categoryError);
}
return YES;
}

    Personnalisez le projet en fonction de vos valeurs

    Ajoutez des valeurs d'ID de jeton et de playlist pour accéder à votre compte Video Cloud.

    Configurer le lecteur

    1. Sous la init fonction, créez une fonction nommée setup , qui est appelée lorsque l'application se charge.

      • Ligne 40 : Utilise le gestionnaire partagé pour créer un contrôleur de lecture. La BCOVPlayerSDKManager classe est un singleton qui vous permet de créer d'autres objets dans l'écosystème SDK.

      • Ligne 42 : Facultatif : Si vous remplacez la BCOVVideo classe ou si vous n'utilisez pas le lecteur Brightcove et le service de lecture ou le catalogue, vous devez envoyer votre ID de compte Video Cloud à Video Cloud Analytics. Cela vous permet d'afficher les données de cette application dans Video Cloud Analytics.

        Pour plus d'informations sur l'envoi de données à Video Cloud Analytics, reportez-vous au guide Utilisation de Brightcove Analytics .
      • Lignes 44-46 : définissez le délégué et activez les fonctions d'avance automatique et de lecture automatique.

      • Ligne 48 : Initialise le service de lecture à l'aide de votre ID de compte et de votre clé de stratégie.

      - (void)setup
{
_playbackController = [BCOVPlayerSDKManager.sharedManager createPlaybackController];

_playbackController.analytics.account = kViewControllerAccountID; // Optional

_playbackController.delegate = self;
_playbackController.autoAdvance = YES;
_playbackController.autoPlay = YES;

}

    Configurer le lecteur

    1. Dans la viewDidLoad fonction, procédez comme suit :

      • Ligne 56 : Créez et définissez les commandes du lecteur Brightcove en utilisant la disposition VOD standard.
      • Ligne 58 : Ajoute la vue du lecteur en tant que sous-vue de la vue principale.
      • Ligne 59 : Désactivez le masque de redimensionnement automatique.
      • Lignes 60-65 : Utilisez la mise en page automatique pour définir des contraintes dynamiques pour la vue du lecteur.
      • Lignes 66 : Affecte la vue du joueur à sa variable globale associée.
      • Lignes 69 : Associe la vue du lecteur à la manette de lecture.

      • Ligne 71 : Appelle la requestContentFromPlaybackService fonction, que vous définirez à l'étape suivante.

      - (void)viewDidLoad {
  [super viewDidLoad];
  // Do any additional setup after loading the view, typically from a nib.

  // Set up our player view. Create with a standard VOD layout.
  BCOVPUIPlayerView *playerView = [[BCOVPUIPlayerView alloc] initWithPlaybackController:self.playbackController options:nil controlsView:[BCOVPUIBasicControlView basicControlViewWithVODLayout] ];

  _videoContainer addSubview:playerView];
  playerView.translatesAutoresizingMaskIntoConstraints = NO;
  [NSLayoutConstraint activateConstraints:@[
      [playerView.topAnchor constraintEqualToAnchor:_videoContainer.topAnchor],
      [playerView.rightAnchor constraintEqualToAnchor:_videoContainer.rightAnchor],
      [playerView.leftAnchor constraintEqualToAnchor:_videoContainer.leftAnchor],
      [playerView.bottomAnchor constraintEqualToAnchor:_videoContainer.bottomAnchor],
  ]];
  _playerView = playerView;

  // Associate the playerView with the playback controller.
  _playerView.playbackController = _playbackController;

  [self requestContentFromPlaybackService];
}

    Demander du contenu à partir de la bibliothèque Brightcove

    Afin de lire du contenu vidéo, vous devez demander une vidéo à partir de votre bibliothèque Video Cloud.

    1. Sous la viewDidLoad fonction, créez une fonction nommée requestContentFromPlaybackService comme suit :

      - (void)requestContentFromPlaybackService
{
[self.playbackService findVideoWithVideoID:kViewControllerVideoID parameters:nil completion:^(BCOVVideo *video, NSDictionary *jsonResponse, NSError *error) {

	if (video)
	{
		[self.playbackController setVideos:@[ video ]];
	}
	else
	{
		NSLog(@"ViewController Debug - Error retrieving video: `%@`", error);
	}

}];
}

}

{
[self.playbackService findVideoWithVideoID:kViewControllerVideoID parameters:nil completion:^(BCOVVideo *video, NSDictionary *jsonResponse, NSError *error) {

	if (video)
	{
		[self.playbackController setVideos:@[ video ]];
	}
	else
	{
		NSLog(@"ViewController Debug - Error retrieving video: `%@`", error);
	}

}];
}

    Gérer la sécurité des médias

    App Transport Security (ATS) applique des connexions sécurisées entre votre application et les services Web. Avec la sortie d'Apple du SDK iOS 9, une nouvelle fonctionnalité appelée App Transport Security (ATS) a été ajoutée.

    1. L'une des situations suivantes s'appliquera à votre application :

      • Par défaut, le SDK natif Brightcove pour iOS utilise une stratégie de sélection de source pour choisir HTTPS sur les sources HTTP, de sorte que vous pouvez créer vos applications avec ATS activé.

        C'est tout, vous êtes prêt à exécuter votre application.

         

      • Si vos ressources distantes utilisent HTTPS au lieu de sources HTTP, vous pouvez créer vos applications avec ATS activé.

        C'est tout, vous êtes prêt à exécuter votre application.

         

      • Si vous utilisez des sources HTTP ou si vous avez d'autres appels HTTP dans votre application, vous pouvez rencontrer le message d'erreur suivant :

        App Transport Security has blocked a cleartext HTTP (http://) resource load since it is insecure.
Temporary exceptions can be configured via your app's Info.plist file.

        Cela signifie que ATS est activé, mais que votre système n'est pas configuré pour répondre aux exigences ATS. Pour résoudre ce problème, consultez le document Utilisation de la sécurité du transport d'applications (ATS) .
