Structure documentaire

Chez Brightcove, nous nous efforçons de garder la documentation structurée, propre et intuitive. En tant que tel, nous utilisons les README de github pour plus de détails sur tous nos plugins et intégrations. Pour votre commodité, nous avons répertorié et lié ces plugins ci-dessous.

Versions prises en charge

Le SDK Native Player pour iOS prend en charge les versions d'iOS suivantes :

Soutien actif

Brightcove fournit un support actif pour le dernier SDK iOS sur la dernière version publique des versions iOS suivantes :

  • iOS 14
  • iOS 13.7
  • iOS 12.5.3

Assistance passive

Brightcove fournit une prise en charge passive des versions iOS suivantes :

  • iOS 11.4.1

Comprendre les niveaux d'assistance

Le soutien est défini en termes de [version majeure].[version mineure].[version corrigée] d'un OS (par exemple : iOS 11.4.1). Afin de profiter des dernières améliorations technologiques d'Apple, nous proposons deux niveaux de support :

  • actif: Brightcove testera et corrigera les bogues sur ces versions du système d'exploitation.
  • Passif: Brightcove effectuera des tests sur ces versions du système d'exploitation, mais les corrections de bogues seront traitées au cas par cas.

En règle générale, Brightcove fournit une prise en charge active des versions du système d'exploitation utilisées par 80 % de la base d'utilisateurs et une prise en charge passive des autres versions utilisées par au moins 5 % de la base d'utilisateurs.

Commencer

Avant de commencer le développement avec le Player SDK pour iOS, vous devez connaître quelques concepts :

  1. Médias

    La version actuelle du Native Player SDK prend en charge la lecture de contenu à partir de votre compte Video Cloud via l'API de lecture, ou en tant qu'actifs distants avec des URL accessibles sur Internet.

  2. Analytique

    Avec le kit de développement logiciel Native Player, les clients de Video Cloud obtiennent automatiquement des données d'analyse robustes rapportées dans Video Cloud.

  3. SDK principal et dépendances

    Le Brightcove Player SDK pour iOS et ses plugins peuvent être ajoutés à votre projet en utilisant le CocoaPods gestionnaire de dépendances. Les podspecs de tous les composants du Brightcove Player SDK sont hébergés dans le Référentiel BrightcoveSpecs.

    CocoaPods version 1.0 ou plus récente est requis.
  4. Référence du SDK natif

    Pour vous aider à comprendre le SDK, vous pouvez trouver une ventilation détaillée de l'architecture dans le SDK du lecteur pour iOS. Découvrez comment les composants critiques interagissent pour gérer les complexités du contrôle du lecteur vidéo natif. Player SDK pour l'installation, le démarrage rapide et l'architecture iOS

    Ici, vous trouverez également la dernière version bêta du SDK, s'il en existe une.

  5. Présentation du SDK natif

    Pour un aperçu rapide, découvrez comment les composants Brightcove Player SDK interagissent avec les composants iOS natifs. Présentation du SDK du lecteur pour iOS

  6. Questions et réponses

    Rejoignez-nous dans le Forum des développeurs de SDK où les questions liées au SDK sont répondues par les ingénieurs de Brightcove et la communauté. Recherchez les questions auxquelles vous avez déjà répondu ou postez-en une nouvelle. En tant que groupe public, vous pouvez y accéder en fournissant votre adresse e-mail.

  7. Formation vidéo

    Visionnez une série de vidéos de formation qui vous expliquent les étapes de création d'une application Swift dans Xcode qui lit une vidéo à partir de votre bibliothèque Video Cloud.

Des astuces:

  • Les vues et les commandes PlayerUI (y compris les commandes publicitaires) utilisent le guide iOS Safe Layout pour éviter un placement incorrect dans les coins, la zone d'encoche ou sous l'indicateur d'accueil.
  • Le composant Sidecar Subtitles prend en charge la lecture sur AirPlay avec des sous-titres. Pour plus de détails sur la synchronisation de vos sous-titres, consultez le Synchronisation des sous-titres WebVTT document.

Remarques:

  • Pour améliorer la conformité CocoaPods, le SDK natif pour iOS 6.7.7 a renommé certains fichiers Podspec. Pour plus de détails, consultez le Référence du SDK natif.
  • La cible de déploiement minimale pour tous les frameworks Brightcove Native Player pour iOS est désormais iOS 11.0 et tvOS 11.0
  • Le Brightcove Native SDK pour iOS 6.5.0+ nécessite Xcode 11.0 ou supérieur.
  • Le comptage automatique de références (ARC) est requis dans votre projet. C'est la valeur par défaut actuelle pour les projets Xcode.
  • Avec la sortie d'iOS 9 et Xcode 7, une nouvelle fonctionnalité appelée App Transport Security (ATS) a été introduite. En savoir plus sur Utilisation de la sécurité du transport d'applications.
  • Un problème Apple connu avec le contenu en streaming FairPlay est qu'il ne sera pas rendu en mode de mise en miroir AirPlay. Pour plus de détails, consultez le Présentation de la WWDC : Protection du contenu pour HLS (faites défiler environ à mi-chemin de la transcription).

Combiner des plugins

Avec le SDK natif pour iOS, le BCOVSessionProviders de chaque plugin sont créés et enchaînés. Cette chaîne est ensuite utilisée pour créer le BCOVPlaybackController . Pour plus de détails, consultez le Section Combiner des plugins de la référence SDK.

Publicité avec le plugin IMA

L'emplacement des annonces VAST est généralement spécifié par un document VMAP. VAST décrit une annonce tandis que VMAP décrit le placement d'annonces dans une vidéo. Pour plus de détails sur les codes, consultez le BasicIMAPlayer iOS échantillon.

Lorsque vous travaillez avec des annonces IMA, il est recommandé de vérifier que vos annonces fonctionnent en dehors du SDK avant de les mettre en œuvre. Les liens suivants seront utiles pour cela :

  • Validateur de réponse aux annonces Google VAST
  • Guide de la liste de lecture d'annonces vidéo numériques (VMAP)
  • Forum technique Google IMA

Notes de plate-forme

Les notes suivantes sont liées à la plate-forme iOS.

Dupliquer votre écran avec Airplay

La fonctionnalité AirPlay fonctionne immédiatement avec le SDK natif pour iOS, permettant aux utilisateurs de diffuser des vidéos sur des écrans haute définition avec Apple TV.

Vous pouvez utiliser les commandes Brightcove ou utiliser vos propres commandes personnalisées pour AirPlay. Pour plus de détails, consultez le Commandes personnalisées pour AirPlay extrait de code.

L'utilisation d'AirPlay avec des appareils plus anciens (génération 3) peut provoquer un comportement inattendu. L'appareil cible peut ne pas savoir quelles pistes de texte ont été sélectionnées sur l'appareil expéditeur.
